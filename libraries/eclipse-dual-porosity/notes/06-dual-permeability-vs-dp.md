# 06 — Dual permeability (DK) vs Dual porosity (DP)

## TL;DR
- DP (DUALPORO): matrix blocks are isolated; only fracture connects neighbours.
- DK (DUALPERM): matrix blocks also connect to neighbouring matrix blocks (and you can perforate wells into the matrix).
- DK is *not* a different physics model — it's DP plus matrix-matrix transmissibilities.
- Cost: DK is meaningfully more expensive because the linear-solver Schur-complement reduction no longer works.

## What changes structurally

### DUALPORO (DP)
Matrix cell connectivity:
```
              [matrix(i,j,k)] — no edges to neighbours —
                   |                                  |
                  NNC                               NNC
                   |                                  |
              [fracture(i,j,k)] ——— [fracture(i+1,j,k)] ——
                   |                                  
                  ...
```

The Jacobian matrix structure:
```
     [A · m] + [C · f] = [r_m]              A is DIAGONAL — no matrix-matrix flow
     [D · m] + [B · f] = [r_f]              B is banded — fracture has normal connections
```
Where m = matrix unknowns, f = fracture unknowns; A & B are matrix & fracture Jacobian blocks, C & D are diagonal coupling matrices.

Because A is diagonal, you can eliminate m:
```
m = A^(-1) · (r_m − C · f)                  (TD Eq. 2.83)
(B − D·A^(-1)·C) · f = r_f − D·A^(-1)·r_m   (TD Eq. 2.82)
```
The new "effective fracture system" `(B − D·A^(-1)·C)` has the same band structure as B (because `D·A^(-1)·C` is diagonal). Problem size effectively halved.

### DUALPERM (DK)
Matrix cell connectivity:
```
        [matrix(i,j,k)] ——— [matrix(i+1,j,k)] ———       ← matrix-matrix
              |                     |
             NNC                   NNC
              |                     |
        [fracture(i,j,k)] ——— [fracture(i+1,j,k)] ——    ← fracture-fracture
```

A is no longer diagonal. The Schur-complement trick fails. Solve the matrix and fracture systems simultaneously.

### How E100 handles the DK Jacobian
The Jacobian is paired into 6×6 blocks (TD Eq. 2.84 in the 3-phase case):
```
     J = [ A_m   D  ]
         [ C    B_f ]
```
Where `A_m` and `B_f` are the diagonal entries (3×3 each in 3-phase) of the matrix and fracture Jacobians. A fourth nesting is added to the Nested Factorization preconditioner to handle the extra dimension.

The full Jacobian uses 7 bands (J + 6 off-diagonal flow bands for 3 directions × 2 porosities):
```
J = D + L1 + U1 + L2 + U2 + L3 + U3                    (TD Eq. 2.85)
```
The nested factorisation builds an approximation B via:
```
T = D + L3 · S^(-1) · U3              (innermost)
S = T + L2 · R^(-1) · U2
R = S + L1 · Q^(-1) · U1
Q = Γ + L · Γ^(-1) · U                (outermost)        (TD Eq. 2.86)
```
where Γ is a diagonal matrix and L, U keep the same structure as the band terms.

OPTIONS item 60 > 0 restores the pre-97A DK solver. Use only as a fallback if the new solver has convergence issues.

## When to pick DK over DP

| Use DUALPERM when | Why |
|-------------------|-----|
| Matrix-matrix flow is physically real (partial fracturing) | DP makes matrix blocks islands; DK lets them communicate |
| Wells perforate into the matrix (production from low-perm regions) | DP wells can only target fracture cells (except DPNUM regions) |
| Coal-bed methane with instant adsorption (CBMOPTS in E300) | Coal matrix permeability matters; recommended config |
| Coal-bed methane with time-dependent adsorption + multi-porosity | One sub-porosity represents matrix-with-conductivity |
| Reservoir is so heavily fractured that some matrix is still partly continuous | Captures partial connectivity |

| Stay with DUALPORO when | Why |
|-------------------------|-----|
| Fractures fully surround every matrix block (the classical naturally-fractured carbonate) | Matrix communication is geologically absent |
| Cost matters and the assumption holds | Schur reduction halves problem size |
| Single-porosity regions are needed via DPNUM | DPNUM not allowed in DK |
| Discretized matrix (NMATRIX) needed for transient | DK incompatible with discretized matrix |

## Restrictions that disappear under DK
Under DP single-permeability (TD p.114), the following are RESTRICTIONS:
1. Wells can connect only to fracture cells
2. Manual NNCs may not involve matrix cells
3. Every active matrix cell must connect to an active fracture cell

Under DK, restriction #1 lifts entirely. Wells can target matrix or fracture. Restrictions #2 and #3 remain (matrix-matrix NNCs are auto-generated by Eclipse; explicit NNCs to matrix are still forbidden, and orphan matrix cells are still illegal).

## Setup deltas vs DP

```
RUNSPEC
  DUALPERM             ← instead of DUALPORO  (DUALPERM auto-enables DUALPORO)
  
GRID
  SIGMA  0.12 /        ← still need this for matrix-fracture coupling
  DZ ...               ← matrix-matrix flow uses the normal grid Tx, computed from DX/DY/DZ × PERMX/Y/Z just like single-porosity
  
PROPS, REGIONS, ...    ← same as DP
SCHEDULE 
  COMPDAT  WELL1 i j k_matrix k_matrix 'OPEN' ... /      ← can perforate matrix layers
  COMPDAT  WELL1 i j k_frac   k_frac   'OPEN' ... /      ← or fracture layers; up to you
```

## What DK doesn't change
- The matrix-fracture coupling formula (TR = CDARCY·σ·K·V) is identical
- All recovery mechanisms (imbibition, gravity drainage, etc.) work the same
- The Z-doubling rule is identical
- All keyword section assignments are identical

## Compatibility matrix (DK)
DK is **incompatible** with:
- Discretized matrix (NMATRIX with the LINEAR/RADIAL/SPHERICAL geometry)
- DPNUM (single-porosity regions)
- Vertical discretized gravity drainage (GRAVDRB)

DK *is* compatible with:
- GRAVDR (standard gravity drainage)
- GRAVDRM (Quandalle-Sabathier)
- BTOBALFA/V (block-to-block connections)
- INTPC (integrated Pc)
- DPKRMOD (kr modification)
- Most everything else

## CBM-specific guidance (TD p.114)
> "For a coal bed methane model, the DUALPERM option should only be used if the instant adsorption model in Eclipse 300 is activated (CBMOPTS) or if the time-dependent adsorption model is used with Multi-Porosity."

In time-dependent sorption + multi-porosity, one of the sub-matrix porosities represents the pore volume of matrix with conductivity to neighbour matrix grid blocks. Cells flagged as coal get zero transmissibility.

This is a domain-specific subtlety; reach for it only when modelling coal seams.

## Performance hints
- Use the new (post-97A) DK solver — OPTIONS item 60 = 0 (default)
- DK runs are typically 2–4× slower than the equivalent DP run on the same grid
- If convergence is poor, try OPTIONS item 60 > 0 to use the older solver as a fallback
- DK + LGR works but is heavy — consider scale-of-effect carefully
