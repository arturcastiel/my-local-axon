# PR-006 — R1: mandatory per-project workflow file
> Phase: pr · Status: spec-ready · Maps to: R1 (workflow-file-per-project) · Depends on: PR-002 (schema)
> Council finding #7: instantiate cheaply; gate NEW only, WARN existing; validate content not mere presence.

## Objective
Every code-dev project gets a workflow file at scaffold, so the project's process is explicit and reachable. Instantiate the existing canonical (`workspace/domains/code-dev/workflows/code-dev.canonical.yml`, 7 synapses, schema-valid) — don't author from scratch.

## Design (v1)
- **Instantiate by copy.** A verbatim copy of a validated file is valid by construction — avoids the schema-inheritance edge cases of a `parent-workflow` stub. Drift from the canonical is a documented v1 limitation (acceptable; projects may customize).
- **No new tool** — uses library `COPY-FILE` + the existing `workflow_run.validate_draft` (test only). No registry/liveness/count ripple.
- **Gate = WARN-existing / guarantee-new.** Scaffold writes the file for every NEW project; `code-dev status` WARNs when a loaded (older) project lacks it. No hard block on existing projects (council #7).

## Change detail
1. **`workspace/programs/code-dev-new.md`** (not compiled):
   - Add `workflow:        _workflow.yml` to the `_meta.md` WRITE block (records the binding alongside `phase`/`branch`/`codebase`).
   - After the project files, `COPY-FILE("workspace/domains/code-dev/workflows/code-dev.canonical.yml", "{proj-dir}/_workflow.yml")` — instantiate the canonical.
2. **`workspace/programs/code-dev-state-status.md`** (not compiled): add a `WORKFLOW` line — WARN when `{proj-dir}/_workflow.yml` is absent (R1 gate, WARN-for-existing), else show it's present.

## Acceptance criteria
1. The canonical workflow validates (`workflow_run.validate_draft` → no errors).
2. A copy of the canonical also validates (instantiation produces a valid file).
3. `code-dev-new.md` instantiates `_workflow.yml` (copies the canonical) and records `workflow:` in `_meta.md`.
4. `code-dev status` WARNs when `_workflow.yml` is absent.
5. Tests added (Core Rule 13); full crucible green.

## Out of scope
`parent-workflow` inheritance (v2 — avoids copy-drift); workflow-file gate for the *workflow* subsystem (T3 mirror, later); hard-blocking existing projects. No kernel edit, no host-harness change.

## Dogfood
Instantiate `_workflow.yml` for `axon-code-dev-improve` itself (scaffolded pre-R1) so it satisfies its own new rule and `code-dev status` reports it present.
