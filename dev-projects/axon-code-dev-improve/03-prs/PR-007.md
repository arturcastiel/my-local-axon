## Objective
Make work that exists in git but has **no DAG node** (code-first drift) capturable into the DAG — *additively only* (insert new nodes, never overwrite). This is the safe half of R4: `dag.add_node` already supports `origin`/`source-id` (PR-002), snapshot fires on every write (PR-002), the reconciler detects `CODE_FIRST_DRIFT` (PR-004). PR-007 adds the **apply** path under the full 6-gate safety contract.

## Design — extend `tools/git_dag_reconciler.py` (no new tool → no liveness/count ripple)
- New `apply_additive(dag, findings, *, dry_run=True)`: for each `CODE_FIRST_DRIFT` finding (a branch with no node), `dag.add_node(... kind="pr", status="pending", origin="machine", source_id=<branch>, disposition="RETRO-REGISTERED <date>")`. Returns a **plan** (list of would-add). `dry_run=True` → no write.
- CLI `--apply` flag, **default OFF (dry-run)**. With `--apply` → persist via `dag._save_and_render` (snapshot fires via `_atomic_write` → `_rollback.snapshot`, PR-002).
- **Idempotent**: a branch that already maps to a node is skipped → second run is a byte-identical no-op.
- **Surface-split origin (D-021 rule 1)**: only `CODE_FIRST_DRIFT` (git-derived, additive) is auto-written, fail-OPEN. `PHANTOM_MERGE` and `DAG_MERGED_NO_COMMIT` are **never auto-written** (flag-and-HALT) — they implicate existing/hand-authored state → fail-CLOSED.
- `--emit-advisory PATH`: write a machine-readable advisory artifact (findings + timestamp + repo) — the Tier-A advisory-stability artifact.
- Human demands: already covered by the PR-002 primitive `dag add-node --origin human --source-id …` (no new surface needed).

## Mandatory safety gates (council Q3 — all hard-required)
1. **dry-run-default** — write only behind `--apply`.
2. **Idempotency proof** — apply-twice == apply-once (second run zero mutations, byte-identical `DAG.json`); asserted in tests.
3. **Snapshot-before-write** — already wired (PR-002); a test asserts it fires on the apply path.
4. **CONFLICT-HUMAN-EDIT 3-branch coverage** — additive=git-wins-nodeless writes; PHANTOM_MERGE/DAG_MERGED_NO_COMMIT = HALT (no write); never touch a hand-authored field.
5. **Per-node audit** — each added node logged (batch-confirm acceptable per D-021 rule 6).
6. **Surface-split origin** — fail-OPEN additive/git-derived; fail-CLOSED on any hand-authored-field path.

## Acceptance criteria / tests (tests-first)
1. dry-run-default: no `--apply` → nothing written, plan returned.
2. `--apply` on a `CODE_FIRST_DRIFT` branch → node added (origin=machine, source-id=branch, disposition RETRO).
3. idempotency: apply twice → second is a no-op, `DAG.json` byte-identical.
4. snapshot fires on apply (a `.rollback` snapshot exists after `--apply`).
5. `PHANTOM_MERGE` / `DAG_MERGED_NO_COMMIT` are **never** auto-written even with `--apply` (HALT).
6. advisory artifact emitted with `--emit-advisory`.
This test file IS the **injected-drift fixture** (Tier-A): it creates real git branches with no node, proving the detector fires and apply captures them.
