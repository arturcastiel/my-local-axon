## Objective
Add a deterministic local dense index (tools/retrieval_index.py) ONLY if live Wave-1 traces prove sparse TF-IDF is actually losing relevant chunks (context_recall below a pinned floor). The unconditional, dense-independent half — collapsing the two byte-identical cosine TF-IDF copies (dispatch.tfidf_similarity, agent_memory._tfidf_rank) into one shared sparse() — ships now because it is a NET tool-surface reduction with zero dependence on the recall gap. The dense substrate itself is HARD-GATED behind the measurement and DEFERS if recall is already ~1.0, to avoid paying the model-free-determinism and surface-area cost for marginal gain.

## Risk
Part A is low risk (mechanical 2->1 collapse of identical pure functions, locked by an equality test). Part B carries the risks honest_callouts names: (1) DETERMINISM — must use HashingVectorizer not sentence-transformers (confirmed the heavy stack was dropped by PR-130); with hashing, 'dense' is only marginally richer than sparse, which is exactly why the recall-gap precondition exists. (2) SURFACE — adds one tool against a project actively reducing its 162-file tools/; mitigated by the Part-A consolidation and by housing query/dump-case as subcommands. (3) KEYWORD THEATRE — B1 audit row is gameable; mitigated by gating on the live-trace recall delta, never the string. (4) The likeliest honest outcome is DEFER: AXON's corpus is small and sparse already scores B2 green, so live recall may well be ~1.0 — in which case the PR correctly ships Part A only and banks the dense points.

## Proportionality (why this scope)
Scope is split to match evidence. The consolidation (Part A) is strictly smaller-is-better: it REDUCES net surface (2 copies -> 1) and is the only piece justified independent of any measurement, so it ships now. The dense index (Part B) is held to exactly the prompt's HARD PRECONDITION rather than built speculatively — bigger scope (the nominated keystone) is explicitly REFUSED until a measured recall gap justifies it, because leverage.json's own dimension keystones and the codebase's anti-mirage commits both say the live RetrievalCase adapter, not B1, is the true keystone, and B1's only eval-lock (dump-case fixtures) is produced by that adapter. Smaller scope (skip dense entirely) is also wrong: if traces DO show a recall gap, a deterministic hashing index is the proportionate fix. The PR encodes the decision rule (recall>=0.95 => DEFER) so scope self-adjusts to the data instead of being guessed now.
