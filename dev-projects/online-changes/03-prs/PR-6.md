## Objective
Add a deterministic, model-free second-stage reranker (`tools/rerank.py`) that re-scores an already-retrieved candidate list — the `top_matches` dispatch already emits and the TF-IDF order `agent_memory.recall` returns — by reusing `library.rank_candidates` (term overlap) as the second-stage scorer, and emits `{precision_before, precision_after, gain}` via `retrieval_eval.context_precision`. This raises precision for every consumer of those sparse retrievers and turns a low post-rerank `gain` into the F2 weak-evidence signal, flipping audit D1 with a REAL second stage plus a before/after delta gate (not a keyword).

## Approach
Ground: `dispatch.main` builds `scored=[{program,confidence}]` and emits `top_matches=scored[:3]` (dispatch.py:239-243,267); `agent_memory.recall` returns id+score+snippet rows in TF-IDF order. Both are one transform from `retrieval_eval.RetrievedChunk(id,text,score)` (retrieval_eval.py:26-31). The reranker is a pure function over `list[RetrievedChunk]`: it builds candidate dicts `{title:text, abstract:text}`, calls `library.rank_candidates(candidates, library.extract_terms(query))` (library.py:197-208 — ranks by term overlap, highest first, deterministic, pure), and re-orders. Because `rank_candidates` is a different signal (query-term overlap) than first-stage cosine, it can promote a relevant chunk above an incidentally-high-cosine noise chunk. precision_before/after reuse `retrieval_eval.context_precision` (retrieval_eval.py:69-74) over a `RetrievalCase` from the top-k slice — no new metric code. Determinism: `rank_candidates` already does `sorted(..., reverse=True)`; add a stable secondary key (original first-stage score, then id) so equal-overlap ties never reorder nondeterministically. READ-ONLY: the tool only reads a case file and writes stdout — never touches `dispatch-feedback.jsonl` or memory; persistence of the gain is left to wave-1's already-built `retrieval_eval._append_log`, not duplicated. Tool-surface REDUCTION: `rerank` ships as a library plus a `retrieval_eval rerank` subcommand rather than a standalone CLI, so the evaluator stays the single retrieval-metrics entrypoint. The `gain` field is load-bearing: a low/negative `gain` after rerank is exactly the weak-evidence signal wave-2's F2/CRAG verdict consumes — this PR emits it but does NOT wire the verdict (separate F2 PR), keeping scope tight.

## Gate (what proves it landed — metric/behavior, not keyword)
On the real `tests/fixtures/retrieval_eval/agentic_anchor.json` (relevant_ids `{anchor-protocol, adapter-contract}`, plus the score-0.11 `unrelated` noise chunk), construct a degraded first-stage order that ranks the noise chunk into the top-2 and assert `rerank(...)["gain"] > 0` AND `precision_after > precision_before` because the term-overlap second stage demotes `unrelated` (its text shares no query terms) below both relevant chunks. PROOF it landed: `rerank` measurably raises `context_precision` on a real fixture via a real second-stage reorder — NOT the audit D1 `_contains_any(\"rerank\")` keyword (rag_maturity_audit.py:264-265). Secondary gate: `retrieval_eval rerank --case agentic_anchor.json` exits 0 and stdout JSON carries `precision_before`, `precision_after`, `gain`. Crucible: `python3 -m pytest tests/ -q` green, `registry-drift` green (REGISTRY entry added), R_NEW_NEEDS_TEST satisfied. Audit D1 reads 2 as a side effect, but the binding gate is the precision delta.

## Risk
Medium-low. (1) `library.rank_candidates` keys on `{title,abstract,keywords}` plus a DOI/authors bonus — feeding chunk text as title/abstract with no DOI is correct (bonus uniformly 0, no ordering distortion), but the adapter must NOT pass `doi`/`authors` or it would skew scores; the test enforces pure term-overlap behavior. (2) Pure term-overlap can tie when no query term matches a chunk — handled by the stable secondary sort on first-stage score then id (pinned by the determinism test). (3) `synapse_suggest.score_candidate` returns a 3-tuple and needs `state`/`weights`; the `--scorer synapse` path must pass `DEFAULT_WEIGHTS` and `state={\"recent-input\":query}` or it scores all-zero — pinned by the synapse-path test. (4) `context_precision` returns 0.0 on empty retrieved — the top-k slice must be non-empty; guarded.

## Proportionality (why this scope)
Scoped to ONE new pure function plus a subcommand and its test — the minimum that produces a measurable precision delta. NOT bigger: it deliberately does not (a) wire dispatch.py/agent_memory.py to call the reranker (consumer opt-in is later, avoids touching hot paths), (b) add the F2/CRAG accept/ambiguous/reject verdict (separate wave-2 PR — this only EMITS the `gain` that verdict reads), (c) add a dense/cross-encoder reranker (model-bearing, violates model-free determinism). NOT smaller: a pure scorer with no before/after delta would flip the D1 keyword without proving anything — exactly the rubric-gaming the leverage doc forbids; the precision-delta gate on a real fixture makes the capability falsifiable. Surface-area: +1 tool file but 0 new standalone CLIs (exposed as a `retrieval_eval` subcommand), reusing two existing scorers (`library.rank_candidates`, `synapse_suggest.score_candidate`) and one existing metric (`retrieval_eval.context_precision`) rather than new ranking or metric code.
