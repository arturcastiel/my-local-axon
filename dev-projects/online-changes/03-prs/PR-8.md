## Objective
Give AXON a live retrieval conscience: a deterministic accept|ambiguous|reject verdict over a scored RetrievalCase, driven by a retrieval-reflect program that retries on ambiguous and escalates to web_search on reject — but only when the AEGIS policy + autonomous grant authorize the network, then re-scores to prove precision actually rose. The web/network path becomes AXON's first genuinely gated capability, named in _policy.md and locked by a governance-denial test, not a rubric keyword.

## Approach
Ground every piece in real symbols. verdict reuses the accept/qualify/reject pattern the leverage doc cites from library.partition_claims (library.py:174-193) applied to RetrievedChunk.score and the already-computed context_precision from score_case (retrieval_eval.py:123) — model-free, deterministic, no disk, no new scoring substrate. score_live/verdict live INSIDE retrieval_eval (REDUCE surface: subcommands of an existing tool, not a new tools/retrieval_quality.py). The verdict is the single trigger: 'ambiguous'->local retry; 'reject'->bounded web. The web arm is the load-bearing governance wire: retrieval-reflect.md calls aegis_policy.resolve(capability='web', policy=load_policy('_policy.md'), grant_active=<grant>) (aegis_policy.py:47) and acts ONLY on allowed:true. Because 'web' defaults to 'human' in CAPABILITIES and _policy.md sets it to 'grant', resolve() returns allowed:false whenever the grant is absent (line 60-62) — autonomy-off suppresses the network with zero extra code. web_search.py is key-free (DDGS, no secret), so wiring it changes no secret surface; routing it through resolve() means F5 cannot regress F4. After a web fold, re-run score_live and require context_precision strictly above the pre-fallback reject floor — the call only 'counts' if it measurably helped. Outcome is captured via agent_memory.capture(source='decision') (line 332, DETERMINISTIC_SOURCES line 299) so it writes inline (episodic). The F-row flips are earned by REAL surfaces (a working verdict, a real gated web call, a real capture), with audit tokens co-located as a side effect of honest naming, not keyword theatre.

## Risk
Medium. (1) The web arm touches the network — but it is fail-closed twice (capability defaults to 'human'; grant-absent denies) and locked by the governance-denial test, so an autonomy-off run provably cannot reach DDGS. (2) F-row tokens must be co-located in the right searched files or the audit deltas silently fail — mitigated by the F2/F5/F6/F4 assertions reading the live audit, not a fixture. (3) verdict thresholds (0.66/0.33) are a judgment call; if Wave-2's noise-gate lands different thresholds there is a reconciliation cost — kept local + documented. (4) web_search.py errors (DDGS unreachable / ddgs not installed, web_search.py:21,54) must degrade to the unchanged reject verdict, not crash — the EVAL treats a web error as 'fallback did not help' and keeps reject.

## Proportionality (why this scope)
Right-sized: ~2 functions + 2 subcommands inside an existing tool, 1 new program, 1 policy line, 1 cap registration, 1 comment, 1 crucible control, 1 test + 2 fixtures. It deliberately does NOT build tools/retrieval_quality.py (the leverage doc's alternate home) — folding verdict into retrieval_eval REDUCES tool surface and reuses score_case. It does NOT add 'web' to GATED (no spurious coupling to the crucible gate; web needs grant, not green). It does NOT touch web_search.py (key-free already) or the kernel. Bigger would be premature dense/index work (Wave-3, deferred until live recall proves a gap); smaller would drop the governance wire that is the whole point — the first genuinely gated network capability, proven suppressed under autonomy-off.
