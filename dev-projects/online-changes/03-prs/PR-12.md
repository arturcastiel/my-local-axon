## Approach
Three independent, additive edits — none change in-AXON behavior. (a) __all__ in both modules is documentation only: it names the surface a future in-process CI gate or composed tool may rely on, and pins it against silent drift via the new equality test. I list the four metric fns plus latency_ms (line 115), aggregate (142), score_case (123), evaluate (165), and the two frozen dataclasses RetrievalCase (35) / RetrievedChunk (27) — every symbol a caller composing metrics needs, nothing private (_tokens, _load_case, _case_paths, _append_log, build_parser, render_text, main stay out). (b) The try/except guard: on ImportError fall back to AXON_ROOT = Path(__file__).resolve().parent.parent (the file lives at tools/, so parent.parent is repo root — identical to _axon_paths._resolve_axon_root's file branch at _axon_paths.py:25) and MYAXON_ROOT = AXON_ROOT/'my-axon' (identical to _resolve_myaxon_root default, _axon_paths.py:32). Inside AXON the real `from _axon_paths import ...` still wins, so zero behavior change; verified today the standalone copy dies with `ModuleNotFoundError: No module named '_axon_paths'`. (c) KEEP flat sibling imports — do NOT add a tools/__init__.py re-export or package-qualified imports: tools/__init__.py already documents script-mode (sys.path[0]=tools/) as the load-bearing contract and that `import tools.retrieval_eval` is intentionally unsupported until the dispatcher migrates to module-mode; adding a shim would contradict that. rag_maturity_audit is deliberately left non-standalone and no standalone audit module is created. The pip wheel and any packaging/ dir are explicitly DEFERRED until a real external-demand signal — no speculative build artifact, honoring REDUCE-tool-surface.

## Rollback
Fully additive and isolated: revert the single commit. __all__ has no runtime effect (Python only consults it for `from module import *`, which AXON never does). The try/except keeps the real `from _axon_paths import ...` as the winning branch inside AXON, so removing it changes nothing in-tree. The doc line and the two tests are independent. No data migration, no schema, no allowlist, no compiled-artifact recompile.

## Risk
Low. The only behavioral edit is the import guard, and its fallback constants are byte-for-byte the documented defaults of _axon_paths (verified at _axon_paths.py:25 and :32), with the real import still preferred in-AXON. Residual risk: a bare `except ImportError` could mask an unrelated import failure — mitigated by scoping the except strictly to the _axon_paths name and keeping the fallback to two pure Path derivations. __all__ drift is itself caught by test_public_surface_is_declared. No write path, no model dependency, no new dependency (stdlib pathlib only), determinism preserved.

## Proportionality (why this scope)
Smallest move that makes the surface honest. Bigger (a package shim / module-mode dispatcher migration, or a pip wheel + packaging/ dir) is rejected: tools/__init__.py documents flat-import-as-contract — Axis A overclaimed a broken package promise — and the wheel has no demand signal yet, so building it now is speculative surface AXON would have to maintain (source-drift + private-data-leak guards for nothing). Smaller (skip the try/except) leaves the doc's standalone-reach claim false. So: declare the contract (__all__), unlock the one verified portability gap (standalone file run), document the already-true cross-install reach in one line — and add NO new tool, NO new dependency, NO new executable or MCP surface. Net surface delta: +1 import-guard branch, +2 __all__ docs lines, +1 doc line, +2 tests; zero new files.
