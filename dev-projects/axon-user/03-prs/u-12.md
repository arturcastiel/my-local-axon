# U-12 — `code-dev pr` reads phase docs; PR specs link parent phase

**Wave**: U.E · **Depends-on**: U-11 · **Status**: ready

## Why

PR specs in `03-prs/pr-NNN.md` currently link "up" to the flat plan via
text reference. With phases promoted to first-class files (U-11), each PR
should declare its **parent phase** explicitly. This enables:
- `code-dev pr-ready` can fetch phase-level acceptance gates.
- `tools/docgen_verify.py` can enforce link integrity (U-14).
- Agents scanning `03-prs/` know which phase a PR belongs to without
  reading the plan index.

## Evidence

- Current PR template has no `parent-phase` field.
- axon-master PRs reference their wave only in prose ("# Wave 2").

## Design notes

- `code-dev pr` reads the relevant `02-phases/phase-N-*.md` to pull phase
  context (must/nice, entry/exit gates) into the PR detail.
- PR template header gains a `Parent-phase:` line.
- If `02-phases/` is absent (legacy flat plan), `code-dev pr` falls back
  to the prior behavior (read `02-plan.md` wave headings). Backward-compat preserved.

## Pitfalls

- **R (phase ambiguity)**: a PR slated for two phases is illegal in the new
  hierarchy. Mitigation: `code-dev pr` errors out with a clear message and
  exits 0 only after the user picks one.
- **F (template churn)**: only ONE line is added to the PR template header;
  no body restructure.

## Interface sketch

```bash
$ code-dev pr U-10
[reads 02-phases/phase-5-planning-hierarchy.md for context]
[asks any missing slots in the PR detail]
✓ wrote 03-prs/u-10.md with Parent-phase: phase-5-planning-hierarchy
```

PR file head:

```
# U-10 — code-dev plan --mode=strategic writes 02-roadmap.md
**Wave**: U.E  **Parent-phase**: phase-5-planning-hierarchy
**Depends-on**: U-1  **Status**: ready
```

## Spec

### Files

- `workspace/programs/code-dev-pr.md` — read phase doc when `02-phases/` exists.
- `workspace/templates/code-dev-pr-opm.tpl.md` — add `Parent-phase:` to header.

### Acceptance

```bash
code-dev pr U-10
head -3 my-axon/dev-projects/<slug>/03-prs/u-10.md | grep -q "Parent-phase:"
```

### Rollback

`git revert`. Existing PR files without `Parent-phase:` continue to parse —
U-14's link checker treats them as legacy.

### Owner

AGENT writes. HUMAN runs `code-dev pr` against a fixture phase.
