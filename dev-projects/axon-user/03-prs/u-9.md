# U-9 — `startup.md` reader gate + `code-dev new` defaults + journal `# when:`

**Wave**: U.C · **Severity addressed**: S2 (F-012, F-016) + S3 (F-017) · **Depends-on**: —
**Owner**: AGENT · **Status**: ready

## Why

Three onboarding/discoverability fixes uncovered by P1 (novice-naomi) and P3
(careful-cassie):

1. **F-016** — `startup.md` has two audience sections (AGENT vs USER); a
   first-day agent can't tell which applies.
2. **F-017** — `code-dev new` fires 4 sequential QUERY prompts with no
   defaults; `first-phase` should default to `1-design`.
3. **F-012** — `journal-log`, `journal-event`, `journal-decision` semantic
   boundaries are not documented in any program's HELP.

Each is small; grouped because they share the "first-day usability" theme.

## Design notes

- **startup.md**: prepend a 3-line reader-gate paragraph.
- **code-dev-new.md**: extend the slug example string; add a default to the
  `first-phase` QUERY.
- **journal-* HELP**: add a `# when:` line to each program right after `# desc:`.

## Pitfalls

- **A (default surprises power-users)**: `first-phase = 1-design` may not
  match every project. Mitigation: shown in the prompt; user can override.
- **F (when-line semantics)**: the three `# when:` lines must agree with
  actual program behavior. Cross-check against each program's body.

## Interface sketch

`startup.md` head:

```
> **Reader**: If you are a human, read **FOR THE USER** below.
> If you are any AI agent, read **FOR THE AGENT** in order. Step 0 only
> applies if your harness is Claude Code.
```

`code-dev new` prompt:

```
$ code-dev new
Project slug (e.g. my-cool-app; a-z 0-9 -, no dots): tiny-test
Project display name: Tiny Test
Absolute path to codebase: /tmp/tiny
First phase name [1-design]:   ← press Enter to accept
```

Journal headers:

```
# desc:    free-form daily implementation notes
# when:    append-only to 04-log.md; user-typed; not for state changes
```


Per F-012:
- log: `append-only to 04-log.md; user-typed; not for state changes`
- event: `atomic state-change emitted by other programs; internal, rarely user-typed`
- decision: `ADR — durable architectural choice with rationale + alternatives`
