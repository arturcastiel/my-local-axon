# U-4 ‚Äî Drop `state-restore.md`; alias `state-save ‚Üí tag` in desc

**Wave**: U.B ¬∑ **Severity addressed**: S1 (F-007, F-008) ¬∑ **Depends-on**: U-1
**Owner**: AGENT writes; HUMAN approves the deletion ¬∑ **Status**: ready

## Why

PR-27 advertised a state-save / state-restore partner. The shipped reality:

- `workspace/programs/code-dev-state-save.md` is a body-copy of
  `code-dev-tag.md` (tag/snapshot semantics, not project-save).
- `workspace/programs/code-dev-state-restore.md` is a 7-line stub. It calls
  `TOOL(session, transition, "--from frozen --to active --tag {tag}")` with
  flags `session.py` doesn't accept, and it never reads any checkpoint file.

We have two options: (a) implement the real round-trip (feature work ‚Äî out of
scope), or (b) accept that `state-save` aliases `code-dev-tag` and drop the
phantom restore. We choose (b) ‚Äî subtractive fix.

## Evidence

- [findings/F-007](../findings/F-007-state-save-is-tag-copy.md)
- [findings/F-008](../findings/F-008-state-restore-stub.md)
- `diff workspace/programs/code-dev-state-save.md workspace/programs/code-dev-tag.md`
  ‚Äî identical body content.

## Design notes

- After U-1, `state-save.md` has the correct `# PROGRAM:` header. This PR
  then refines the `# desc:` line to document the alias-to-tag relationship.
- `code-dev-tag` provides built-in rewind (`code-dev tag --rewind <tag>`), so
  "restore" semantics are already available under a different verb.
- `code-dev-state-restore.md` is deleted via `git rm`.
- Verify no other file `EXEC`s `code-dev-state-restore` before deletion.

## Pitfalls

- **D3 (silent caller)**: a downstream program or doc might reference
  `code-dev-state-restore`. Pre-deletion grep mandatory.
- **F-F (doc drift)**: CHANGELOG entry for PR-27 advertises state-restore;
  U-V1's errata block will document the withdrawal.
