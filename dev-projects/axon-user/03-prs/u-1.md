# U-1 — Rename-header sweep for PR-26/27/28 umbrella

**Wave**: U.A · **Severity addressed**: S1 (F-001, F-002 cascade) · **Depends-on**: —
**Owner**: AGENT writes diff; HUMAN runs `pytest tests/test_programs_md.py`
**Status**: ready

## Why

PR-26, PR-27, PR-28 renamed 24 program files using `shutil.copy2(old, new)`,
which preserves the file body verbatim. Line 1 of every renamed file is
`# PROGRAM: <old-name>`. If the AXON dispatcher or `call_graph.py` resolves
programs by reading this header (rather than the filename), the entire W4
rename umbrella silently dispatches to the OLD slug — defeating the purpose
of W4 and silently breaking the `code-dev review` router, `code-dev chats`,
`code-dev state-*`, `code-dev journal-*`, `code-dev knowledge-*`,
`code-dev safety-*`, and `code-dev lifecycle-tour`.

This is the **single highest-leverage fix** in the entire axon-user plan.

- 5/5 personas hit this finding in some form.

## Design notes

- Single Python loop with an explicit `MAPPING` dict (filename → old-header).
- `str.replace(old, new, count=1)` to avoid touching any other line 1 string match.
- Header line should match filename stem exactly: filename
  `code-dev-state-save.md` → header `# PROGRAM: code-dev-state-save`.
- No body changes. No `# desc:`, no `EXEC` rewrites. Headers only.
- Verification grep included in acceptance.

## Pitfalls

- **F-D1 (mis-rename)**: typo in MAPPING leaves a header stale. Mitigation:
  exhaustive list in spec + post-sweep grep -L.
- **F-F1 (cheatsheet drift)**: cheatsheet is regenerated automatically by U-9
  / `cheatsheet_gen.py`; no manual sync needed.
- **F-D2 (alias stubs already fine)**: the 14 alias stubs in PR-26/27/28 are
  correctly named (their bodies were *written* not *copied*). This sweep
  targets the *destination* files only.

## Codebase grounding

- The original rename loops live in this conversation's W4 PR-26/27/28
  implementation. The bug is that those loops used `shutil.copy2` without a
  body sed-step.
- `tools/call_graph.py` traverses programs via filename glob — *not* by
  header. So `call_graph` reported clean. That's why the W4 audit passed.
- If the actual program dispatcher (`tools/dispatch.py` if present) reads the
  header, the cascade is real. Even if dispatch is filename-based, the header
  is the single visible declaration in the file and *will* mislead future
  readers and tooling.
