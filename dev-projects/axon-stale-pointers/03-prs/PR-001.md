Add a read-only `pointers` area to tools/self_care.py that cross-checks the four
state stores for coherence and surfaces findings in the standard areas/attention
pattern. This is the detector every other PR in this project surfaces (PR-002),

## Changes Required
### tools/self_care.py
- **What:** new function `pointer_check(ws, projects_dir=None) -> dict` + wire into
  `run_check()` as `areas["pointers"]` with attention lines.
- **How:** four sub-checks, each yielding `{ok, findings[]}` with one fix command per finding:
  1. **active-phase validity** — read `workspace/memory/working/active-phase.md`.
     Parse `{program}:{step}`. Findings: (a) `unknown-program` — non-terminal token
     whose program file is absent from workspace/programs/ (the "code-dev-pr:1"
     incident class); (b) `stale-nonterminal` — non-terminal token whose key-file
     mtime is older than 24h (mechanical staleness proxy — no schema change).
     Terminal tokens (:done/:failed/:aborted) are always ok.
  2. **manifest split-brain** — for every `{projects_dir}/*/` with `_meta.md`
     `status: active`: run phase_model.check(dir); collect `ok:false` (unresolvable
     meta phase) AND the "meta points at a phase already done" note.
  3. **complete-claim reconciliation** — for every project whose `_meta.md` status
     OR phase contains "complete": load manifest; if any phase is not `done`,
     finding `complete-claim-unbacked` listing the non-done phase ids. (The
     axon-obsidian class: prose says complete, manifest says pending.)
  4. **test-run freshness** — compare `workspace/memory/longterm/last-test-run.json`
     `ts` against `git -C <repo> log -1 --format=%ct`. Commit newer than last
     recorded run → `test-record-stale` (advisory; fix: run tests / PR-004 stamps).
- **Why:** one sweep, four stores, zero new top-level surface; every finding is
  actionable next boot.

### tests/test_self_care_pointers.py (new)
- **What:** fixture-driven unit tests, one class per incoherence.
- **How:** tmp workspace + tmp projects dir; cases: (1) non-terminal active-phase
  naming a missing program → unknown-program; (2) terminal token → clean;
  (3) meta "phase: complete" + manifest pr=pending → complete-claim-unbacked;
  (4) meta phase pointing at done phase → note surfaced; (5) last-test-run older
  than HEAD commit → test-record-stale; (6) fully coherent fixture → ok:true,
  zero findings. Follow existing self-care test conventions (claude-dir override
  pattern shows how dirs are parameterized).
- **Why:** tests-with-neurons (Core Rule 13) — no ACTIVE registration without them.
