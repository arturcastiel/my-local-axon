menu-snapshot and a menu.md OS STATE line, plus a guard so a resume offer is never
rendered from an invalid W:active-phase (the 2026-07-09 false-fire class).

- **What:** add `out["pointers"]` — import self_care, call pointer_check() with the

  Follow the prompt_log convention: None on probe failure so menu fallback fires.
- **Why:** one snapshot field = zero extra envelopes at boot (axon-plus pr-6 pattern).

### workspace/programs/menu.md
- **What:** (a) OS STATE: `IF snap.pointers.ok ≡ false → "Pointers ⚠ N stale/incoherent

  resume line ONLY IF snap.pointers.active_phase_valid ≢ false, else render
  "⚠ stale pointer (invalid resume target) — run: self-care report".
- **Why:** stale = seen next boot; invalid pointers stop masquerading as resumable work.
