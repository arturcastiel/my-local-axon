## Summary
Every bare pytest run of the full suite stamps last-test-run.json — the "5296/0/15
invisible run" class dies. Partial runs never masquerade as suite verdicts.

## Changes Required
### conftest.py (new, repo root)
- **What:** pytest_sessionfinish hook: gather passed/failed/skipped from the
  terminal reporter (controller process only under xdist); write
  workspace/memory/longterm/last-test-run.json {suite, ts, verdict, passed,
  failed, skipped, total, duration, source: "conftest"} ONLY when collected >=
  AXON_TEST_RUN_FLOOR (default 1000 — full-suite marker; targeted runs skipped).
  Record path overridable via AXON_TEST_RUN_RECORD (tests). Exception-swallowed
  (sitecustomize precedent): a stamp failure never breaks a test run.
- **Why:** the record follows reality regardless of how tests are invoked.

### tools/test_runner.py
- **What:** keep its stamp (it adds suite names); both writers produce the same
  schema; conftest adds source field — no dedupe conflict (last writer wins,
  both truthful).

### tests/test_conftest_stamp.py (new)
- Subprocess pytest on a 2-test fixture with FLOOR=1 + RECORD=tmp → stamp written,
  fields correct; FLOOR=1000 → no stamp (partial-run guard); broken RECORD path →
  run still green (swallow guard).

## Acceptance Criteria
- [ ] full-suite pytest (however invoked) updates last-test-run.json
- [ ] targeted runs do NOT update it
- [ ] stamp failure cannot fail a test run
- [ ] PR-001's test-record-stale finding clears after the next full suite/gate run

## Risks & Gotchas
- ⚠ xdist: sessionfinish fires on workers too — guard on not hasattr(config,
  "workerinput") so only the controller stamps
- ⚠ coverage sitecustomize interplay: write-only, no imports from tools/
