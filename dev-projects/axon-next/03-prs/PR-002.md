# PR-002 — Build G1c for real: the cited write-barrier
Status: implemented · Wave: A (t3-floor) · Complexity: M · Deps: none

## What (tools/write_barrier.py NEW, tools/shell.py)
- write_barrier.py: the runtime G1c barrier via `sys.addaudithook`. Fires on CPython
  filesystem-mutation audit events (open write-modes, os.remove/unlink/rmdir/rename/
  replace, shutil.* ) and DENIES (PermissionError → hard abort) any mutation under a
  protected subtree (kernel/BOOT/core, workspace/programs/memory/tools/harness), no
  matter how the path was COMPUTED. Audit hooks are un-removable by CPython guarantee,
  so an unattended interpreter run cannot uninstall its own barrier. arm() idempotent;
  extra_roots widens for run-scoped paths; AXON_WRITE_BARRIER=off for tests.
- shell.py cmd_exec: for an unattended run, injects the arm() preamble into the python
  child env (AXON_WB_PREAMBLE) — the residual the static classifier cannot decide is now
  actually closed. Attended dev keeps plain execution.
- The two phantom "OS write-barrier (G1c)" citations in shell.py are now TRUTHFUL
  references to tools/write_barrier.py (council: "build the hook or delete the comments").
