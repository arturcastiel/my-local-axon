# PR-003 — Grant TTL + human-only renewal + ledger reconciliation
Status: implemented · Wave: A (t3-floor) · Complexity: M · Deps: none

## What (tools/autonomous_mode.py, tools/hooks/enforce_pretooluse.py)
- `DEFAULT_TTL_HOURS = 72`; grant_on stamps `ttl_hours`. `_expired()`: granted_ts + ttl
  vs now; legacy grant with no granted_ts → EXPIRED (fail-closed, mirrors run_active's
  legacy posture); ttl_hours<=0 = explicit human-set no-expiry.
- authorized() consults `_expired()` right after the active check: an expired grant
  AUTHORIZES NOTHING. THE G6 INVARIANT (named test): expiry denies ops WITHOUT touching
  run_active() — the unattended-run enforcement (kernel-floor/breaker/cadence) stays
  ARMED so a run halts at a node boundary rather than being disarmed mid-flight.
- `renew()`: HUMAN-ONLY — requires the literal `--human-confirm yes` token; re-stamps
  granted_ts + audits. CLI `renew` action; `status` surfaces `expired`.
- Anti-self-renew hook (enforce_pretooluse.anti_self_renew_block): DENIES
  `autonomous-mode on|renew` / autonomy-contract write while run_active — an unattended
  agent can never grant or renew its own autonomy (red-team: else TTL is theater).
- grant_off reconciles the accountability contract entry (best-effort; ledger, not a
  blocker).
