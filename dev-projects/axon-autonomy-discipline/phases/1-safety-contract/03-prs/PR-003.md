# PR-003 — Autonomy contract (ask for powers)

- **Status:** spec
- **Phase:** 1-safety-contract
- **Complexity:** M
- **Depends on:** none (ready-frontier)
- **Why:** Today AXON's two authority systems are decoupled and there is no interactive "ask for powers"
  flow (01-study B2). The contract unifies them behind one human interview and becomes the overnight ENTRY
  GATE — a run's powers are declared, scoped, and audited up front. Least-privilege: it only selects
  WITHIN `autonomous_mode`'s inviolable floor (kernel-change never delegable; destructive default-off).

## Goal
An interactive **autonomy contract** — a powers interview — that writes the contract: the AEGIS capability
policy (`_policy.md`) + the `autonomous_mode` grant + an `accountability` ledger entry. **Thin-waist:** the
write logic lives in a tested tool (`autonomy_contract.py`); the interview lives in a thin program
(`autonomy-contract.md`) that QUERY/decide-s the human and calls the tool.

## Acceptance criteria
1. `autonomy_contract.write(repo, ops, destructive, budget, level, scope, …)` →
   (a) `_policy.md` written with the capability block for `level`;
   (b) the `autonomous_mode` grant is active for `repo` with `ops` (+ any `destructive`);
   (c) an `accountability` ledger entry is opened recording the granted powers.
   Asserted as OBSERVED effects (read back), not just "the call was made".
2. Least-privilege: `destructive` defaults empty; the tool never adds kernel-change; a `destructive` not in
   `autonomous_mode.DESTRUCTIVE_OPS` is rejected.
3. The program `autonomy-contract.md`: identity lock → interview (repo · scope · level · ops · destructive
   · budget via QUERY/`TOOL(decide)`) → `TOOL(autonomy-contract, write, …)` → render. Static test locks
   the orchestration (it calls decide/QUERY, then the write tool, then DONE).
4. No new crucible control: "is the run authorised" is already the reanchor's `autonomous` gate (the grant
   being active); the contract is what CREATES the grant.

## Changes Required

### tools/autonomy_contract.py  (new)
- `write(*, repo, ops, destructive, budget, level, scope, workspace, myaxon=None)` → writes `_policy.md`
  (capabilities mapped from `level`), calls `autonomous_mode.grant_on(...)`, opens an `accountability`
  entry; returns `{ok, policy_path, grant, ledger_ref}`. Reject a `destructive` op not in
  `autonomous_mode.DESTRUCTIVE_OPS`. CLI: `write --repo … --ops … --destructive … --budget … --level …
  --scope … [--workspace …] [--myaxon …]`.

### workspace/programs/autonomy-contract.md  (new)
- Thin orchestration: identity lock → interview → `TOOL(autonomy-contract, write, …)` → output.

### tools/REGISTRY.json + workspace/programs/REGISTRY.json
- Register the tool + (via programs_registry generate) the program.

### tests/test_autonomy_contract.py  (new)
- Tool effect test (tmp workspace+myaxon): assert all 3 effects + least-privilege rejection.
- Program static test: the interview → write-tool → DONE control flow.

## Test plan
`pytest tests/test_autonomy_contract.py` → green; neuron-audit the program → PASS; full suite (gate's
pytest) green; `crucible gate` → parse `passed` SEPARATELY; doc-anchors clean. No dev-mode (tools/ +
workspace/programs/ + tests/).
