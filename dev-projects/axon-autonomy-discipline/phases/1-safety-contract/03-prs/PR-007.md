merge. The repo-wide flip is therefore a code change: invert each rule's `_required()` to **BLOCK by
default**, with the flag (file or ctx state) now an explicit **opt-OUT** — `value: false` degrades that
rule to WARN locally; absent / anything-else → BLOCK.

Rules flipped: `R_CODE_CHANGE_REQUIRES_PR_PHASE`, `R_AUTONOMY_CADENCE`, `R_AUTONOMY_BREAKER`.

## Safety (verified before flipping)

`turn-count` absent → cadence `since=0` (not lapsed); breaker state empty (not tripped). So the flip does
not brick the gate, and PR-007's own on-workflow changeset passes at BLOCK.
