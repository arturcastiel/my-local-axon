## Goal
Extend `autonomy-reanchor` (today: identity + grant + project-anchor) to ALSO re-assert **workflow
position**: in an autonomous run with a loaded project, require an active phase AND ≥1 open PR spec (the
same weakest-sound predicate PR-001 enforces at the gate). Fail CLOSED (halt) on off-workflow.

## Acceptance criteria
1. autonomous + project loaded + active phase + open PR spec → `ok: true` (on-workflow).
2. autonomous + project loaded + **no active phase** → `ok: false`, violation mentions workflow position.
3. autonomous + project loaded + active phase + **no open PR spec** → `ok: false`, violation.
4. **not autonomous** → workflow-position NOT checked (identity-only, unchanged).
5. The program HALTs on `ok: false` (it already does — verify the new violations trigger it).
6. Reuses PR-001's predicate helpers (DRY; makes the PR-001 dependency concrete).
