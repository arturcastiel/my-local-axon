# PR-004 ‚Äî Circuit breakers

- **Status:** spec
- **Phase:** 1-safety-contract
- **Complexity:** M
- **Depends on:** PR-003 (the contract supplies the budget; breakers enforce run limits)
- **Why:** Halt-and-surface, never push through. Directly from lessons L1 (I committed a RED gate twice by
  retrying) and L7 (fail-open seams). A run that keeps hitting red must STOP and surface, not grind on.

## Design (mechanism now, signal in PR-005)
Breakers are run-STATE machines (they trip on accumulated gate outcomes), so this PR builds the tested
mechanism ‚Äî a state machine `autonomy_breaker.py` (record gate outcomes; check the trip conditions) + a
changeset rule `r_autonomy_breaker.py` that BLOCKS at the gate when the breaker is tripped. PR-005's
operate-loop wires the SIGNAL (records each gate's outcome / PR landed). Until then the breaker is a tested
tripwire over an empty state (no-op) ‚Äî the same mechanism/signal split as the cadence (PR-005/006).

## Trip conditions (v1)
- **same-change red ‚â• 2** ‚Äî a change whose gate went RED is retried at most once; a 2nd red on the SAME
  change-id trips (the L1 lesson). change-id = a stable hash of the sorted changed paths.
- **consecutive gate failures ‚â• N** (default 3) ‚Äî too many reds in a row ‚Üí stop.
- (deferred, noted) budget-exhausted (needs the contract budget in state) + out-of-scope (needs a
  structured scope allow-list) ‚Äî extensible conditions wired once PR-005 provides the signals.

## Acceptance criteria
1. `autonomy_breaker.record(ws, change_id, "red"|"green")` updates per-change reds + consecutive-fails
   (green resets consecutive); persisted to `workspace/memory/working/autonomy-breaker-state.json`.
2. `autonomy_breaker.check(ws, change_id)` ‚Üí `{tripped, reason}`: trips on same-change red ‚â•2 OR
   consecutive ‚â• N; else not tripped. Asserted as observed state (record ‚Üí check).
3. `r_autonomy_breaker.check(ctx)` ‚Üí Violation (halt-and-surface) when the breaker is tripped for the
   changeset's change-id; None otherwise. WARN‚ÜíBLOCK via `L:autonomy-breaker-required`.
4. Empty state (nothing recorded) ‚Üí not tripped ‚Üí rule is silent (no false-positive before PR-005).
