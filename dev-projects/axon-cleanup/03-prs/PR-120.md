# PR-120: Resolve 30 Unknown TOOL('shell') refs

**Wave:** 1 — Corpus sweep
**Status:** SHIPPED
**Depends on:** PR-101..118

## Goal

30 programs call `TOOL(shell, cmd=...)` but `shell` is not in `REGISTRY.json`. Either `shell` should be registered (real tool we forgot to declare) or each caller should be rewritten to use an existing tool.

## Touch

- **Step 1:** inspect — is there a `tools/shell.py` on disk? If yes, register it in `REGISTRY.json` with a smoke entry. If no, decide on a replacement (`bash`/`run`/`_run`).
- **Step 2:** apply the chosen path. If registering: smoke test in `tests/test_tool_smoke_gaps.py`. If rewriting: ~30 caller edits.

## Tests (mandatory per axon-tests PR-020)

- axon-audit `Unknown TOOL` count drops to 0 for `shell`.
- If new tool registered: add to `tests/test_tool_smoke_gaps.py`'s TOOLS_TO_SMOKE list.

## Doc co-output (mandatory)

- Update `workspace/AXON-DOCS-COMPILER.md § Tool registry` and the chosen tool's doc. Guarded-by row added.

## Notes / risks

Decide step 1 before drafting. If `shell` semantically means "run a bash one-liner", prefer rewriting to whatever the canonical shell-exec tool is (no new tools rule).
