# PR-140: CI: regenerate compiled snapshot + diff-assert

**Wave:** 3 — Hardening
**Status:** SHIPPED
**Depends on:** PR-121

## Goal

User decision Q4: keep + CI regenerate. Add a CI job that re-runs `compile_optimizer` and asserts the committed snapshot under `generated/compiled/` is byte-identical to the freshly compiled output.

## Touch

- `.github/workflows/ci.yml` — new job `compiled-snapshot-fresh`:
  - Run `python3 tools/compile_optimizer.py --check` (new flag) or compile then `git diff --exit-code generated/compiled/`.
- `tools/compile_optimizer.py` may need a `--check` flag if it doesn't have one.

## Tests (mandatory per axon-tests PR-020)

- New `tests/test_compiled_freshness.py` (smoke) that does the same check locally.

## Doc co-output (mandatory)

- Update `workspace/AXON-DOCS-COMPILER.md § Snapshot regeneration` with Guarded-by row.

## Notes / risks

If `compile_optimizer` is non-deterministic, this gate will be flaky. Verify determinism before landing.
