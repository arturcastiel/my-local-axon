## Council finding (decisive)
PR-004's seal/is_advisory_verdict machinery was DEAD CODE on the live path: the deliberator
neuron built the verdict as a dict literal (no 'kind') and the router surfaced it with no
discriminator check. The CLI/import path was hardened; the actual fail-open path (neuron→router)
was not. An in-process self-attested "receipt" is theater (the agent controls the process) —
un-forgeable provenance must be out-of-band (harness-minted ledger, signed, per-seat, nonce).

## Change (council-adopted scope)
1. hr-team-deliberator.md: stamp the 'kind: advisory_verdict' discriminator on the verdict (seal it).
2. hr-team.md (router): refuse to surface any result whose kind ≠ advisory_verdict (fail-closed) —
   closes the dead-code hole; an unsealed/fabricated dict literal can no longer be surfaced.
3. tools/hr_team.py: verify_surfaceable(verdict, require_receipt=False) — the consumer/surface gate
   (discriminator + advisory_only; + ledger-verified receipt when require_receipt). verify_fanout_receipt()
   — the receipt-verification skeleton, fail-closed (no valid receipt can exist without a real backend).
4. Tests: gate accepts sealed / rejects unsealed/fabricated; receipt fail-closed; self-attested ≠ valid.
