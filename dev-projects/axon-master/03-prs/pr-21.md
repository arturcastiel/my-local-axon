# pr-21 — Token-ceiling + usage aggregator

**Wave**: W3 · **Goals**: G.tok.08, G.obs.02 · **Depends-on**: PR-13 (usage log), PR-20 (per-program budgets)

## Why (problem statement)
Per-program budgets (PR-20) declare expected token cost; usage log (PR-13) records actual. Need an aggregator that compares them and a project-level `token-ceiling` default that gates against runaway sessions.

## Evidence (from studies)
- `helpers/cd-gap-c3-p4-cost-budgeting.md` → token-ceiling per project; aggregator `meta usage --by=...`.
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.tok.08, G.obs.02.

## Design notes
- `tools/usage.py.aggregate(by='program'|'session'|'day', since=<date>)` returns `[{key, runs, in_tokens, out_tokens, cache_read, cache_creation, vs_budget}]`.
- `vs_budget` is the actual/declared ratio (PR-20 declares).
- `_meta.md.token-ceiling: 32000` (default; PR-3 migrator sets it on new projects).
- `workspace/programs/code-dev-meta-usage.md`: prints aggregates in 3 layouts (`--by program / session / day`).
- Warns when any program's mean usage > 1.5 × budget over 7 days.

## Pitfalls
- Provider returns nulls for cache_* → aggregator skips nulls cleanly.
- Empty log on first run → graceful "no data yet".

## Interface sketch
```text
$ code-dev meta usage --by program --since 7d
program             runs  in_tokens  out_tokens  cache_read  vs_budget
code-dev-status      24      28,800      7,920      19,200      0.6×
code-dev-pr-review    8     180,400     32,400     124,000      1.4×  ⚠ over budget
…
```
