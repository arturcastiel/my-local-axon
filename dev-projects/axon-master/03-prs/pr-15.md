# pr-15 — Compaction recovery + sess.04 harness

**Wave**: W2 · **Goals**: G.sess.04, partial G.sess.02 (doc) · **Depends-on**: PR-9 (`_session.md`)

## Why (problem statement)
PR-9 lays the per-chat session record. PR-15 closes the loop: detects compaction (process restart while `_session.md.state == active`), runs a recovery flow that announces what the previous "incarnation" was doing, and offers to resume. Also writes the first version of `AXON-DOCS-SESSIONS.md` so the model is documented. The 2026-05-15 persona-drift incident is exactly this scenario — compaction + no anchor → drift.

## Evidence (from studies)
- `helpers/cd-gap-c3-p2-session-model.md` → state enum transitions; "recovered" state semantics.
- `helpers/cd-gap-c2-p4-failure-modes.md` → F-C4 (compaction loses state), F-A1 (persona drift after compaction).
- User memory `/memories/operational-safety.md` → "AXON workspaces — compaction == cold boot" — explicit rule.
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.sess.04 acceptance: "synthetic compaction → recovery announces last action + pending verbs".

## Design notes
- `tools/session.py` gains `recover()`:
  - If `_session.md.state == active` AND current process ID differs from `last-pid`, flip state → `recovered`.
  - Read last 3 checkpoints and `pending-actions`.
  - Emit a banner via boot: `"Compaction detected. Last action: <…>. Pending: <…>. Resume? [y/N]"`.
- `tests/fixtures/compaction/`: synthetic project with `_session.md` in `active` state and a fake `last-pid`. Test forces recovery path and asserts the banner contents.
- `workspace/AXON-DOCS-SESSIONS.md`:
  - Documents the state machine.
  - Documents the recovery flow.
  - References the user-memory rule.
  - PR-33 expands this doc.

## Pitfalls (from failure-mode catalog)
- **F-A1 persona-bleed after compaction** → recovery banner is the in-product anchor.
- **F-C4 compaction loses state** → checkpoints + recovery.
- False-positive recovery (process really did continue) → PID check filters.

## Interface sketch
```text
$ python3 axon.py boot        # post-compaction
Compaction detected.
  last action:    code-dev pr-review 3 --phase=3 @ 2026-05-17T15:22Z
  pending:        code-dev pr-review 3 --phase=4
                  code-dev pr-ready 3
Resume? [y/N] y
✓ session state: active → recovered
```
