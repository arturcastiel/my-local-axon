# pr-20.7 — Study-evals workstream (NS-1)

**Wave**: W3 · **Goals**: NS-1 (R5 next-study seed) · **Depends-on**: PR-13 (usage), PR-25 (idempotence harness shape)

## Why (problem statement)
R5 `cd-study-c4-p3-next-study.md` lists NS-1 ("study output evals") as the top recommended next-study. Without scoring study outputs against fixtures, we cannot tell if a `study --mode=subsystem` run is regression-free across kernel updates. Evals close this loop.

## Evidence (from studies)
- `helpers/cd-study-c4-p3-next-study.md` → NS-1 top pick: "evaluate study outputs against labeled fixtures".
- `helpers/cd-study-c4-p4-web-findings.md` → eval frameworks (langfuse, promptfoo).
- `helpers/cd-gap-c2-p3-test-surface.md` → Tier 3 ("behavioral T3"): golden outputs.

## Design notes
- `tools/study_evals.py`:
  - Iterates `tests/fixtures/study-evals/<repo>/` (small representative codebases).
  - For each fixture, runs `study --mode={overview, subsystem, deep}` and scores against `expected.md` using:
    - Structural-match: section presence (10 expected sections).
    - Key-fact coverage: ≥ N expected facts found (regex/string match).
  - Emits JSONL to `my-axon/log/study-evals/<date>.jsonl`.
- Fixture repos: ≥ 5 (e.g. `tiny-py-cli`, `tiny-ts-lib`, `flask-mini`, `axon-fixture-A`, `axon-fixture-B`).
- Read-only on the codebases under test.

## Pitfalls
- Flaky model output → score allows 80% match threshold.
- Large fixture sizes → keep each fixture ≤ 50 files.

## Interface sketch
```text

✓ all 15 evals passed (5 fixtures × 3 modes)
```
