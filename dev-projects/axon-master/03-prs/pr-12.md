# pr-12 — Rename-safety harness

**Wave**: W2 · **Goals**: G.umb.04, G.test.07 · **Depends-on**: PR-1 (T1 ground truth)

## Why (problem statement)
R3 proposes 30+ renames (Wave 4 of this plan executes them). With 57 programs and ~300 internal `EXEC` cross-refs, a rename that misses one reference will break dispatch silently (F-D2). There is no harness today to detect this. The harness must exist before PR-14 (router stubs) and any rename PR (PR-26..28) ships.

- `helpers/cd-wf-c3-p2-rename-proposal.md` → full rename table (30+ moves).
- `helpers/cd-tools-p3-migration.md` → "rename-safety harness" listed as Wave-1 prerequisite for tool reorganization.
- `helpers/cd-gap-c2-p4-failure-modes.md` → F-D2 "stub forwards to wrong target".

## Design notes
- `tools/rename_snapshot.py`:
  - `snapshot`: walks `workspace/programs/*.md`, emits JSONL row per program: `{program, desc, sections, status, exec_refs, registry_entry}`.
  - `diff <pre> <post>`: compares two snapshots; emits regression lines (renames where `desc` or `sections` lost; refs that no longer resolve).
- Snapshot stored at `tests/snapshots/programs-pre-rename.jsonl` (committed).
- `tests/test_rename_safety.py`: integration test runs `snapshot` on current tree, diffs against committed snapshot; fails if regression.
- Workflow for rename PRs (PR-14, 26..28):
  1. Run `rename_snapshot snapshot --out pre.jsonl` BEFORE rename.
  2. Perform rename (file move + EXEC ref updates + REGISTRY.json edit).
  3. Run `rename_snapshot snapshot --out post.jsonl`.
  4. `rename_snapshot diff pre.jsonl post.jsonl --allow-rename old→new` confirms intentional renames; flags anything else.
- Snapshot file checked into repo so PR-12 establishes the "ground truth" of current state.

## Pitfalls (from failure-mode catalog)
- **F-D2 stub forwards to wrong target** → diff catches.
- **F-F3 tour gets stale** → out of scope; PR-31.5 handles cross-ref lint of tour.
- Renames legitimately change names → `--allow-rename old→new` whitelist.

## Interface sketch
```text
$ python3 tools/rename_snapshot.py snapshot --out pre.jsonl
✓ snapshot of 57 programs → pre.jsonl

$ # …perform rename: code-dev-tag → code-dev-state-save…

$ python3 tools/rename_snapshot.py diff pre.jsonl post.jsonl --allow-rename code-dev-tag→code-dev-state-save
✗ broken EXEC ref:
  workspace/programs/code-dev-status.md line 42:  EXEC(code-dev-tag) — target missing
fix: update reference, or add stub for code-dev-tag.
exit 1
```

## Spec (canonical)
- **Files**:
  - new: `tools/rename_snapshot.py`, `tests/test_rename_safety.py`, `tests/snapshots/programs-pre-rename.jsonl`.
- **Acceptance**:
  1. Snapshot captures `{program, desc, sections, status, exec_refs}` for every program.
  2. Diff fails on regression (missing program, broken EXEC, lost section).
  3. `--allow-rename a→b` whitelists intentional moves.
  4. Test `test_rename_safety.py` runs in CI flow.
  5. `tools/lint_paths.py` clean.
- **Rollback**: revert; rename PRs lose their safety net (only T1 catches some regressions).
- **Owner**: AGENT writes; HUMAN sanity-checks snapshot once.
