# pr-13 — Usage logging

**Wave**: W2 · **Goals**: G.obs.01 / D-A2 · **Score**: 4.0 (R2 #3) · **Depends-on**: none

## Why (problem statement)
`compile-suggest` and the rest of AXON's optimizer surface are **blind today** — there are no per-program execution counters or token measurements. R2 ranks D-A2 (usage instrumentation) at score 4.0, #3 on the executive backlog. Every downstream observability and perf PR (PR-19 dispatch quality, PR-20 budgets, PR-21 token-ceiling aggregator, PR-15.6 igap feedback, PR-20.5 caches, PR-34.5 cheatsheet auto-sort by frequency) depends on this data being collected.

## Evidence (from studies)
- `helpers/cd-c4-p3-improvements.md` Rank 3 → D-A2 "Record `usage` for every code-dev run (compile-suggest blind today)" score 4.0.
- `helpers/cd-c2-p1-internals.md` → "`compile-suggest` reads no usage data — recommendations are heuristic only".
- `helpers/cd-c3-p4-web-findings.md` → JSONL pattern for agent observability (one line per run, append-only, easy to grep/aggregate).
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.obs.01.

## Design notes
- `tools/usage.py` (modified): exposes `log_run(program, in_tokens, out_tokens, cache_creation, cache_read, duration_ms, ts)` that appends one JSONL line to `my-axon/log/usage/<YYYY-MM-DD>.jsonl`.
- Schema:
  ```json
  {"ts":"2026-05-17T14:22:31Z","session":"chat-…","program":"code-dev-status","in_tokens":1200,"out_tokens":340,"cache_creation":null,"cache_read":null,"duration_ms":420}
  ```
- Nulls accepted (Anthropic cache-* metrics depend on provider API).
- `tools/dispatch.py` wraps every code-dev invocation with `log_run`.
- Daily-rolled file; no rotation needed (file size will be modest).
- Read API: `usage.aggregate(by='program'|'session'|'day', since=<ISO>)` for PR-21.

## Pitfalls (from failure-mode catalog)
- **F-H1 secret in log** → out of scope; usage log records counts/timing only, no content.
- **F-B3 corruption** → JSONL append-only; atomic_write per line (single `write+fsync`).
- Provider API may return `null` for cache metrics → schema accepts nulls.

## Interface sketch
```text
$ tail -1 my-axon/log/usage/2026-05-17.jsonl
{"ts":"2026-05-17T14:22:31Z","session":"chat-abc","program":"code-dev-status","in_tokens":1240,"out_tokens":380,"cache_creation":null,"cache_read":820,"duration_ms":402}

$ python3 -c "from tools.usage import aggregate; print(aggregate(by='program', since='2026-05-01'))"
{"code-dev-status": {"runs": 24, "in_tokens": 28800, "out_tokens": 7920}, …}
```

## Spec (canonical)
- **Files**:
  - modified: `tools/usage.py`, `tools/dispatch.py`.
  - new: `my-axon/log/usage/.keep`.
- **Acceptance**:
  1. JSONL `{ts, session, program, in_tokens, out_tokens, cache_creation, cache_read, duration_ms}` (nulls allowed).
  2. ≥ 1 turn logged after the PR merges (sanity check).
  3. `aggregate(by=...)` API works.
  4. `tools/lint_paths.py` clean.
- **Rollback**: revert; observability vanishes; downstream PRs lose their data source.
- **Owner**: AGENT writes; HUMAN runs `code-dev status` once and confirms file written.
