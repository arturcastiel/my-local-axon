# pr-16.5 — Plan DAG emission (`tools/plan_dag.py`)

**Wave**: W2 · **Goals**: G.plan.06 (DAG emission), partial G.plan.02 · **Depends-on**: PR-16 (plan modes), PR-8 (per-PR file format established by then)

## Why
`code-dev plan` claims "PRs ordered by dependency" but emits no graph. Users can't visualize critical path, blockers, or parallel-safe sets. The plan v5 acceptance ("DAG acyclic; verified topologically") is asserted, not checked. PR-16.5 makes it machine-checked and renderable.

## Evidence
- Audit finding (this session, 2026-05-16): no `DAG.md`, `DAG.json`, or Mermaid block in any axon-master artifact; only inline `**Depends-on**:` lines in per-PR files.
- `helpers/cd-study-c4-p2-targets.md` → T-S1 ergonomics: "plan output should be navigable".
- `helpers/cd-wf-c1-p1-canonical-flows.md` → WF1/WF3 both start by reading the plan; a graph is the natural index.

## Design notes
- `tools/plan_dag.py`:
  - Walks `<project>/03-prs/pr-*.md`.
  - Parses `**Depends-on**: <list>` line from each file (regex: `\*\*Depends-on\*\*: (.+?)(?:·|$)`).
  - Normalizes tokens (`PR-9.5`, `pr-9.5`, `pr-9.5 (note)` → `pr-9.5`); strips parenthetical commentary.
  - Drops `none`, `all prior PRs merged`, free-text waveclose dependencies → wave-close PRs depend on the union of their wave's PRs.
  - Forward-reference comments ("PR-17 will fill _index.md") are NOT treated as deps; only the `**Depends-on**:` line counts.
  - Validates acyclicity (Kahn's algorithm); exits 1 on cycle, naming the cycle.
  - Computes critical path (longest path by edge count).
  - Emits two artifacts:
    - `03-prs/DAG.md` — Mermaid `graph LR` + wave-grouped topo table + critical-path highlight + leaf set (parallel-safe entry points).
    - `03-prs/DAG.json` — `{"nodes": [...], "edges": [[from, to], ...], "topo": [...], "critical_path": [...]}`.
- `code-dev plan` (PR-16 router) gains a `--emit-dag` flag (default true post-PR-16.5).
- `tests/test_plan_dag.py`:
  - Fixture project with 3 PRs (linear), validates topo.
  - Fixture with synthetic cycle → exits 1.
  - Fixture mirroring axon-master sample (5 PRs) → asserts JSON shape.

## Pitfalls
- `**Depends-on**:` line format drift → strict regex; lint fails loudly with line number.
- Wave-close bumpers (`pr-v1..v4`) have "all prior" deps → handled by special-cased token expansion, not free-text parsing.
- Mermaid file grows unwieldy for > 100 nodes → group by wave with subgraphs; collapsible in GitHub render.
- Forward references in narrative ("PR-X will fill Y") falsely parsed → only the dedicated `**Depends-on**:` line is the source of truth; document this clearly in `AXON-DOCS-PLAN.md` (PR-23).

## Interface sketch
```text
$ python3 tools/plan_dag.py --project axon-master
✓ parsed 51 PRs
✓ DAG acyclic (51 nodes, 78 edges)
✓ critical path: pr-1 → pr-2 → pr-3 → pr-9 → pr-15 → pr-33 → pr-34 → pr-v4 (8 hops)
✓ wrote 03-prs/DAG.md (Mermaid + topo table)
✓ wrote 03-prs/DAG.json

$ python3 tools/plan_dag.py --project axon-master --check
✓ acyclic
$ echo $?
0
```

## Codebase grounding
- **new**: `tools/plan_dag.py` — argparse: `--project <slug>`, `--check` (silent acyclicity), `--output {md,json,both}`. Walks `{my-axon/dev-projects/<slug>}/03-prs/pr-*.md`.
- **parse**: regex `\*\*Depends-on\*\*:\s*(.+?)(?:\s*·|$)` against each file's header block; tokenize on `,`; strip parentheticals via `re.sub(r"\s*\([^)]*\)", "", token)`; normalize `pr-N` (case-insensitive); drop `none`, `all prior PRs merged`.
- **algorithm**: Kahn topological sort (`collections.deque`, in-degree dict); on cycle, walk back and print path.
- **critical path**: longest path on the DAG via DP on topo order.
- **outputs**: `{project}/03-prs/DAG.md` (Mermaid `graph LR` with wave subgraphs + topo table + critical-path highlight) and `DAG.json` (matching schema in current seed file [`DAG.json`](DAG.json)).
- **modify**: [`workspace/programs/code-dev-plan.md`](../../../../workspace/programs/code-dev-plan.md) — after writing `02-prs.md`, call `TOOL(plan_dag)` if `03-prs/` exists.
- **modify**: [`tools/REGISTRY.json`](../../../../tools/REGISTRY.json) — register `plan_dag`.
- **new tests**: `tests/test_plan_dag.py` + fixtures `tests/fixtures/plan_dag/{linear,cycle,sample}/03-prs/pr-*.md`.
- **manual seed exists**: [`DAG.md`](DAG.md), [`DAG.json`](DAG.json) — use as golden output for axon-master fixture parity.
