# pr-31 — Context-switch ergonomics (`chats list / show / switch`)

**Wave**: W4 · **Goals**: G.sess.05, partial G.wf.07 · **Depends-on**: PR-9 (`_session.md`), PR-9.7 (`meta context use`)

## Why
PR-9.7 lets users switch project context. PR-31 lets them switch chat context within a project: `chats list / show / switch` so a developer can jump between in-flight chats (one PR per chat).

## Evidence
- `helpers/cd-gap-c3-p2-session-model.md` → "chats are first-class; need list/show/switch".
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.sess.05.

## Design notes
- `workspace/programs/code-dev-chats.md` (router).
  - `list`: all `_session.md` files under current project; columns `{chat-id, state, started, last-action, last-program}`.
  - `show <id>`: prints last 5 checkpoints + pending actions.
  - `switch <id>`: writes `W:code-dev-chat`, loads state.
- `code-dev-next.md` augmented to mention other open chats with pending actions.

## Pitfalls
- Cross-chat data leakage → switch always goes through `session.transition()` (PR-9).
- Stale chat (state=closed) → `list` hides by default; `--all` shows.

## Spec
- **Files**: new `workspace/programs/code-dev-chats.md`; modified `code-dev-next.md`.
- **Acceptance**:
  1. `chats list` shows active+frozen by default.
  2. `chats show <id>` prints last 5 checkpoints.
  3. `chats switch <id>` updates `W:code-dev-chat`.
  4. `tools/lint_paths.py` clean.
- **Rollback**: revert.
- **Owner**: AGENT writes; HUMAN exercises switching.
