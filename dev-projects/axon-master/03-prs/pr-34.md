# pr-34 — Docgen verify

**Wave**: W4 · **Goals**: G.doc.12 · **Depends-on**: PR-23, PR-33

## Why
With 7 AXON-DOCS files plus cheatsheet, cross-refs proliferate. Without an automated verifier, doc drift is inevitable.

## Evidence
- `helpers/cd-gap-c3-p3-documentation.md` → "docgen verify: cross-ref lint across all AXON-DOCS".
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.doc.12.

## Design notes
- `tools/docgen.py.verify()`:
  - Walks `workspace/AXON-DOCS-*.md`.
  - Verifies every internal link/anchor resolves.
  - Verifies referenced program names exist in `workspace/programs/`.
  - Verifies referenced helper paths exist.
- Exit non-zero on broken refs; report counts.

## Spec
- **Files**: modified `tools/docgen.py`.
- **Acceptance**:
  1. `docgen verify` lints cross-refs across all AXON-DOCS-*.md.
  2. Reports broken refs with file:line.
  3. Exit 1 on regression.
  4. `tools/lint_paths.py` clean.
- **Rollback**: revert.
- **Owner**: AGENT writes; HUMAN runs once.

## Codebase grounding
- **modify**: [`tools/docgen.py`](../../../../tools/docgen.py) (751 lines today, already generates architecture diagrams + program catalog) — add `verify` subcommand:
  - Walks `workspace/AXON-DOCS-*.md` files.
  - For each markdown link `[txt](path)`, asserts target exists. For anchor link `[txt](file#anchor)`, asserts H2 with matching slug exists.
  - For program-name mentions (`code-dev-<X>`), asserts file present in [`workspace/programs/`](../../../../workspace/programs/).
  - For helper paths (`helpers/<X>`), asserts present under project's `helpers/`.
- **CI integration**: `tests/test_docgen_verify.py` — invokes verify; non-zero exit fails CI.
- **registry**: `docgen` already registered; no REGISTRY.json change.
