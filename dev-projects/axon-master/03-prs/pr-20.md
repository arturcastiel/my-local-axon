# pr-20 — Per-program budget blocks

**Wave**: W3 · **Goals**: G.tok.05 · **Depends-on**: PR-2 (gate exists)

## Why (problem statement)
Compile gate (PR-2) blocks regression at the byte level but says nothing about **per-program input/output budgets** — yet R2 cycle-3 measured wide variance: e.g. `pr-review` is a 30-60 KB context consumer where `status` is 1-2 KB. Without declared budgets, `compile-write` can't warn when a program's compiled output exceeds its declared cache-prefix and `meta usage` (PR-21) has no anchor for "expected vs actual" reporting.

## Evidence (from studies)
- `helpers/cd-c3-p1-tokens.md` → measured per-program byte/token tables; large variance.
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.tok.05 (per-program token-budget header).
- `helpers/cd-gap-c3-p4-cost-budgeting.md` → "budget block in frontmatter, single source of truth".

## Design notes
- Every `workspace/programs/code-dev*.md` gains frontmatter:
  ```
  budget:
    input-cap: 8000        # tokens
    output-cap: 2000
    cache-prefix: 2048     # bytes of static prefix that should be cache-hit
  ```
- `tools/budget_lint.py`: AST-checks every program; missing or invalid block → fail.
- `tools/compile-write.py` warns (not blocks) if compiled output exceeds `cache-prefix`.
- Initial values: heuristic from PR-2 audit + observed usage; HUMAN reviews PR.
- The block is **declarative** — no runtime enforcement yet (PR-30 lands per-mode enforcement).

## Pitfalls (from failure-mode catalog)
- **F-C2 static-prefix drift** → `cache-prefix` declares expected length; PR-2 lint enforces stability.
- **F-C3 token-budget overflow** → declarations make overflow detectable post-hoc.
- Wrong initial values → declared as advisory; PR-30 promotes to enforced after one round of tuning.

## Interface sketch
```text
$ head -6 workspace/programs/code-dev-status.md
# PROGRAM: code-dev-status
# desc:    show current project state and next-action
status: active
budget:
  input-cap: 4000
  output-cap: 800
  cache-prefix: 1024
```

## Spec (canonical)
- **Files**:
  - modified: every `workspace/programs/code-dev*.md` (frontmatter).
  - new: `tools/budget_lint.py`.
- **Acceptance**:
  1. Every program has a valid `budget:` block.
  2. `tools/budget_lint.py` exits 0 on clean tree, non-zero on missing/invalid.
  3. `compile-write.py` warns if compiled > `cache-prefix`.
  4. `tools/lint_paths.py` clean.
- **Rollback**: revert frontmatters (and lint); budgets advisory only.
- **Owner**: AGENT writes; HUMAN reviews initial values.
