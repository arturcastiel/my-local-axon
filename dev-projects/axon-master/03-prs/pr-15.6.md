# pr-15.6 — `igap` feedback from code-dev low-confidence

**Wave**: W3 · **Goals**: D-A3 (R2 #14) · **Depends-on**: PR-13 (usage), PR-15.5 (events bus)

## Why (problem statement)
When `code-dev` hits a HALT-with-QUERY (low-confidence dispatch, ambiguous state, conflict during merge), the moment is lost: no structured record, no later review. R2 D-A3 proposes wiring those moments to AXON's `igap` ("information-gap") log so that:
- The user (or a later analysis pass) can review patterns ("we keep halting on…").
- Future automation can target the recurring blockers.

## Evidence (from studies)
- `helpers/cd-c4-p3-improvements.md` Rank 14 → D-A3 "Feed `igap` from code-dev low-confidence moments" I=4/E=2.
- `helpers/cd-c2-p3-gaps.md` → "low-confidence dispatch is fired silently — no analytics surface".
- `tools/igap.py` already exists in repo as an absent-instruction log.

## Design notes
- `tools/igap.py` gains a `record_halt(program, query, context)` API.
- Code-dev programs invoke `igap.record_halt(...)` at every HALT-with-QUERY point: dispatch ambiguity, missing input, conflict during cascade, governance HALT, ≤ 80% rule-filter trigger from PR-11.
- Igap log row: `{ts, program, project, kind: "halt-query", query, context_keys}`.
- `code-dev meta igap`: lists recent entries (last 20 by default), with `--since=7d` filter.
- One synthetic test fires a HALT in a fixture and asserts the entry lands.

## Pitfalls (from failure-mode catalog)
- **F-A3 hallucinated tool output** → igap is itself the proper response to "I'm not sure": log + ask, do not fabricate.
- Sensitive data in query/context → redact via PR-5 (`redact()`) before write.

## Interface sketch
```text
$ code-dev meta igap --since=7d
2026-05-17T16:02  code-dev-plan         halt-query  "rule R-INLINE-1 filtered 90% of options"
2026-05-15T11:14  code-dev-pr-respond   halt-query  "reviewer comment ambiguous — which file?"
```

## Spec (canonical)
- **Files**:
  - modified: `tools/igap.py`; code-dev programs add `record_halt` at QUERY points.
  - new: `workspace/programs/code-dev-meta-igap.md`.
- **Acceptance**:
  1. HALT-with-QUERY events write a JSONL row to igap log.
  2. `code-dev meta igap` lists last N entries; `--since` filter works.
  3. Redaction (PR-5) applied before write.
  4. One synthetic test passes.
  5. `tools/lint_paths.py` clean.
- **Rollback**: revert; HALTs stay invisible.
- **Owner**: AGENT writes; HUMAN reviews after a real HALT.
