# pr-22 — `rules audit`

**Wave**: W3 · **Goals**: G.gov.03 · **Depends-on**: PR-4 (rules schema)

## Why (problem statement)
Two rules can contradict (F-E3). A rule can also become dead (never referenced after 90 days). Today nothing detects either. `rules audit` is the catalog hygiene checker.

## Evidence (from studies)
- `helpers/cd-gap-c2-p4-failure-modes.md` → F-E3 rule contradiction (Class E).
- `helpers/cd-gap-c3-p1-governance.md` → "rules audit: contradictions + dead rules".
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.gov.03.

## Design notes
- `tools/rules.py.audit()`:
  - Pairwise check: any two rules with same `applies-to` and opposing predicates → flag.
  - Reference scan: rule never logged in `_actions.log` or `rules.log` for > 90 days → "dead" warning.
  - Output: human-readable report with rule IDs, line numbers, suggested action.
- `workspace/programs/code-dev-rules-audit.md` invokes the audit.

## Pitfalls
- False contradictions (rules are semantically distinct but textually opposing) → audit reports as warning, not error; HUMAN judges.
- Dead-rule detection too aggressive → 90-day threshold; configurable in `_meta.md`.

## Interface sketch
```text
$ code-dev rules audit
2 warnings:
  R-002 (no migrations without restore test) ⚠ contradicts R-007 (skip restore for dry-run PRs)
  R-005 dead (last referenced 2026-01-15, 122 days ago)
```

## Spec
- **Files**: modified `tools/rules.py`; new `workspace/programs/code-dev-rules-audit.md`.
- **Acceptance**:
  1. Detects synthetic contradiction in a 2-rule test fixture.
  2. Flags dead rule after 90 days of non-reference.
  3. Output human-readable; exit 0 always (advisory).
  4. `tools/lint_paths.py` clean.
- **Rollback**: revert.
- **Owner**: AGENT writes; HUMAN exercises with real rules.

## Codebase grounding
- **modify**: `tools/rules.py` (PR-4) — add `audit()` returning `{contradictions: [...], dead: [...]}`.
  - Contradictions: pairwise check rules with overlapping `applies-to`; flag opposing predicates (regex on the boolean of predicate text or explicit `permits` vs `forbids`).
  - Dead detection: scan `_actions.log` for last reference to rule ID; > 90 days since last hit → dead.
- **new**: `workspace/programs/code-dev-rules-audit.md` — invokes audit; emits markdown report.
- **thresholds**: configurable via [`workspace/preferences/`](../../../../workspace/preferences/) new `rules.md` pref file (parsed by existing [`tools/prefs.py`](../../../../tools/prefs.py)).
