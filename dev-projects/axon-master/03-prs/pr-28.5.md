# pr-28.5 — PR-ergonomics suite (`pr sync` / `pr drift` / `pr export` / `pr suggest-reviewer` / `review --mode=coverage`)

**Wave**: W4 · **Goals**: G-I3, G-I5, G-I9, G-I11, G.wf.09-11 · **Depends-on**: PR-14 (`pr` umbrella), PR-9.5 (`pr list`)

## Why
R4 industrial-gap study identified **five missing PR-flow commands** that engineers expect from a modern PR tool. Together they close the CI-awareness + portability + review-quality gaps.

## Evidence
- `helpers/cd-wf-c2-p1-industrial-gaps.md` → G-I3 review coverage, G-I5 suggest-reviewer, G-I9 drift, G-I11 export.
- `helpers/cd-wf-c2-p2-ci-cd-integration.md` → `pr sync` design: pull CI checks via `gh pr checks`, parse JSON.
- `helpers/cd-wf-c4-p2-roadmap.md` Wave 3 → 5 PRs listed.

## Design notes
- 5 commands, each a thin program + small tool:
  - **`pr sync N`** — `gh pr checks` (graceful skip if `gh` absent) → CI status into `_meta.pr-N.ci-status`.
  - **`pr drift N`** — spec-vs-code drift: diff PR spec acceptance against actual diff; emits unmet items.
  - **`pr export N`** — single self-contained markdown packet (spec + diff + tests + reviewer state).
  - **`pr suggest-reviewer N`** — reads `_actions.log` history + CODEOWNERS if present; suggests reviewer.
  - **`review --mode=coverage`** — reads `--input coverage.json`; emits coverage delta + flags untested new lines.
- Splittable into 3 sub-PRs (28.5a/b/c) if review burden is heavy:
  - 28.5a: sync + drift
  - 28.5b: export + suggest-reviewer
  - 28.5c: review-coverage

## Pitfalls
- `gh` missing → graceful skip with explanatory message; never block.
- CODEOWNERS missing → fallback to most-frequent reviewer from `_actions.log`.
- Coverage tool variance → `--input` accepts any JSON with `{file: {line: hit_count}}` shape; documented.

## Interface sketch
```text
$ code-dev pr sync 3
✓ gh pr checks → 4 ✓ · 1 pending · 0 failed
✓ wrote _meta.pr-3.ci-status (synced 2026-05-17T16:42Z)

$ code-dev pr drift 3
✗ spec says "tests/test_migrator.py covers --restore" — no such test found in diff.
✓ all other acceptance items reflected in diff.
```

  1. `pr sync N` pulls CI; graceful when `gh` absent.
  2. `pr drift N` detects spec-vs-diff drift.
  3. `pr export N` produces self-contained markdown.

- **Owner**: AGENT writes; HUMAN runs each against a real PR.

## Codebase grounding
- **new programs**: under [`workspace/programs/`](../../../../workspace/programs/):
  - `code-dev-pr-sync.md` — calls `gh pr checks --json` via `TOOL(shell, ...)`; parses; writes `_meta.pr-<N>.ci-status: {pass, pending, fail counts, last-sync}`. Graceful skip if `gh` absent (`command -v gh` check).
  - `code-dev-pr-drift.md` — reads PR spec acceptance items, scans `git diff`, flags unmet items.
  - `code-dev-pr-export.md` — emits self-contained markdown (spec + diff + tests + reviewer-state) at `my-axon/exports/pr-<N>-<date>.md`.
  - `code-dev-pr-suggest-reviewer.md` — reads `_actions.log` for reviewer history + optional `CODEOWNERS`; ranks.
  - `code-dev-review-coverage.md` — reads `--input coverage.json` (Cobertura-shaped or `{file: {line: hits}}`); compares to git diff lines; flags untested new lines.
- **new tools**: `tools/pr_sync.py`, `tools/pr_drift.py`, `tools/pr_export.py`.
- **register**: each in [`tools/REGISTRY.json`](../../../../tools/REGISTRY.json).
- **splittable**: if review is heavy, split into 28.5a (sync+drift), 28.5b (export+suggest-reviewer), 28.5c (review-coverage).
- **router integration**: subs of `code-dev-pr.md` umbrella (PR-14).
