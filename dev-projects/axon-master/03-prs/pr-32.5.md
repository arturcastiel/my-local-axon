# pr-32.5 — Nightly `shadow refresh` cron

**Wave**: W4 · **Goals**: T-F1 (R2 #8), partial scheduler hardening · **Depends-on**: PR-13 (usage to record runs), PR-20.5 (shadow LRU)

## Why
Shadows go stale; without a nightly refresh, the first interactive use of the day pays the cold-cache cost. R2 ranks T-F1 (cron nightly `shadow refresh`) at score 3.0, #8 on the executive backlog.

## Evidence
- `helpers/cd-c4-p3-improvements.md` Rank 8 → T-F1.
- `helpers/cd-c2-p1-internals.md` → shadow system tutorial; staleness model.

## Design notes
- `tools/cron.py` gets a `shadow-refresh` registration helper.
- `tools/shadow.py.refresh(project)` is callable from cron.
- New `workspace/scheduler/shadow-refresh.cron.md` declares the cron entry (the AGENT can't install crontab; emits the snippet for the HUMAN to install).
- Active projects: `_meta.status == "active"`. Opt-out via `_meta.shadow-cron: false`.
- Results to `my-axon/log/cron/<date>.jsonl`.

## Pitfalls
- Cron clashes with active session → write a `.lock` file; refresh skips if user session is active.
- Long-running refresh → log start+end; bound at 5 min per project.

## Interface sketch
```text
$ cat workspace/scheduler/shadow-refresh.cron.md
# Install with:
#   crontab -e
#   0 2 * * *  cd /mnt/c/projects/axon && python3 tools/shadow.py refresh --all-active >> my-axon/log/cron/$(date +\%F).jsonl 2>&1
```

## Spec
- **Files**:
  - modified: `tools/cron.py`, `tools/shadow.py`.
  - new: `workspace/scheduler/shadow-refresh.cron.md`.
- **Acceptance**:
  1. Cron entry refreshes shadows for `status=active` projects.
  2. Opt-out via `_meta.shadow-cron: false`.
  3. Results to `my-axon/log/cron/<date>.jsonl`.
  4. AGENT emits install snippet; HUMAN installs cron.
  5. Lock file prevents collision with active session.
  6. `tools/lint_paths.py` clean.
- **Rollback**: revert; shadows go stale.
- **Owner**: AGENT writes; HUMAN installs cron.
