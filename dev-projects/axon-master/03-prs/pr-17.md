# pr-17 — `study/_index.md` + staleness flags + journal vocabulary

**Wave**: W2 · **Goals**: G.study.02 (full), G.study.03, T-S0.4 · **Depends-on**: PR-3 (schema; creates empty `study/_index.md` skeleton), PR-8 (study modes write to study/)

## Why (problem statement)
After PR-3 + PR-8 each project has a `study/` folder with one file per study run, but **no index of what's there or how old it is**. R5 mandates `study/_index.md` as the universal entry point: every run appends a row, every consumer (`state next`, `pr ready`, `meta board`) reads it. Staleness flags close F-E4 (stale study not flagged): a 92-day-old auth study should not be treated the same as a fresh one.

## Design notes
- `study/_index.md` schema (one block per run, append-only):
  ```
  ### overview · 2026-05-17T14:22Z
  mode: overview
  output: engineering
  target: (full repo)
  tokens: 4200
  status: complete
  last-run: 2026-05-17T14:22Z
  ```
- `tools/study_index.py`:
  - `append(run_metadata)`: atomic_write + read-modify-write; locks via `study/_index.md.lock` (PID file).
  - `list(project)`: returns `[{mode, target, last-run, age-days, staleness}]`.
  - `stale_for(target_path)`: returns rows where staleness ∈ {stale, strict-block} touching `target_path`.
- Staleness thresholds (configurable in `_meta.md.study-staleness`):
  - fresh ≤ 30 days
  - warn 30-60 days
  - stale 60-90 days
  - strict-block > 90 days
- All timestamps UTC ISO 8601.
- Journal-vocabulary doc (T-S0.4) inline in `code-dev-study.md`:
  - `study.overview`, `study.subsystem.<target>`, `study.deep.<target>`, etc.
  - Used by `journal search --kind=study.overview` to find past runs.
- `code-dev-study.md` modified to emit a journal event with the canonical kind on every run.

## Pitfalls (from failure-mode catalog)
- **F-E4 stale study not flagged** → primary fix.
- **F-B2 concurrent index write** → atomic_write + lock file.
- Threshold tuning wrong → user-configurable in `_meta.md`.

## Interface sketch
```text
$ cat study/_index.md
# Study index — axon-master

### overview · 2026-05-17T14:22Z
mode: overview · output: engineering · status: complete · age-days: 0 · staleness: fresh

### subsystem/auth · 2026-02-14T09:10Z
mode: subsystem · target: src/auth · age-days: 92 · staleness: strict-block

$ python3 -c "from tools.study_index import stale_for; print(stale_for('src/auth'))"
[{"mode": "subsystem", "target": "src/auth", "age-days": 92, "staleness": "strict-block"}]
```

## Spec (canonical)
- **Files**:
  - new: `tools/study_index.py`.
  - modified: `workspace/programs/code-dev-study.md` (auto-append + journal-event + vocab inline).
- **Acceptance**:
  1. Index lists every study with `last-run`, `age-days`, `staleness`.
  2. Thresholds 30/60/90 enforced.
  3. UTC ISO 8601 timestamps.
  4. `stale_for(target)` API works.
  5. Journal vocabulary (T-S0.4) documented inline in `code-dev-study.md`.
  6. Atomic append under concurrent writes (test).
  7. `tools/lint_paths.py` clean.
- **Rollback**: revert; index stays empty; consumers gracefully degrade.
- **Owner**: AGENT writes; HUMAN runs `code-dev study` once and checks index.
