# PR-002 — Memory-key reader/writer lint (unguarded-orphan ERROR + allowlist + baseline)
Status: implemented · Wave: A (contract-guard) · Complexity: L · Deps: none

## What
- `tools/memory_key_lint.py` (new, lint family, registered): every `RETRIEVE(W|L:key)` site is
  classified. ERROR = UNGUARDED read (no `|` fallback) of a key with NO findable writer, not
  `W:_*`-prefixed, not declared, not baselined. Guarded orphans = advisory only (196 today —
  the `| default` optional-config idiom; the naive rule's 67% noise class, pinned rejected by test).
- Writers recognized: `STORE(scope:key)` in scanned .md sources; `"W:key"`/`"L:key"` literals in
  axon.py/tools/boot.py; `TOOL(memory, set, "--scope X --key K")` call-sites; MYAXON.md STORE lines.
- Spec-reference docs (LANG/TRANSLATE/GRAMMAR/COMPILER) excluded — example snippets, not programs.
- `tools/config_keys.json`: DECLARED runtime-contract keys (boot-seeded: W:tool-registry,
  W:ws-programs · protocol-seeded: W:user-reply, W:text · user-config: L:prompt-log-enabled).
  Declaration ≠ suppression: dead reads go to the baseline and get fixed, never here.
- `tools/memory_key_baseline.json`: 46 grandfathered entries, identity (scope,key,file),
  line-number-free. Real findings owned by PR-010 (L:cron-jobs in menu + compiled mirror),
  PR-018 (arg-seeded unguarded reads get `| QUERY` guards; quickstart L:health-score), et al.
- Crucible: registered WARN (`promotes_on`: baseline empty → BLOCK at PR-019). REGISTRY.json entry.
- IO-lint conformance: baseline writes via `_axon_io.atomic_write_json`.
