# PR-009 — Shell-result lint: no substring success-sniffing after TOOL(shell)
Status: implemented · Wave: C (safety) · Complexity: M · Deps: none

## What (tools/shell_result_lint.py — new, lint family, registered)
Mechanical guard for cross-cutting pattern 2 (bugfix01 H25 recurring). Narrow by council
false-positive bar — two kinds only:
1. success-render-after-unchecked-shell: bare TOOL(shell,…) (result discarded) followed
   within 5 lines by a "✓" render.
2. shell-result-never-referenced: captured result never mentioned again (dead capture).
Value-reads of shell output ({git-branch} interpolations, stdout grepping) are legal and
never fire — the first draft's "never-checked" kind measured 39 hits, mostly legitimate
value-reads, and was narrowed to 3 true findings before landing.
- One true finding in this wave's own rewrite fixed at source (r-origin best-effort now
  logged on failure); 2 pre-existing dead captures baselined
  (tools/shell_result_baseline.json), owned by the Wave F sweep.
- Registered: crucible WARN (promotes_on: baseline empty → BLOCK), REGISTRY.json + mirror,
