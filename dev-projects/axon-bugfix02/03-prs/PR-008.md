# PR-008 — my-axon-init: close the data-loss windows
Status: implemented · Wave: C (safety) · Complexity: M · Deps: none

## What (workspace/programs/my-axon-init.md)
- `fresh-no-prompt` (migrate-workspace automation) keeps its existence-guard bypass but
  now runs in IDEMPOTENT-ENSURE mode: the shared SECTION:ENSURE-TREE only creates what is
  missing — the unconditional `echo '[]' >` that TRUNCATED a populated event log on any
  re-run is existence-guarded, as are .gitignore/INDEX creates.
- Prompt dispatch has an explicit `F` branch and an else → re-prompt (SECTION:PROMPT);
  a typo can no longer fall through into the FRESH writes.
- CLONE path routes through ENSURE-TREE before WRITE-MYAXON: a cloned repo lacks its
  gitignored dirs (memory/local, memory/working, log/turns, …) so downstream writes used
  to target nonexistent paths. MYAXON.md rewrite in clone mode is intentional (paths are
  machine-specific).
- Unknown --mode FAILs loud instead of falling through.

## Verification
- 3 content pins (guarded event-log write; explicit dispatch + re-prompt; clone→ensure-tree).
