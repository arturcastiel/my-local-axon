# PR-4 — Parser INCLUDE + property pass-through

> Wave 2 · Theme: real parser
> Branch: `pr-04-grdecl-include-props`
> Status: finalized (drift-fixed 2026-05-21; open Qs resolved; ready for implementation)
> Dependencies: PR-3 (merged · 6d9aeac · GH #2)

## One-liner
Extend the GRDECL reader to follow `INCLUDE 'file.inc'` recursively,
and to slurp scalar property keywords (PORO, PERMX, PERMY, PERMZ, NTG,
SATNUM …) into `Grdecl.props: dict[str, np.ndarray]`.

## Why this PR
Real reservoir decks are composed: a top-level `.grdecl` file pulls in
geometry, properties, and regions via `INCLUDE`. PR-3 stops at single-
file decks, which is enough for fixtures but not for production data.

Properties (PORO, PERMX/Y/Z, NTG, SATNUM) are also a contract for the
later property pass-through (PR-12). Storing them as raw `np.ndarray`s
keyed by Eclipse keyword name keeps PR-3 + PR-4 logic small and pushes
shape/active-cell filtering into PR-12.

## Decisions baked in

- **INCLUDE paths are resolved relative to the file containing the
  INCLUDE statement**, not the top-level file. Matches Eclipse semantics
  and most MRST loaders.
- **No quote handling beyond the simplest case.** INCLUDE filenames are
  expected to be `'name.inc'` or `"name.inc"`; spaces inside filenames
  are not supported (raise `GrdeclSyntaxError` if the filename token
  contains whitespace inside its quotes).
- **Cycle detection is mandatory.** A `set[Path]` of resolved-canonical
  paths in the parser stack — re-include of an already-open file raises
  `GrdeclIncludeError`.
- **Property keywords are recognized by a positive list**, not by
  "anything float-shaped." This keeps the unrecognized-keyword behaviour
  intact for genuinely unknown keywords.
- **Property arrays are stored 1-D, length `NX·NY·NZ`.** No reshape, no
  active-cell filtering, no unit conversion. Those are PR-12's job.
- **`Grdecl.unrecognized` semantics are unchanged from PR-3.** Property
  keywords now graduate out of `unrecognized` into `props`; everything
  else still goes to `unrecognized`.
- **D-PR4-1 — Duplicate property keyword = WARN, not raise.** Real decks
  legitimately override properties via late INCLUDE files; raising would
  reject valid production data. (Resolves Q-PR4-1 in spec's favour.)
- **D-PR4-2 — Defer EQUALS/COPY/ADD/MULTIPLY operator keywords to a
  later PR (out of v0.1.0).** They are not needed for the smoke/parser
  path; bundling them now would double PR-4's scope. (Resolves Q-PR4-2.)
- **D-PR4-3 — Integer properties (SATNUM/PVTNUM/EQLNUM/FIPNUM) use int32,
  not int8.** PR-3's `_kw_ints` returns int8 (sufficient for ACTNUM ∈ {0,1}).
  SATNUM region IDs commonly exceed 127, so reusing `_kw_ints` would
  overflow silently. Implementation choice: either parametrize
  `_kw_ints(dtype=np.int32)` or add `_kw_ints_property(toks)` returning
  int32. Both acceptable; prefer parametrisation to keep the dispatch
  table small.

## Detailed specs

### Property keyword whitelist

Recognized cell-property keywords (1-D float, length `NX·NY·NZ`):

```python
_PROP_KEYWORDS_FLOAT = frozenset({
    "PORO", "PERMX", "PERMY", "PERMZ", "NTG",
    "MULTX", "MULTY", "MULTZ",
    "PRESSURE", "SWAT", "SGAS", "SOIL",
})

_PROP_KEYWORDS_INT = frozenset({
    "SATNUM", "PVTNUM", "EQLNUM", "FIPNUM",
})
```

Anything outside both sets keeps PR-3's unknown-keyword behaviour.

### Tokenizer / parser changes (sketch)

- Wrap the top-level loop in `_parse_stream(path, stack)` so it can
  recurse on `INCLUDE`.
- `INCLUDE` keyword handler:
  ```python
  elif kw == "INCLUDE":
      block = _read_block(tokens)                # one filename token + `/`
      if not block:
          raise GrdeclSyntaxError("INCLUDE empty")
      raw = block[0].strip()
      if raw[:1] in {"'", '"'} and raw[-1:] == raw[:1]:
          raw = raw[1:-1]
      if any(ch.isspace() for ch in raw):
          raise GrdeclSyntaxError(f"INCLUDE filename with whitespace: {raw!r}")
      child = (path.parent / raw).resolve()
      if child in stack:
          raise GrdeclIncludeError(f"INCLUDE cycle: {child}")
      _parse_stream(child, stack | {child}, state=state)
  ```
- `state` is a small mutable struct (or a closure-captured set of
  locals) that accumulates `coord/zcorn/actnum/props/unrecognized` as
  the recursion descends.
- `read_grdecl(path)` becomes a thin wrapper: `state = State();
  _parse_stream(Path(path).resolve(), {Path(path).resolve()}, state);
  return _finalize(state)`.

### `_finalize` validation

After all INCLUDEs resolve:

- Every property array length must equal `NX·NY·NZ` → otherwise
  `GrdeclConsistencyError` with the offending keyword name.
- A property keyword that appears twice (e.g. PORO in two INCLUDE files)
  is **last-write-wins** but logs a warning via the standard library
  `warnings.warn(GrdeclDuplicatePropertyWarning, …)`. Add the warning
  class to `_errors.py`.

### `tests/test_grdecl_reader.py` (additions)

```python
def test_include_resolves_relative_to_includer(tmp_path):
    g = read_grdecl(FIXTURES / "with_include" / "main.grdecl")
    assert g.cart_dims == (2, 2, 2)
    assert g.coord.shape == (9, 6)
    assert "PORO" in g.props
    assert g.props["PORO"].shape == (8,)
    assert g.props["PORO"].dtype == np.float64
    assert "PERMX" in g.props
    assert "PORO" not in g.unrecognized


def test_include_cycle_raises():
    from cpg2unstructured._errors import GrdeclIncludeError
    with pytest.raises(GrdeclIncludeError):
        read_grdecl(FIXTURES / "with_include_cycle" / "a.grdecl")


def test_property_array_wrong_length_raises(tmp_path):
    # NOTE: do NOT prepend an extra SPECGRID — splitting after "SPECGRID"
    # already returns the keyword's payload (`" 2 2 2 1 F /\n…"`). A naive
    # prepend produces two SPECGRID payloads in one file and the parser
    # raises GrdeclSyntaxError on the orphaned `2` before reaching PORO.
    base = (FIXTURES / "simple_2x2x2.grdecl").read_text()
    # Take from SPECGRID (inclusive of the keyword) up to (but not
    # including) any pre-existing ACTNUM, then append our own ACTNUM
    # and the malformed PORO.
    head = "SPECGRID" + base.split("SPECGRID", 1)[1].split("ACTNUM", 1)[0]
    p = tmp_path / "bad.grdecl"
    p.write_text(
        head
        + "ACTNUM\n 8*1 /\n"
        + "PORO\n 0.1 0.2 0.3 /\n"   # 3 values, need 8
    )
    with pytest.raises(GrdeclConsistencyError):
        read_grdecl(p)


def test_duplicate_property_warns(tmp_path):
    from cpg2unstructured._errors import GrdeclDuplicatePropertyWarning
    p = tmp_path / "dup.grdecl"
    base = (FIXTURES / "simple_2x2x2.grdecl").read_text()
    p.write_text(base + "\nPORO\n 8*0.10 /\nPORO\n 8*0.20 /\n")
    with pytest.warns(GrdeclDuplicatePropertyWarning):
        g = read_grdecl(p)
    np.testing.assert_allclose(g.props["PORO"], 0.20)


def test_include_quotes_stripped(tmp_path):
    p = tmp_path / "q.grdecl"
    sub = tmp_path / "sub.inc"
    sub.write_text((FIXTURES / "simple_2x2x2.grdecl").read_text())
    p.write_text(f"INCLUDE\n \"{sub.name}\" /\n")
    g = read_grdecl(p)
    assert g.cart_dims == (2, 2, 2)
```

## Acceptance criteria

1. `pytest` exits 0 (all PR-1..PR-4 tests).
2. `read_grdecl(with_include/main.grdecl).props["PORO"].shape == (8,)`.
3. INCLUDE paths resolve relative to the file containing the directive.
4. Re-include of the same canonical path → `GrdeclIncludeError`.
5. Property arrays of wrong length raise `GrdeclConsistencyError`
   citing the keyword name.
6. Duplicate property keyword → warning, last-write-wins.
7. Single-file decks parsed in PR-3 still parse identically (no regressions).
8. PORO / PERMX absent in `unrecognized` after parse.
9. No new top-level deps (still numpy + stdlib).

## Out of scope

- INCLUDE inside another INCLUDE's keyword block (Eclipse doesn't allow it; we don't either).
- Macros, conditional `IFDEF`, `MESSAGES` etc.
- Unit conversion for properties.
- Active-cell filtering of properties (PR-12).
- Validating PORO ∈ [0, 1] etc. — domain checks belong elsewhere.

## Implementation notes for the human

- Driver: `/mnt/c/projects/pr4-build.sh` (verbose, `--commit`, branch `pr-04-grdecl-include-props`).
- The recursion stack is `set[Path]` of `Path.resolve()`-canonicalised
  paths — symlinks compare equal to their target.
- Make sure the warnings filter in `pytest.ini` (or `pyproject.toml
  [tool.pytest.ini_options]`) doesn't suppress `GrdeclDuplicatePropertyWarning`.
- **Refactor shape (per D-PR4-3 + the +60-net budget):** PR-3's
  `read_grdecl` is a single ~75-line function with state interleaved
  in locals (`coord`, `zcorn`, `actnum`, `unrecognized`,
  `cart_dims_dimens`, `cart_dims_specgrid`). PR-4 needs to extract
  the keyword-dispatch loop into `_parse_stream(path, stack, state)`
  so it can recurse on INCLUDE. The post-loop validation block
  (size checks, ACTNUM default, reshape COORD) becomes `_finalize(state)`
  and runs ONCE after the top-level stream returns. Net LOC stays
  around +60, but expect ~80 LOC moved/reshaped on top of that —
  budget review time accordingly.
- **Int property dtype:** parametrise `_kw_ints(toks, dtype=np.int8)` so
  ACTNUM keeps int8 (PR-3 callers unchanged) and property dispatch can
  call `_kw_ints(block, dtype=np.int32)` for SATNUM et al.
