## One-liner
Wrap PR-7's `original_ijk` in a tiny `IndexMap` class that goes both
ways between active cell id and Eclipse `(i, j, k)`, with constant-time
lookup once the reverse map is built.

## Why this PR
Many user-facing operations need the `(i, j, k)` of an active cell or
the active id of a known `(i, j, k)`:

- "What's the porosity at column 5, row 12, layer 3?"
- "Plot a 2D slice at k=0."
- "Print neighbours of cell (i=4, j=4, k=2)."

PR-7 already stores `original_ijk` per active cell. This PR formalises
the reverse direction and gives both a clean API.

## Decisions baked in

- **D-02 resolved → lazy reverse map.** Forward (`active_id → (i,j,k)`)
  is a simple `np.ndarray` lookup. Reverse (`(i,j,k) → active_id`) is
  built on first access, cached on the instance. Storage choice: a
  3-D `np.int64` array of shape `cart_dims` filled with `-1` for
  inactive — fast, but uses `~8·NX·NY·NZ` bytes. For typical reservoir
  sizes (≤ 10⁷ cells → 80 MB) this is fine; out-of-scope: spilling to a
  sparse representation for billion-cell grids.
- **`linear_to_cell: np.ndarray (n_active,)`** stores `i + j·NX + k·NX·NY`
  (Eclipse-natural). Mirrors MRST's `G.cells.indexMap`.
- **Inactive cells return `None`**, never raise. Symmetric with
  Python-style `dict.get`.
- **`IndexMap` is independent of `_TopoMesh`.** Constructed from
  `(cart_dims, original_ijk)` only — no reference cycle.
- **D-PR9-1 — Lazy reverse stays lazy.** No eager-build-over-threshold
  shortcut. The reverse-map cost is borne ONLY by callers that
  actually need it; predictability matters more than amortising the
  build across some-but-not-all users. (Resolves Q-PR9-1.)
- **D-PR9-2 — `cells_in_layer(k)` is the only batch helper for PR-9.**
  `cells_in_column(i, j)` and `cells_in_pillar(i, j)` are deferred —
  PR-13's example will tell us whether they're worth shipping or just
  YAGNI. (Resolves Q-PR9-2.)

## Detailed specs

### `cpg2unstructured/index_map.py`

```python
"""Active-id ↔ (i, j, k) translation, lazy reverse map."""
from __future__ import annotations

import numpy as np

__all__ = ["IndexMap"]


class IndexMap:
    """Bi-directional active-cell id ↔ (i, j, k) map.

    Parameters
    ----------
    cart_dims
        (NX, NY, NZ).
    original_ijk
        (Ncells_active, 3) int64. Eclipse-natural ordering — produced by
        ``build_topology`` (PR-7).

    Notes
    -----
    Forward lookup is array-indexing; reverse lookup builds a dense
    ``(NX, NY, NZ)`` int64 array on first call and caches it.
    """

    __slots__ = ("_cart_dims", "_original_ijk", "_linear_to_cell", "_reverse")

    def __init__(self, cart_dims: tuple[int, int, int],
                 original_ijk: np.ndarray) -> None:
        nx, ny, nz = cart_dims
        if original_ijk.ndim != 2 or original_ijk.shape[1] != 3:
            raise ValueError(
                f"original_ijk must be (n_active, 3); got {original_ijk.shape}"
            )
        self._cart_dims = (int(nx), int(ny), int(nz))
        self._original_ijk = original_ijk.astype(np.int64, copy=False)
        i, j, k = original_ijk.T
        self._linear_to_cell = (i + j * nx + k * nx * ny).astype(np.int64)
        self._reverse: np.ndarray | None = None

    # ── forward ──────────────────────────────────────────────────────

    def cell_ijk(self, active_id: int) -> tuple[int, int, int]:
        i, j, k = self._original_ijk[active_id]
        return int(i), int(j), int(k)

    @property
    def linear_to_cell(self) -> np.ndarray:
        return self._linear_to_cell

    @property
    def n_active(self) -> int:
        return int(self._original_ijk.shape[0])

    # ── reverse (lazy) ───────────────────────────────────────────────

    def active_id(self, i: int, j: int, k: int) -> int | None:
        nx, ny, nz = self._cart_dims
        if not (0 <= i < nx and 0 <= j < ny and 0 <= k < nz):
            raise IndexError(f"({i},{j},{k}) outside cart_dims {self._cart_dims}")
        rev = self._build_reverse()
        a = int(rev[i, j, k])
        return a if a != -1 else None

    def _build_reverse(self) -> np.ndarray:
        if self._reverse is None:
            nx, ny, nz = self._cart_dims
            rev = np.full((nx, ny, nz), -1, dtype=np.int64)
            i, j, k = self._original_ijk.T
            rev[i, j, k] = np.arange(self._original_ijk.shape[0])
            self._reverse = rev
        return self._reverse

    # ── batch helpers ────────────────────────────────────────────────

    def cells_in_layer(self, k: int) -> np.ndarray:
        """All active ids whose Eclipse k = ``k``."""
        mask = self._original_ijk[:, 2] == k
        return np.where(mask)[0]

    def __repr__(self) -> str:
        return (f"IndexMap(cart_dims={self._cart_dims}, "
                f"n_active={self.n_active})")
```

### Tests

```python
import numpy as np
import pytest
from cpg2unstructured import IndexMap


def test_round_trip_dense():
    nx, ny, nz = 4, 5, 3
    actnum = np.ones((nx, ny, nz), dtype=bool)
    ijk = np.stack(np.where(actnum.transpose(2, 1, 0)), axis=1)[:, ::-1]
    # ↑ Eclipse-natural i fastest
    im = IndexMap((nx, ny, nz), ijk)
    assert im.n_active == nx * ny * nz
    for a in range(im.n_active):
        i, j, k = im.cell_ijk(a)
        assert im.active_id(i, j, k) == a


def test_round_trip_sparse_actnum():
    rng = np.random.default_rng(42)
    nx, ny, nz = 6, 6, 4
    mask = rng.random((nx, ny, nz)) > 0.3
    ijk = np.argwhere(mask)
    im = IndexMap((nx, ny, nz), ijk)
    assert im.n_active == int(mask.sum())
    for a in range(im.n_active):
        i, j, k = im.cell_ijk(a)
        assert im.active_id(i, j, k) == a
    # Inactive cell returns None
    inactive = np.argwhere(~mask)[0]
    assert im.active_id(*inactive) is None


def test_out_of_range_raises():
    im = IndexMap((2, 2, 2), np.array([[0, 0, 0]]))
    with pytest.raises(IndexError):
        im.active_id(5, 0, 0)


def test_cells_in_layer():
    nx, ny, nz = 3, 3, 3
    ijk = np.argwhere(np.ones((nx, ny, nz), dtype=bool))
    im = IndexMap((nx, ny, nz), ijk)
    layer0 = im.cells_in_layer(0)
    assert len(layer0) == nx * ny


def test_linear_to_cell_eclipse_natural():
    nx, ny, nz = 3, 3, 3
    ijk = np.argwhere(np.ones((nx, ny, nz), dtype=bool))
    im = IndexMap((nx, ny, nz), ijk)
    # Cell at (1, 0, 0) → linear = 1 + 0*nx + 0*nx*ny = 1
    a = im.active_id(1, 0, 0)
    assert im.linear_to_cell[a] == 1
    # Cell at (0, 1, 0) → linear = nx
    a2 = im.active_id(0, 1, 0)
    assert im.linear_to_cell[a2] == nx
```

## Acceptance criteria

1. Round-trip on a fully-active grid: `active_id(*cell_ijk(a)) == a` for all `a`.
2. Random sparse ACTNUM round-trip — same.
3. `active_id` on an inactive cell returns `None`.
4. `active_id` outside `cart_dims` raises `IndexError`.
5. `linear_to_cell` matches Eclipse natural order (`i + j·NX + k·NX·NY`).
6. Reverse map built lazily — first `active_id` call sets `_reverse`,
   subsequent calls reuse it.
7. `cells_in_layer(k)` returns all active ids in layer `k`.
8. `__repr__` shows `cart_dims` and `n_active`.

## Out of scope

- Sparse storage for billion-cell grids.
- Coordinate transforms (utm, projection).
- (i, j, k) layouts other than Eclipse natural — defer to v2.
- Caching reverse map on disk.

## Implementation notes for the human

- Driver: `/mnt/c/projects/pr9-build.sh`.
- `__slots__` is intentional — IndexMap is created once per mesh, kept
  for its lifetime; preventing per-instance dict bloat matters at
  scale.
- The reverse map allocates `(NX·NY·NZ)·8` bytes — 80 MB for a 10⁷ grid.
  Acceptable for v0.1.0; revisit at v0.2.
