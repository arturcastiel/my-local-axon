> Wave 4 · Theme: geometry + index map

> Dependencies: PR-7 (merged · 2d7bb3d · GH #6)

## One-liner
Compute face centroids + areas and cell centroids + volumes for the
topology mesh produced by PR-7. Pure numpy, no scipy. Validates against
unit-cube ground truth at 1e-12 precision.

## Why this PR
Topology alone is not enough for the graph layer (PR-10 needs distances)
or the v0.1.0 demo (PR-13 example). Volumes matter for scientific use
(porosity-volume products, mass balances).

The math is small but easy to get wrong. This PR pins the formulas
behind a tested API once and for all.

## Decisions baked in

- **Face centroid = arithmetic mean of face nodes.** For the matching-Z
  + 4-node faces produced by PR-7 this is the correct centroid; for
  3-node faces (degenerate after a corner collapse) the mean is also
  the correct centroid. Documented limitation: for >4-node faces (not
  produced by v0.1.0) this would be wrong; out-of-scope.
- **Face area = sum of triangle areas via cross-product.** Triangulate
  the face by fan from node 0 (or, for 4-node faces, two triangles:
  `(0,1,2)` + `(0,2,3)`). Numerically stable enough for typical
  reservoir scales.
- **Face normal direction = consistent with `face_cells[:, 0] → face_cells[:, 1]`.**
  Determined by the right-hand rule on the face's first triangle and
  flipped if it disagrees with the inward direction from cell 0 toward
  cell 1's centroid. For boundary faces (col 1 = -1), normal points
  outward from the active cell.
- **Cell volume = divergence theorem: `(1/6) Σ_faces (centroid · normal) · area`.**
  Equivalent to `Σ Σ x_i · (cross product piece)`. Robust for convex
  hexahedra; unit-cube → 1.0 exactly to 1e-12.
- **Cell centroid = `(1/V) Σ_faces face_centroid · (centroid · normal · area / 3)`.**
  Mass-property weighted. Documented as exact for convex polyhedra.
  For non-convex degenerate hexes it is approximate but always lies
  inside the convex hull.
- **All outputs `float64`.** No mixed precision.
- **D-PR8-1 — `_TopoMesh → UnstructuredGrid` graduation deferred again.**
  Per Q-PR8-1's recommendation (FaceGeometry + CellGeometry as
  SEPARATE dataclasses — lazy-compute, swappable), PR-8 ships them as
  standalone. The rename happens at PR-13 (the version-0.1.0 polish PR)
  or sooner if downstream PRs need the attached form. _TopoMesh stays
  internal in `unstructured.py`.
- **D-PR8-2 — Per-face cell ids come from `faces.neighbours`, not
  `face_cells`.** Spec text used `face_cells[f, 0]` etc., but merged
  PR-7's Faces dataclass calls this field `neighbours`. The name
  `cell_faces` (per-cell → face ids) is a separate thing on _TopoMesh.
  Implementation reads `faces.neighbours[f, 0]` for "low" cell and
  `faces.neighbours[f, 1]` for "high" cell (-1 = boundary).
- **D-PR8-3 — `face_geometry(nodes, faces, cell_corners)` — three
  required args.** The spec's optional `cell_centroids_estimate=None`
  was contradicted by the implementation note. Cleanest: pass
  `cell_corners` (mesh.cell_corners from _TopoMesh) and compute
  per-cell centroid estimates internally for normal orientation:
    cell_centroid_estimate = nodes[cell_corners].mean(axis=1)
  Normals are flipped per face so that for each face f with
  faces.neighbours[f, 0] ≡ cellA, normal points AWAY from cellA's
  centroid estimate (= "from cellA toward cellB" for inner faces;
  outward for boundary faces).
- **D-PR8-4 — Degenerate-face normal guard.** Wedge cells (kept by
  D-PR7-8's per-face uniqueness rule) can have faces with 3 unique
  nodes whose triangulated cross-product is small. Implementation
  guards `normal / np.linalg.norm(normal_raw)` via
  `np.where(norm_len < 1e-15, 1.0, norm_len)` to avoid NaN; affected
  faces get a zero-magnitude `normal` and zero `area` — which is the
  geometrically correct answer for degenerate triangles.
- **D-PR8-5 — Area + normal from the two DIAGONALS, not fan-triangle
  cross-products.** The spec's formula
  `area = (1/2)(|cross(e01,e02)| + |cross(e02,e03)|)`
  `normal = (cross(e01,e02) + cross(e02,e03)) / |...|`
  assumes vertices are in cyclic order around the quad. But PR-6's
  `Faces.nodes` is **sorted by node ID** (for the np.unique-based
  dedup, which needs a canonical vertex set). After sorting, the
  triangle-fan from vertex 0 mixes diagonals with edges — the two
  triangle normals point in opposite directions and CANCEL when summed,
  producing a zero-vector normal and zero volume.
  Order-independent fix: on a planar convex quad the two diagonals are
  the 2 longest of the 6 pairwise inter-vertex distances. Then
  `area = (1/2) |d1 × d2|`, `normal = (d1 × d2) / |...|`. Works for
  any vertex order; areas are unchanged (still magnitudes); normals
  now correct → volumes correct to 1e-12 on unit cubes.

### Math

**Face centroid + area (4-node face):**
```python
v = nodes[face_nodes]                # (4, 3)
c = v.mean(axis=0)
e1, e2 = v[1] - v[0], v[2] - v[0]
e3, e4 = v[2] - v[0], v[3] - v[0]
n1 = np.cross(e1, e2);   a1 = 0.5 * np.linalg.norm(n1)
n2 = np.cross(e3, e4);   a2 = 0.5 * np.linalg.norm(n2)
area = a1 + a2
normal_raw = (n1 + n2)
normal = normal_raw / np.linalg.norm(normal_raw)
```

**Cell volume (divergence theorem):**
```python
V = 0.0
for f in cell_faces[c]:
    sign = +1.0 if face_cells[f, 0] == c else -1.0    # face normal points 0→1
    V += sign * np.dot(face_centroids[f], face_normals[f]) * face_areas[f]
V /= 3.0   # divergence theorem
```

**Cell centroid:**
```python
xc = np.zeros(3)
for f in cell_faces[c]:
    sign = +1.0 if face_cells[f, 0] == c else -1.0
    xc += sign * face_centroids[f] * np.dot(face_centroids[f], face_normals[f]) * face_areas[f]
xc /= (4.0 * V)
```

(Standard divergence-theorem moment formulation; matches the reference
in MRST's `computeGeometry`.)

## Acceptance criteria

1. Unit-cube cell volume = 1.0 ± 1e-12.
2. Unit-cube face area = 1.0 ± 1e-12.
3. Unit-cube cell centroid = (0.5, 0.5, 0.5) ± 1e-12.
4. 2×2×2 unit-cell deck total volume = 8.0 ± 1e-12.
5. All face normals are unit length.
6. Inner-face normal direction agrees with `face_cells[:, 0] → face_cells[:, 1]`.
7. Boundary-face normals point outward (away from cell 0).
8. No scipy import.
9. `face_geometry + cell_geometry` runs on 100k cells in < 5 s.

## Out of scope

- Non-planar quadrilaterals (deferred — matching-Z gives planar by construction).
- 3-node face support beyond what falls out naturally from the formulas
  (PR-7 doesn't produce them either, but the formulas degrade gracefully).
- Anisotropic permeability scaling, transmissibility — out of scope for v0.1.0.
