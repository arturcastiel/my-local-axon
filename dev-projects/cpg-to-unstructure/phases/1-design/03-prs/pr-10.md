# PR-10 — `graph.py` adjacency (numpy-only default)

> Wave 5 · Theme: graph layer
> Branch: `pr-10-graph-numpy`
> Status: finalized (drift-fixed 2026-05-21; open Qs resolved; ready for implementation)
> Dependencies: PR-7 (merged · 2d7bb3d · GH #6), PR-8 (merged · 281a50e · GH #7) for `distance` weights

## One-liner
Convert the topology mesh into a cell-cell adjacency graph (list-of-list,
indexed by active cell id) with three weighting modes: `unit`,
`distance` (Euclidean centroid-to-centroid), and `None` (raw neighbour
ids only). Pure numpy — scipy comes in PR-11.

## Why this PR
Shortest-path between two cells is the headline v0.1.0 use case. PR-10
gives the lightweight default that always works (no scipy dep), and
PR-11 swaps in scipy CSR + Dijkstra for speed on bigger grids.

A clean numpy-only implementation also serves as the **reference** that
PR-11's scipy path is compared against in tests.

## Decisions baked in

- **D-03 resolved → list-of-list, indexed by active cell id.** Cell ids
  are contiguous `[0, n_active)` after PR-7's rebuild. `list[list[tuple[int, float]]]`
  is the right Python-side representation: O(1) neighbour lookup,
  numpy-friendly to flatten, and matches the pattern of `nx.Graph`'s
  internal adjacency.
- **Weights live on edges, not nodes.** Every neighbour entry is
  `(neighbour_id, weight: float)`. `weights="none"` returns
  `(neighbour_id, 1.0)` so downstream code never has to special-case.
  (Lowercase — matches the `WeightMode` Literal.)
- **No node weights, no edge metadata beyond a single float.** Keeps
  the API minimal; richer attributes can be carried via parallel
  arrays in user code or via a future `graph_with_attrs` helper.
- **Distance weight uses cell-centroid Euclidean L2.** Computed lazily
  via PR-8's `cell_geometry`; cached on the grid instance once
  computed.
- **Adjacency is symmetric.** `(a, b)` and `(b, a)` both appear; total
  edge count = `2 · n_inner_faces`. PR-11's CSR is built from the same
  symmetric source.
- **Inner faces only.** Boundary faces (`faces.neighbours[:, 1] == -1`)
  are skipped — they don't define a cell-cell edge.
- **D-PR10-1 — No `to_dict()` adapter.** `enumerate(adj)` or
  `dict(enumerate(adj))` is one line for users that want a dict view.
  (Resolves Q-PR10-1.)
- **D-PR10-2 — Single-destination Dijkstra in PR-10.** Multi-destination
  deferred to PR-11's scipy path (`csgraph.dijkstra` returns the full
  distance vector natively). (Resolves Q-PR10-2.)
- **D-PR10-3 — `weights="none"` is lowercase.** Matches the
  `WeightMode = Literal["unit", "distance", "none"]` type. Spec text
  said `"None"` once (capital N) — that's a typo, not a third mode.
- **D-PR10-4 — Validate `weights` mode at runtime.** Unknown values
  raise `ValueError`; otherwise a typo like `"Unit"` would silently
  fall through to weight=1.0 and look like a working but incorrect
  run.

## Detailed specs

### `cpg2unstructured/graph.py`

```python
"""Cell-cell adjacency + shortest-path. Pure numpy default.

PR-11 adds an optional scipy CSR + dijkstra path under the [fast] extra.
"""
from __future__ import annotations

import heapq
from typing import Literal
import numpy as np

from cpg2unstructured.unstructured import _TopoMesh

__all__ = ["cell_adjacency", "shortest_path"]

WeightMode = Literal["unit", "distance", "none"]


def cell_adjacency(
    mesh: _TopoMesh,
    weights: WeightMode = "unit",
    cell_centroids: np.ndarray | None = None,
) -> list[list[tuple[int, float]]]:
    """Build the cell-cell adjacency list.

    Parameters
    ----------
    mesh
        ``_TopoMesh`` from ``build_topology``.
    weights
        - ``"unit"``     : weight = 1.0 on every edge.
        - ``"distance"`` : weight = ‖centroid_i − centroid_j‖₂.
                           Requires ``cell_centroids`` (PR-8 output).
        - ``"none"``     : same as ``"unit"`` but documented as
                           "no weighting" (Dijkstra still works).
    cell_centroids
        Required when ``weights="distance"``. Shape ``(n_active, 3)``.
    """
    if weights == "distance" and cell_centroids is None:
        raise ValueError("weights='distance' requires cell_centroids")

    nb = mesh.faces.neighbours
    inner = nb[(nb[:, 0] != -1) & (nb[:, 1] != -1)]
    n = mesh.cell_faces.shape[0]
    adj: list[list[tuple[int, float]]] = [[] for _ in range(n)]

    for a, b in inner:
        if weights == "distance":
            w = float(np.linalg.norm(cell_centroids[a] - cell_centroids[b]))
        else:
            w = 1.0
        adj[a].append((int(b), w))
        adj[b].append((int(a), w))

    return adj


def shortest_path(
    adj: list[list[tuple[int, float]]],
    src: int,
    dst: int,
) -> tuple[float, list[int]]:
    """Dijkstra's algorithm (heapq + dict). Slow but always works.

    Returns ``(total_weight, path_as_list_of_cell_ids)``.
    Raises ``ValueError`` if dst is unreachable.
    """
    n = len(adj)
    dist = np.full(n, np.inf, dtype=np.float64)
    prev = np.full(n, -1, dtype=np.int64)
    dist[src] = 0.0
    heap: list[tuple[float, int]] = [(0.0, src)]

    while heap:
        d, u = heapq.heappop(heap)
        if u == dst:
            break
        if d > dist[u]:
            continue
        for v, w in adj[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                prev[v] = u
                heapq.heappush(heap, (nd, v))

    if not np.isfinite(dist[dst]):
        raise ValueError(f"no path from {src} to {dst}")

    path = [dst]
    while path[-1] != src:
        path.append(int(prev[path[-1]]))
    path.reverse()
    return float(dist[dst]), path
```

### Tests

```python
import numpy as np
import pytest

from cpg2unstructured import (
    read_grdecl, from_grdecl, build_topology,
    cell_adjacency, shortest_path,
)


@pytest.fixture
def mesh_2x2x2(fixtures):
    cube = from_grdecl(read_grdecl(fixtures / "simple_2x2x2.grdecl"))
    return build_topology(cube)


def test_edge_count_2x2x2(mesh_2x2x2):
    adj = cell_adjacency(mesh_2x2x2, weights="unit")
    n_edges_directed = sum(len(a) for a in adj)
    # 12 inner faces × 2 directions = 24
    assert n_edges_directed == 24


def test_corner_to_corner_path_2x2x2(mesh_2x2x2):
    adj = cell_adjacency(mesh_2x2x2, weights="unit")
    # Cell ids: with active mask all true and Eclipse-natural i fastest,
    # corner (0,0,0) = id 0, corner (1,1,1) = id 7.
    dist, path = shortest_path(adj, 0, 7)
    assert dist == 3.0
    assert len(path) == 4   # 3 hops → 4 cells visited


def test_distance_weight_unit_cube(mesh_2x2x2):
    # Centroids at (0.5, 0.5, 0.5), (1.5, 0.5, 0.5), ... → unit-spaced.
    centroids = np.array([
        [0.5, 0.5, 0.5], [1.5, 0.5, 0.5], [0.5, 1.5, 0.5], [1.5, 1.5, 0.5],
        [0.5, 0.5, 1.5], [1.5, 0.5, 1.5], [0.5, 1.5, 1.5], [1.5, 1.5, 1.5],
    ])
    adj = cell_adjacency(mesh_2x2x2, weights="distance",
                         cell_centroids=centroids)
    dist, _ = shortest_path(adj, 0, 7)
    np.testing.assert_allclose(dist, 3.0, atol=1e-12)


def test_unreachable_raises(mesh_2x2x2):
    adj = cell_adjacency(mesh_2x2x2)
    # Force an artificial disconnect by emptying cell 7's neighbours
    adj[7] = []
    for a in range(7):
        adj[a] = [(b, w) for b, w in adj[a] if b != 7]
    with pytest.raises(ValueError, match="no path"):
        shortest_path(adj, 0, 7)


def test_n_inner_faces_match_edges(mesh_2x2x2):
    adj = cell_adjacency(mesh_2x2x2)
    nb = mesh_2x2x2.faces.neighbours
    n_inner = ((nb[:, 0] != -1) & (nb[:, 1] != -1)).sum()
    n_edges_undirected = sum(len(a) for a in adj) // 2
    assert n_edges_undirected == n_inner
```

## Acceptance criteria

1. 2×2×2 graph: 8 nodes, 12 undirected edges (24 directed entries).
2. Dijkstra `(0 → 7)` on unit weights = 3 hops, distance 3.0.
3. Dijkstra `(0 → 7)` on distance weights (unit-cell centroids) = 3.0.
4. Unreachable destination raises `ValueError` with `"no path"` in message.
5. `weights="distance"` without `cell_centroids` raises `ValueError`.
6. Edge count check: `len(unique edges) == n_inner_faces`.
7. No scipy import.
8. Adjacency build on 100k cells in < 2 s.

## Out of scope

- All-pairs shortest paths (Floyd-Warshall).
- Multi-source Dijkstra / heatmap.
- Edge attributes beyond a single float (e.g. transmissibility).
- Visualisation.
- scipy code path (PR-11).
