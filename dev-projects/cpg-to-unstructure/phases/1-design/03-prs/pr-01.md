## One-liner
Make `cpg2unstructure` a `pip install`-able Python package with numpy
as the only hard dep, scipy as `[fast]` extra, and pytest configured.

## Why this PR
We need a stable platform every later PR sits on:
- import path that works from anywhere on the developer's machine
- a `pyproject.toml` enforcing the dependency story decided in study
- pytest discoverable so PR-2 can land its first real test
- license + README stub so the repo isn't a black box from day one

No business logic in this PR — that's PR-2's job.

```toml
requires = ["setuptools>=68", "wheel"]

[project]
name = "cpg2unstructured"
version = "0.1.0.dev0"
description = "Lightweight CPG (Eclipse corner-point) to unstructured-mesh + graph"
readme = "README.md"
requires-python = ">=3.10"
license = {text = "MIT"}
authors = [{name = "Dr. Artur Castiel Reis de Souza"}]
classifiers = [
    "Development Status :: 3 - Alpha",
    "License :: OSI Approved :: MIT License",
    "Programming Language :: Python :: 3 :: Only",
    "Programming Language :: Python :: 3.10",
    "Programming Language :: Python :: 3.11",
    "Programming Language :: Python :: 3.12",
    "Topic :: Scientific/Engineering",
]
dependencies = [
    "numpy>=1.24",
]

[project.optional-dependencies]
fast = ["scipy>=1.10"]
dev  = ["pytest>=7", "pytest-cov>=4"]

[tool.setuptools.packages.find]
where = ["."]
include = ["cpg2unstructured*"]

[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-ra"
```

**Decisions baked in:**
- Python 3.10+ floor (PEP 604 `X | Y`, structural pattern matching, no 3.9 noise).
- numpy 1.24 floor (stable enough; `np.unique(..., axis=0, return_inverse=True)` reliable).
- No mypy/ruff config in this PR — add later if needed (tooling drift is its own PR).
- setuptools build backend (not poetry/hatch) — least excess machinery.

## Acceptance criteria

1. `pip install -e .[dev]` succeeds in a clean venv.
2. `python -c "import cpg2unstructured; print(cpg2unstructured.__version__)"` prints `0.1.0.dev0`.
3. `pytest` exits 0 with the smoke test passing.
4. `pip install -e .[fast]` succeeds (pulls scipy).
5. No code in `cpg2unstructured/` other than `__init__.py` and `py.typed`.
6. `python -m build` produces a clean sdist + wheel.

## Test plan

Single test `tests/test_smoke.py::test_import`:
- imports the package
- asserts `__version__` is non-empty

This test must continue to pass through every later PR.
