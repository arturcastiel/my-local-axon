# PR-12 — Property pass-through hook

> Wave 6 · Theme: polish
> Branch: `pr-12-props`
> Status: draft (not yet implemented)
> Dependencies: PR-7 (merged), PR-9 (merged), PR-4 (merged for parsing)

## One-liner
Expose Eclipse cell-properties (PORO, PERMX, …) on the unstructured
mesh, filtered to active cells in the correct order via `IndexMap`.
Read-only `mesh["PORO"]` accessor returns a 1-D `np.ndarray` of length
`n_active`.

## Why this PR
PR-4 already pulls properties into `Grdecl.props: dict[str, np.ndarray]`
of length `NX·NY·NZ` (Eclipse-natural, including inactive cells). The
unstructured-mesh side has dropped + reordered cells via PR-7 and PR-9.
PR-12 is the bridge: re-index the property arrays so user code talks
in active-cell ids without having to track the Eclipse layout.

## Decisions baked in

- **Read-only.** `mesh["PORO"]` returns a view (or a copy) but no
  setter is exposed. Mutation happens upstream (via re-running the
  parser) — keeps the mesh side immutable.
- **Storage backed by `dict[str, np.ndarray]`** on the public mesh
  dataclass — mirrors `Grdecl.props` but keys are uppercase Eclipse
  names and values are length-`n_active`.
- **Lazy filtering**: `mesh.props` may be empty at construction time;
  `attach_props(mesh, grdecl)` populates it. Cheap to call repeatedly.
- **`__getitem__` is the user-facing accessor** (`mesh["PORO"]`),
  consistent with pandas / xarray ergonomics.
- **Missing keyword** → `KeyError` with a list of available keys.
- **Property arrays are NOT renamed.** Keep Eclipse keyword names
  verbatim (uppercase). User code can alias if they want lowercase.
- **Active-cell filtering uses `IndexMap.linear_to_cell`** (PR-9):
  `mesh.props["PORO"] = grdecl.props["PORO"][index_map.linear_to_cell]`.
- **Float and int property dtypes preserved** from PR-4's whitelist
  classification.

## Detailed specs

### `cpg2unstructured/unstructured.py` (final dataclass shape)

```python
@dataclass(frozen=True)
class UnstructuredGrid:
    """Public face of the package: nodes + faces + cells + props.

    Constructed by ``build_topology(cube)`` (PR-7) and optionally
    enriched with cell properties via ``attach_props(grid, grdecl)``
    (PR-12).
    """
    nodes: np.ndarray
    faces: Faces
    cell_faces: np.ndarray
    cell_corners: np.ndarray
    original_ijk: np.ndarray
    cart_dims: tuple[int, int, int]
    props: dict[str, np.ndarray] = field(default_factory=dict)
    _index_map: "IndexMap | None" = None

    def __getitem__(self, key: str) -> np.ndarray:
        try:
            return self.props[key]
        except KeyError as exc:
            avail = sorted(self.props)
            raise KeyError(
                f"property {key!r} not attached. Available: {avail}"
            ) from exc

    def __contains__(self, key: str) -> bool:
        return key in self.props

    @property
    def n_cells(self) -> int:
        return self.cell_faces.shape[0]

    @property
    def index_map(self) -> "IndexMap":
        from cpg2unstructured.index_map import IndexMap
        if self._index_map is None:
            object.__setattr__(self, "_index_map",
                               IndexMap(self.cart_dims, self.original_ijk))
        return self._index_map
```

> Implementation note: because the dataclass is `frozen=True`, the
> lazy `_index_map` cache uses `object.__setattr__`. An alternative is
> `frozen=False, eq=False`, which is simpler but loses immutability
> guarantees — PR-12 chooses the `object.__setattr__` route. This is
> the only mutable cache on the class.

### `attach_props`

```python
def attach_props(grid: UnstructuredGrid, grdecl: Grdecl) -> UnstructuredGrid:
    """Filter each property in ``grdecl.props`` to active cells in the
    grid's order. Returns a NEW UnstructuredGrid (frozen) with the
    populated ``props`` dict — the original is not mutated.
    """
    if grid.cart_dims != grdecl.cart_dims:
        raise ValueError(
            f"cart_dims mismatch: grid={grid.cart_dims} vs "
            f"grdecl={grdecl.cart_dims}"
        )
    idx = grid.index_map.linear_to_cell      # (n_active,)
    new_props = {k: v[idx].copy() for k, v in grdecl.props.items()}
    return dataclasses.replace(grid, props=new_props)
```

### Tests

```python
import numpy as np
import pytest

from cpg2unstructured import (
    read_grdecl, from_grdecl, build_topology, attach_props,
)


def test_porotype_round_trip(fixtures):
    g = read_grdecl(fixtures / "with_include" / "main.grdecl")
    cube = from_grdecl(g)
    grid = build_topology(cube)
    grid = attach_props(grid, g)
    assert "PORO" in grid
    assert grid["PORO"].shape == (grid.n_cells,)
    assert grid["PORO"].dtype == np.float64
    np.testing.assert_allclose(grid["PORO"][0], g.props["PORO"][0])


def test_active_cell_filtering(tmp_path, fixtures):
    src = (fixtures / "simple_2x2x2.grdecl").read_text().replace(
        "ACTNUM\n  8*1 /", "ACTNUM\n  1 1 1 1  1 1 1 0 /"
    ) + "\nPORO\n 0.10 0.11 0.12 0.13  0.20 0.21 0.22 0.99 /"
    p = tmp_path / "deck.grdecl"
    p.write_text(src)
    g = read_grdecl(p)
    grid = attach_props(build_topology(from_grdecl(g)), g)
    assert grid["PORO"].shape == (7,)
    # The 0.99 belongs to the deactivated cell (id 7) → must be absent.
    assert 0.99 not in grid["PORO"]


def test_missing_property_raises(fixtures):
    g = read_grdecl(fixtures / "simple_2x2x2.grdecl")
    grid = attach_props(build_topology(from_grdecl(g)), g)
    with pytest.raises(KeyError, match="not attached"):
        _ = grid["PORO"]   # 2x2x2 fixture has no properties


def test_cart_dims_mismatch_raises(fixtures):
    g = read_grdecl(fixtures / "simple_2x2x2.grdecl")
    grid = build_topology(from_grdecl(g))
    # Synthesize a Grdecl with mismatched cart_dims
    from dataclasses import replace
    bad = replace(g, cart_dims=(3, 3, 3))
    with pytest.raises(ValueError, match="cart_dims mismatch"):
        attach_props(grid, bad)


def test_int_property_preserves_dtype(tmp_path, fixtures):
    src = (fixtures / "simple_2x2x2.grdecl").read_text() \
        + "\nSATNUM\n 1 2 1 2  3 1 2 3 /"
    p = tmp_path / "deck.grdecl"
    p.write_text(src)
    g = read_grdecl(p)
    grid = attach_props(build_topology(from_grdecl(g)), g)
    assert grid["SATNUM"].dtype.kind == "i"


def test_attach_is_pure(fixtures):
    """attach_props returns a new grid without mutating the input."""
    g = read_grdecl(fixtures / "with_include" / "main.grdecl")
    grid = build_topology(from_grdecl(g))
    grid2 = attach_props(grid, g)
    assert grid is not grid2
    assert grid.props == {}
    assert "PORO" in grid2.props
```

## Acceptance criteria

1. PORO round-trip: `attach_props(grid, grdecl)["PORO"]` matches
   `grdecl.props["PORO"][index_map.linear_to_cell]`.
2. Active-cell filter: deactivated cells' property values are absent.
3. Missing-key access raises `KeyError` listing available keys.
4. `cart_dims` mismatch raises `ValueError`.
5. `int` properties preserve their `int8`/`int32` dtype.
6. `attach_props` is pure: input grid not mutated.
7. `grid.n_cells == grid["PORO"].shape[0]`.
8. `grid.index_map` returns the same `IndexMap` instance on repeat
   access (lazy cache).
9. `grid in g.unrecognized` is False for whitelisted properties.

## Out of scope

- Computed properties (e.g. `transmissibility = …`).
- Setting properties from user code.
- Vectorised group operations on properties (`grid.props.where(…)`).
- Property unit conversion.
- Cell-attribute serialisation (VTK / HDF5).
