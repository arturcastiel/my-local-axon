# PR-1 — Faulted test fixture + `_faulted_pillar_mask_*` helpers

> Wave 1 · Theme: foundations
> Branch: `pr-fault-01-fixture-mask`
> Status: draft (not yet implemented)
> Dependencies: v0.1.0 main (no prior 2-faults PR)

## One-liner
Add a 2×2×2 faulted GRDECL test fixture and replace the three
`_verify_matching_z_{i,j,k}` raise-on-fault helpers with
`_faulted_pillar_mask_{i,j,k}` predicates that RETURN a per-pillar-pair
bool mask. Behaviour-preserving — the existing `find_*_faces` keep
raising `NonMatchingZError` if the mask is non-empty.

## Why this PR
First step of phase 2-faults. Doesn't change observable behaviour
yet — but lands the two prerequisites every later PR needs:

1. **A real faulted fixture** so PR-2..PR-9 have something concrete
   to test against. Without it, we'd be unit-testing in the abstract.
2. **A mask-returning helper** so PR-6 (the eventual widening of
   `find_i_faces`) has a primitive to branch on. The rename is honest
   about the new behaviour — the old name implied "raise on fault"
   and we want that signal gone before any caller starts relying on
   the new semantics.

Zero behavioural drift: `find_*_faces` get a one-line shim:
```python
faulted = _faulted_pillar_mask_i(P, actnum_3d)
if faulted.any():
    raise NonMatchingZError(...)
```
which is bit-identical to the current `_verify_matching_z_i` raise.

## Decisions baked in

- **Honest naming.** `_verify_matching_z_*` implied "raise on fault".
  After this PR, the helpers RETURN a mask. Renamed to
  `_faulted_pillar_mask_*` to match the new behaviour. (Q-2F-3
  resolved.)
- **Mask shape.** `(NX, NY, NZ)` bool per direction — True means
  "this pillar-pair stack has at least one non-matching i-face on
  the (i, j, k) cell pair" (or analogous for j, k). PR-6+ will use
  `.any()` for the early-out and the index pattern for selecting
  faulted-stack inputs.
- **Fixture geometry.** 2×2×2 all-active cartesian, fault between
  i=0 and i=1: cells in the i=1 column have their entire k-column
  shifted down by 0.5 in Z relative to i=0. Half-cell offset.
  Single fault plane → easy to hand-verify `find_connections`
  output (expected: 3 fault-face polygons per the worked example
  in `01-study.md` §6b).
- **No widening yet.** `find_i_faces`, `find_j_faces`, `find_k_faces`
  keep their current raise-on-fault behaviour by composing the new
  mask helper with an explicit raise. PR-6/PR-7/PR-8 remove those
  raises. This isolation makes PR-1 a pure refactor that can't
  regress any Tier A test.

## Detailed specs

### `tests/fixtures/simple_faulted_2x2x2.grdecl`

A 2×2×2 cartesian deck with the i=1 column of cells shifted down by
0.5 in Z. Vertical pillars (COORD top + bot at z=0, z=3 to leave room
for the offset).

```

-- 2x2x2 cartesian with HALF-CELL Z FAULT between i=0 and i=1.
-- All-active; vertical pillars 0..3 (extra headroom for offset).
-- Cells at i=0 use Z = {0, 1, 2}; cells at i=1 use Z = {0.5, 1.5, 2.5}.

SPECGRID
  2 2 2 1 F /

COORD
  0 0 0  0 0 3
  1 0 0  1 0 3
  2 0 0  2 0 3
  0 1 0  0 1 3
  1 1 0  1 1 3
  2 1 0  2 1 3
  0 2 0  0 2 3
  1 2 0  1 2 3
  2 2 0  2 2 3  /

ZCORN
  -- k=0 cells: top z; for i=0 column z=0, for i=1 column z=0.5
  -- Layout per ZCORN slice = (2*NX, 2*NY) = (4, 4), then top/bot per cell
  -- F-order over (2*i, 2*j) — i fastest. 16 values per top/bot plane.
  -- Top of k=0:  z=0 for i-slots 0,1; z=0.5 for i-slots 2,3.
  0 0 0.5 0.5  0 0 0.5 0.5  0 0 0.5 0.5  0 0 0.5 0.5
  -- Bot of k=0 / Top of k=1:  z=1 for i=0 col; z=1.5 for i=1 col

  1 1 1.5 1.5  1 1 1.5 1.5  1 1 1.5 1.5  1 1 1.5 1.5
  -- Bot of k=1 / Top of k=2:  z=2 for i=0 col; z=2.5 for i=1 col
  2 2 2.5 2.5  2 2 2.5 2.5  2 2 2.5 2.5  2 2 2.5 2.5
  2 2 2.5 2.5  2 2 2.5 2.5  2 2 2.5 2.5  2 2 2.5 2.5
  -- Bot of k=2:  z=3 for i=0 col; z=3.5 for i=1 col... wait that's > pillar bot.
  3 3 3.5 3.5  3 3 3.5 3.5  3 3 3.5 3.5  3 3 3.5 3.5 /

  8*1 /
```

⚠ **Note for PR-1 authoring:** the exact ZCORN block above is a
sketch. The implementer must hand-walk the F-order layout of ZCORN
((2·NX, 2·NY, 2·NZ) with i fastest) to make sure each cell's 8
corners land at the right Z. Sanity check:

```python
g = read_grdecl("tests/fixtures/simple_faulted_2x2x2.grdecl")
assert g.cart_dims == (2, 2, 2)
assert int(g.actnum.sum()) == 8
# i=0 column cells should have z-extent (0,1), (1,2), (2,3)
# i=1 column cells should have z-extent (0.5,1.5), (1.5,2.5), (2.5,3.5)
```

If the deck's pillar bot needs to be deeper to accommodate the
offset, bump COORD pillar bot from 3 to 4. The widening doesn't
matter for the topology.

### `cpg2unstructured/unstructured.py` — rename + behaviour shift

Three helpers transition from raise-on-fault to return-mask. For
each direction:

**Before (Tier A code):**
```python
def _verify_matching_z_i(P: np.ndarray, actnum_3d: np.ndarray) -> None:
    """Raise NonMatchingZError if an active-active i-plane disagrees."""
    nx, ny, nz = actnum_3d.shape
    for k in range(nz):
        for j in range(ny):
            for i in range(nx - 1):
                if not (actnum_3d[i, j, k] and actnum_3d[i+1, j, k]):
                    continue
                # ... extract shared-pillar Z values for cells (i,j,k) and (i+1,j,k)
                if not np.allclose(z_left, z_right):
                    raise NonMatchingZError(...)
```

**After (PR-1 code):**
```python
def _faulted_pillar_mask_i(P: np.ndarray, actnum_3d: np.ndarray) -> np.ndarray:
    """Per-cell-pair faulted mask along the i direction.

    Returns
    -------
    (NX-1, NY, NZ) bool — True at (i,j,k) means the active i-face
    between cells (i,j,k) and (i+1,j,k) is NON-MATCHING (faulted).
    False means matching-Z OR at least one of the two cells is
    inactive (in which case the face isn't an active-active i-face).
    """
    nx, ny, nz = actnum_3d.shape
    mask = np.zeros((nx - 1, ny, nz), dtype=bool)
    for k in range(nz):
        for j in range(ny):
            for i in range(nx - 1):
                if not (actnum_3d[i, j, k] and actnum_3d[i+1, j, k]):
                    continue
                # ... extract z_left, z_right
                if not np.allclose(z_left, z_right):
                    mask[i, j, k] = True
    return mask
```

Then `find_i_faces` keeps its raise-on-fault contract for now:
```python
def find_i_faces(cube: CornerCube) -> Faces:
    ...
    faulted = _faulted_pillar_mask_i(P, actnum_3d)
    if faulted.any():
        # Surface the first offending cell pair like the old check did.
        i, j, k = map(int, np.argwhere(faulted)[0])
        raise NonMatchingZError(
            f"non-matching Z values on i-face between "
            f"cells ({i},{j},{k}) and ({i+1},{j},{k})"
        )
    # ... rest of the function unchanged (matching-Z fast path) ...
```

Same for `_faulted_pillar_mask_j` (shape `(NX, NY-1, NZ)`) and
`_faulted_pillar_mask_k` (shape `(NX, NY, NZ-1)`).

### `tests/test_faulted_mask.py`

```python
import numpy as np
import pytest

from cpg2unstructured import from_grdecl, read_grdecl
from cpg2unstructured._errors import NonMatchingZError
from cpg2unstructured.unstructured import (
    _faulted_pillar_mask_i,
    _faulted_pillar_mask_j,
    _faulted_pillar_mask_k,
    find_i_faces,
)


# ── matching-Z decks: mask should be empty ───────────────────────────

def test_simple_2x2x2_no_fault_i(fixtures):
    g = read_grdecl(fixtures / "simple_2x2x2.grdecl")
    cube = from_grdecl(g)
    m = _faulted_pillar_mask_i(cube.P, cube.actnum_3d)
    assert m.shape == (1, 2, 2)
    assert not m.any()


def test_simple_2x2x2_no_fault_j(fixtures):
    g = read_grdecl(fixtures / "simple_2x2x2.grdecl")
    cube = from_grdecl(g)
    m = _faulted_pillar_mask_j(cube.P, cube.actnum_3d)
    assert m.shape == (2, 1, 2)
    assert not m.any()


def test_simple_2x2x2_no_fault_k(fixtures):
    g = read_grdecl(fixtures / "simple_2x2x2.grdecl")
    cube = from_grdecl(g)
    m = _faulted_pillar_mask_k(cube.P, cube.actnum_3d)
    assert m.shape == (2, 2, 1)
    assert not m.any()


def test_simple_3x3x2_no_fault(fixtures):
    g = read_grdecl(fixtures / "simple_3x3x2.grdecl")
    cube = from_grdecl(g)
    assert not _faulted_pillar_mask_i(cube.P, cube.actnum_3d).any()
    assert not _faulted_pillar_mask_j(cube.P, cube.actnum_3d).any()
    assert not _faulted_pillar_mask_k(cube.P, cube.actnum_3d).any()


# ── faulted deck: mask should pinpoint the fault stack ──────────────

def test_simple_faulted_2x2x2_i_fault(fixtures):
    g = read_grdecl(fixtures / "simple_faulted_2x2x2.grdecl")
    cube = from_grdecl(g)
    m = _faulted_pillar_mask_i(cube.P, cube.actnum_3d)
    # Fault is between i=0 and i=1 → row i=0 of the (NX-1,NY,NZ) mask
    # should be ALL TRUE (every active i-face in that stack is faulted).
    assert m.shape == (1, 2, 2)
    assert m.all(), "every active i-face should be flagged faulted"


def test_simple_faulted_2x2x2_no_j_fault(fixtures):
    g = read_grdecl(fixtures / "simple_faulted_2x2x2.grdecl")
    cube = from_grdecl(g)
    m = _faulted_pillar_mask_j(cube.P, cube.actnum_3d)
    # j-faces compare cells (i,j,k) vs (i,j+1,k) — both at the SAME i.
    # Since the offset is per-i-column, j-neighbours within the same
    # column are vertically aligned → no j-fault.
    assert not m.any(), "no j-fault expected on this fixture"


def test_simple_faulted_2x2x2_no_k_fault(fixtures):
    g = read_grdecl(fixtures / "simple_faulted_2x2x2.grdecl")
    cube = from_grdecl(g)
    m = _faulted_pillar_mask_k(cube.P, cube.actnum_3d)
    # k-faces compare cells (i,j,k) vs (i,j,k+1) — both at the SAME i,j.
    # The offset doesn't break k-direction matching.
    assert not m.any()


# ── behaviour-preserving: find_*_faces still raise on the fixture ──

def test_find_i_faces_still_raises_on_faulted_fixture(fixtures):
    g = read_grdecl(fixtures / "simple_faulted_2x2x2.grdecl")
    cube = from_grdecl(g)
    with pytest.raises(NonMatchingZError):
        find_i_faces(cube)


def test_find_i_faces_no_raise_on_matching_fixture(fixtures):
    """Tier A regression: the matching-Z path must be unchanged."""
    g = read_grdecl(fixtures / "simple_2x2x2.grdecl")
    cube = from_grdecl(g)
    # Should not raise — this is the existing PR-6 behaviour.
    faces = find_i_faces(cube)
    assert faces is not None
```

## Acceptance criteria

1. `tests/fixtures/simple_faulted_2x2x2.grdecl` parses with
   `read_grdecl`. `g.cart_dims == (2,2,2)`, `g.actnum.sum() == 8`.
2. `_faulted_pillar_mask_i/_j/_k` exist as importable names from
   `cpg2unstructured.unstructured` (private but accessible for tests).
3. `_verify_matching_z_i/_j/_k` are GONE — no fallback. Removed
   cleanly in one sweep, no aliases.
4. On all 5 existing matching-Z fixtures (`simple_2x2x2`,
   `simple_3x3x2`, `with_include/main`, `with_include_cycle`,
   `examples/quickstart`), the three masks return all-False.
5. On `simple_faulted_2x2x2`:
   - `_faulted_pillar_mask_i` returns shape `(1, 2, 2)` all-True.
   - `_faulted_pillar_mask_j` returns shape `(2, 1, 2)` all-False.
   - `_faulted_pillar_mask_k` returns shape `(2, 2, 1)` all-False.
6. `find_i_faces` raises `NonMatchingZError` on the faulted fixture
   (behaviour-preserving — the widening comes in PR-6).
7. All 1-design tests (97 tests at v0.1.0) pass unchanged.
8. New `test_faulted_mask.py` has 9 tests, all green.
