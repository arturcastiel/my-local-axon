# PR-01 ‚Äî Phase model + explicit DONE + trajectory keys
Wave A ¬∑ **AUTO** (non-kernel) ¬∑ serves R1, R2, R7, R8, R9 ¬∑ deps: none (foundation)

## Objective
Replace code-dev's phase tracking (hardcoded string literals in 3 sites + "file-exists == done"
heuristic) with a **data-driven phase manifest** carrying an explicit `done` status + dependency
edges ‚Äî the single source of truth for the dashboard, the in-order gate (PR-3/PR-6), explicit
DONE-to-advance (R1), and backward cascade-invalidation (PR-5). Also lands two small prerequisites:
phase_ledger trajectory keys (R8, for replay/promote in PR-11) and surfacing `active-phase` to the
verifier (prereq for R_STATE_SURFACED in PR-7).

## Design
- **`_phases.json`** (per project, `{project}/_phases.json`):
  `{schema:"v1", phases:[{id, name, order, status:pending|active|done|stale, deps:[ids]}], updated}`.
  code-dev default seed (lazy, on first access if absent): `study‚Üíplan‚Üípr‚Üílog‚Üíaudit`, linear deps.
  Authoritative; if a v4 `DAG.json` exists, its phase nodes/edges are reconciled into it (deps win
  from DAG.json). NOT stored in `_meta.md` (whitespace-fragile REPLACE).
- **`phase_model.py` API:** `load(project)` (seed default if absent) ¬∑ `status(project, phase)` ¬∑
  `advance(project, phase)` (set active; assert all `deps` are `done`, else error) ¬∑ `done(project,
  phase)` (set done; only if currently active AND deps done) ¬∑ `stale_downstream(project, phase)`
  (mark every transitive dependent `stale`; used by PR-5) ¬∑ `render(project)` (ordered list +
  statuses for the dashboard + the PR-7 state line). All pure over the json; no network.
- **`phase_ledger.py`:** `append_row(ws, program, phase, note="", run_id=None, workflow=None)` ‚Äî table
  gains two columns; `_ROW_RE` + `read_rows` updated; **back-compat:** rows written before the bump
  (4 columns) still parse (new fields default to ""). This makes a run's trajectory sliceable by
  `(run_id, workflow)` for PR-11 replay/promote.
- **`verify.load_state`:** add `active_phase = <read memory/working/active-phase.md or "">` and put
  `"active_phase": active_phase` in the returned dict (purely additive; no behavior change to existing
  rules). Unblocks R_STATE_SURFACED (PR-7) which needs the active phase.
- **`code-dev.md` dashboard:** the `study-ok/plan-ok/...` checks switch from `FILE-EXISTS(...)` to
  `phase_model.status(project, <phase>) ‚â° "done"`, with file-existence as a migration fallback when
  `_phases.json` is absent. Add the `done` subcommand ‚Üí `phase_model.done(project, <phase>)` + verbose
  narration of the transition.

## Gate-safety (cannot red the gate)
- `_phases.json` seeded lazily ‚áí existing projects (axon-bug-free, etc.) unaffected until they opt in.
- `load_state` change is additive (new key; no existing rule reads it yet).
- phase_ledger columns are back-compatible (old 4-col rows parse; new rows carry 6).
- New tool + program changes ship tests (R_NEW_NEEDS_TEST) and are invoked (code-dev.md calls
  phase_model) ‚áí pass the future liveness gate (PR-9) ‚Äî no new orphan.
