## Implementation outline

1. **Scaffold** — create `workspace/domains/{code-dev,library-dev}/{programs,workflows}/`
   each with an `_index.md` listing the programs/workflows that belong
   to the domain. The `_index.md` files are manifest-references
   (markdown lists), NOT moves; programs stay at their canonical path.
2. **Migrate** — write a one-shot script `tools/migrate_synapse_blocks.py`
   (lands as part of this PR) that:
     a. Iterates `workspace/programs/*.md`.
     b. Skips files that already carry a `# synapse:` block.
     c. Calls `synapse_infer.infer_program(path)` in-process.
     d. Serialises the inferred record into a `# synapse:` YAML-style
        comment block.
     e. Inserts the block immediately after the `# desc:` line (or, if
        absent, after the `# PROGRAM:` line).
     f. Writes the file in place (preserving line endings).
3. **Audit** — after migration, run `synapse-validate --all-corpus`
   and full-tree validate. Both rates must exceed the gate.
4. **Test** — add `tests/test_synapse_metadata_coverage.py` that:
     - asserts every `workspace/programs/*.md` contains a
       `# synapse:` block.
     - re-runs full-tree infer + validate and asserts rate ≥ 90 %.
5. **Rollback** — single-shot revert: `git checkout HEAD~1 -- workspace/`.
