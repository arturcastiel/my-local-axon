## Implementation outline

1. Promote spec → `workspace/DAG-SPEC.md` with provenance header.
2. `dag.py` subcommands:
   - `bootstrap {project|phase|plan|pr|study} --owner <slug>`
   - `mutate add|remove|status --id <node> [--depends-on <id>]`
   - `render <path>` → emit/refresh sibling `DAG.md`
   - `verify <path> [--all]` → schema + cycle + dangling-edge check
   - `sync <parent.json> <child.json>` → ensure subtree consistency
3. Migration pass: walk every `**/DAG.json` in repo, add
   `schema-version: v1` if missing, leave nodes/edges untouched, run
   `verify` on each post-migration.
4. Tests: snapshot render of a 5-node sample; deliberate cycle rejected;
   sync round-trip = zero diff.
