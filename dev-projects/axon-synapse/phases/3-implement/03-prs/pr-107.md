## Goal

Statement:       Ship two complementary tools:
                 `synapse-infer` parses program headers + bodies (and tool
                 REGISTRY entries) to emit a neuron-contract JSON record
                 per the v1.1 schema. `synapse-validate` checks an emitted
                 record against the schema + glossary terms + predicate
                 well-formedness.

Acceptance:      `tools/synapse_infer.py` and `tools/synapse_validate.py`
                 exist; both registered in REGISTRY; running
                 `synapse-infer --target workspace/programs/menu.md`
                 emits valid JSON that `synapse-validate --stdin` accepts;
                 corpus run over 20 sampled programs reaches ≥ 80 %
                 successful validation (D-6 coverage bar).

                 validation rate < 80 %; emitted records fail their own
                 JSON-schema.

## Implementation outline

1. **synapse-infer**:
   - Input: a program path OR a tool name. Walks header front-matter +
     section headings + control-flow ops.
   - Heuristics:
     · `precondition` ← extracted from any `ASSERT(...)` calls in LOAD
     · `post-state`   ← extracted from `STORE(W:...)` + `EMIT(...)` calls
     · `synapses`     ← extracted from `EXEC(...)` calls and `IF → EXEC`
     · `domain`       ← from `workspace/domains/*/manifest.md` verb-map
     · `glossary`     ← from front-matter `glossary:` line; default
                        AXON-GLOSSARY v2.
   - Output: stdout JSON conforming to neuron-contract v1.1.
2. **synapse-validate**:
   - Loads a contract record (file or stdin), checks: schema shape,
     glossary version, predicate well-formedness (calls `predicate
     eval --dry-run`), referenced synapses exist.
3. Register both in REGISTRY; smoke each via `--help`.
4. Corpus: pick 20 programs spanning code-dev, library-dev, kernel
   utilities; commit inferred outputs to
   `tests/synapse/corpus/*.contract.json` as snapshots.

(`synapse-suggest`, the orchestrator, shadow gates) consumes records
emitted here. The 80 % coverage bar is intentionally below the D-21 90 %
target — D-21 is a Phase-4 acceptance, not a PR-107 gate.

never overwrites an existing contract; it only emits, leaving merge to
`synapse-validate --merge` (out of scope here; future PR).
