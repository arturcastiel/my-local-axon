## Implementation outline

1. Lexer → token stream (whitespace-insensitive, quoted strings supported).
2. Parser → recursive-descent into the AST node set:
   `And · Or · Not · Cmp · Call · Var · Literal`.
3. Evaluator → walks AST against a context dict; built-ins from the spec.
4. CLI: `predicate eval --expr "..." --ctx '{json}'` returning JSON
   `{result, error?, trace?}`.
5. REGISTRY entry: `{name, path, category, smoke}`; smoke command =
   `predicate eval --expr "1 == 1" --ctx "{}"` returns `result: true`.
6. Tests: every spec example reproduced; rejection cases (type errors,
   unknown identifiers) produce structured errors, not crashes.

## Tests

- 50 fixture predicates (mix of valid / invalid).
- Unit tests for each built-in (`file.exists`, `file.contains`, `len`,
