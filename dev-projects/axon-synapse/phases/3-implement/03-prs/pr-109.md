## Goal

Statement:       Ship `synapse-suggest`, the ranker that combines current
                 workspace state, active goal, and history into a ranked
                 list of candidate synapses (programs + tools) for the
                 next step. Implements the combiner formula from
                 `phases/2-design/specs/orchestrator-composition-v1.md`.

Acceptance:      `tools/synapse_suggest.py` exists; registered in REGISTRY;
                 `synapse-suggest rank --state <fixture> --goal <fixture>`
                 returns a non-empty ranked JSON list with `score` ∈ [0,1]
                 per entry; against the 50-pair ranker fixture
                 (`tests/synapse/ranker_fixtures.json`), top-1 hit rate
                 ≥ 70 % (initial bar per D-21; Phase-4 target is 90 %).

Rejection:       Tool crashes on any fixture; output schema malformed;
                 top-1 hit rate < 70 %; combiner formula deviates from
                 orchestrator-composition-v1 § "The combiner".

## Implementation outline

1. Inputs: current state vector (W:/L: snapshot, active goal, recent
   history), candidate-pool path (defaults to `tools/REGISTRY.json` +
   indexed program contracts under
   `workspace/synapse/contracts/`).
2. For each candidate, score = combiner(
       precondition-match  (via predicate eval),
       goal-progress       (Δ predicted from post-state vs goal),
       history-weight      (recent failures penalised),
       domain-weight       (domain manifest match),
   ) — exact weights per spec § "The combiner".
3. Output: JSON list `[{name, score, why[]}]` sorted descending.
4. Tests: 50 hand-curated `(state, goal) → expected top-1` fixtures
   covering code-dev + library-dev + meta paths.
