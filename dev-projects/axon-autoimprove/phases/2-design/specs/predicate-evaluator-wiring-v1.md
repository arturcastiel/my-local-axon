## Problem

`tools/synapse_suggest.py:140-150` defines `_eval_simple_if(cond, state)` — a regex
placeholder that recognizes only two grammar forms:

```python
m = re.match(r"^state\.([A-Za-z0-9_\-]+)\s*==\s*['\"]?([^'\"]+)['\"]?$", cond)
m = re.match(r"^state\.([A-Za-z0-9_\-]+)$", cond)
return False   # ← anything else: silent False
```

Then at line 231:

```python
if pre and pre.lower() != "true" and not _eval_simple_if(pre, state):
    return False, f"precondition-false:{pre}"
```

Any precondition the regex doesn't understand is treated as **false** — the
candidate is silently filtered out. This is the wrong default for a ranker
filter: a filter that doesn't understand a condition should **let it through**
(fail-open), not drop it.

The fault is amplified by reality: every synapse generated by
`synapse-infer (PR-108 bulk migration)` is stamped with a precondition of
the shape `L:cognition-frame ≡ "AXON-OS"`. Confirmed at:

- `workspace/programs/orchestrator.md:9`
- `workspace/programs/auto-improve.md:9`
- `workspace/programs/code-dev-state.md:15`
- …and ~60 more.

None of these match the two regexes (different scope `L:`, different operator
`≡`). All are dropped. The orchestrator's "automatic ranking" path falls into
the zero-candidate branch (`workspace/programs/orchestrator.md:74-83`) and
surfaces a `QUERY user` to disambiguate — the ranker is effectively dead for
the bulk of real synapses.

## Why it matters

This is silent. `synapse-suggest rank` returns an empty list with no error;
the orchestrator interprets that as "no contracts match" rather than
"the filter is broken." The ranker has been shipping in this state since
PR-108. **Every dispatch where the orchestrator falls back to QUERY is
potentially a victim.** The hand-rolled regex was always a stub —
`tools/predicate.py` (the real evaluator, shipped in PR-102) has been
available the whole time.

## Solution

Wire `tools/predicate.py::evaluate_expr` into `tools/synapse_suggest.py` as
the primary evaluator. The placeholder regex stays as a **second-tier
fallback** only for the two original forms (defensive — predicate.py already
handles those, but keeping it costs nothing). Parse errors and unknown
references **default True** (fail-open).

### 1. Symbol normalization shim

The predicate language v1.1 uses ASCII operators (`==`, `!=`). Synapses
stamped by `synapse-infer` use the symbolic AXON-LANG forms (`≡`, `≢`).
Both are valid input to the ranker. Add a tiny normalizer in
`tools/synapse_suggest.py`:

```python
_SYMBOL_MAP = {
    "≡": "==",
    "≢": "!=",
    "∈": " in ",
    "∉": " not in ",   # predicate has 'not' as a keyword
    "∧": " AND ",
    "∨": " OR ",
    "¬": " NOT ",
}

def _normalize_symbolic(expr: str) -> str:
    for sym, ascii_ in _SYMBOL_MAP.items():
        expr = expr.replace(sym, ascii_)
    return expr
```

Pure string substitution. No regex; no escaping needed (the AXON-LANG
operators are single non-ASCII codepoints that cannot occur inside
quoted-string literals coming from `synapse-infer`).

### 2. Context-shape mapping

`synapse_suggest.py:filter_candidate` is called with a flat `state` dict
containing keys like `cognition-frame`, `drift-state`, `context-pressure`.
Existing preconditions use `state.<key>`. New preconditions use `L:<key>`
and `W:<key>`. Map all into the scope shape `predicate.py` expects:

```python
def _build_predicate_ctx(state: dict) -> dict:
    """Shape: predicate.py reads ctx[scope][path...].

    Synapse-suggest's flat `state` dict serves three scopes simultaneously
    in real usage:
      * `state.X` — the orchestrator runtime state (legacy synapses)
      * `L:X`    — locals (synapse-infer output)
      * `W:X`    — workspace memory (synapse-infer output)
    The orchestrator stamps both `state` and `L`/`W` into the same flat
    dict at call time. Mirror it into all three scopes; predicate.py
    resolves on first hit.
    """
    return {
        "state": dict(state),
        "L": dict(state.get("L") or state),
        "W": dict(state.get("W") or state),
        "S": dict(state.get("S") or {}),
    }
```

This is the minimum-surprise default. Callers that already segregate
scopes (`state["L"] = {...}`) keep the segregation; legacy flat callers
get every key visible under every scope (which is what the old regex did
implicitly via `state.X`).

### 3. Replace `_eval_simple_if`

New body — fail-open on any uncertainty:

```python
def _eval_simple_if(cond: str, state: dict) -> bool:
    cond = (cond or "").strip()
    if not cond or cond.lower() == "always" or cond.lower() == "true":
        return True
    expr = _normalize_symbolic(cond)
    ctx = _build_predicate_ctx(state)
    try:
        from tools import predicate
        result = predicate.evaluate_expr(expr, ctx, safe_null=True)
        return bool(result)
    except (predicate.ParseError, predicate.EvalError):
        # Fail-open: a filter that doesn't understand a condition
        # MUST NOT drop the candidate. Log via stderr for observability.
        sys.stderr.write(f"synapse-suggest: precondition unparseable, defaulting True: {cond!r}\n")
        return True
```

Module-level lazy import inside the function avoids a circular-import
risk during test collection (synapse_suggest is imported by many tests
that mock predicate).

### 4. New observability counter

Add a process-local counter `_PREDICATE_ERRORS = 0` incremented on every
fail-open path. Expose via a new `synapse-suggest stats` subcommand:

```
$ python3 axon.py synapse-suggest stats
{"predicate_errors": 3, "session": "..."}
```

Non-blocking. The counter resets per process. Future work: persist to
`MYAXON_ROOT/memory/local/synapse-suggest-stats.jsonl` (out of scope for
this spec — listed as a follow-up demand candidate).

## Tests (`tests/test_predicate_wiring.py`, 10 hermetic cases)

1. **legacy `state.X == "v"`** — current happy path still works.
2. **legacy `state.X`** — bare ref, truthy resolution.
3. **`always` / `true` / empty** — pass through.
4. **real synapse-infer precondition** — `L:cognition-frame ≡ "AXON-OS"`
   with `state={"cognition-frame": "AXON-OS"}` → True (the key regression).
5. **`W:dev-mode ≡ true`** — workspace scope works.
6. **AND composition** — `L:cognition-frame ≡ "AXON-OS" AND L:dev-mode`.
7. **NOT** — `NOT L:dev-mode` with dev-mode=false → True.
8. **`≢` normalized to `!=`** — `L:cognition-frame ≢ "OTHER"` → True.
9. **unparseable expression fails open** — `cond="???$#@"` → True + stderr msg.
10. **missing ref safe-null** — `L:never-set ≡ "X"` returns False (not crash).

Each test uses `filter_candidate(c, state)` end-to-end. No mocking of
predicate.py — actual evaluator runs. Hermetic: pure in-memory dicts.

## Per-consumer policy

This change is **filter-only** — `filter_candidate` is the single call site
of `_eval_simple_if` inside synapse_suggest. `score_candidate`'s use at
line 135 (inside `next_conditional`) is separate: it sums confidences across
clauses. That path is also fail-open today (no match → 0.0 contribution,
not -∞), so the only behavioral change there is that real preconditions
now contribute their confidence instead of silently zero. No widening
required at line 135 — predicate.evaluate_expr replaces the regex
transparently.
