# PR-12.1 — enforce.py: remove `user:` prefix bypass + raise on stubs

## 1. Why
F-D7-007a (BLOCKER, surfaced iter 2): `tools/enforce.py:73` returns `{valid: true}` for any source string starting with `"user:"` without inspection. Any caller (program, tool, user instruction) can pass `--source "user:fabricated"` and Rule 2 ("never execute a task with no instruction source") signs off.

F-D7-007 (BLOCKER): `check-arithmetic` and `check-source` print JSON and fall through (no `sys.exit`). Compare to `check-write:43-63` which actually `sys.exit(1)`. Callers using subprocess return-code semantics get exit code 0 regardless.

The kernel cites `enforce.py` as the "machine-executable form of the write gate" (KERNEL-SLIM:478). Today only `check-write` actually gates.

## 2. Evidence
- `tools/enforce.py:71-75`:
  ```python
  if args.action == "check-source":
      exists = os.path.exists(args.source) if not args.source.startswith("user:") else True
      print(json.dumps({"action": "check-source", "source": args.source, "valid": exists}))
  ```
- `tools/enforce.py:65-69`:
  ```python
  if args.action == "check-arithmetic":
      print(json.dumps({"action": "check-arithmetic", "expression": args.expression,
                        "required_tool": "calculator", "advisory": "use TOOL(calculator,...)"}))
  ```
- Verified directly 2026-05-22.

## 3. Design notes
- `user:` short-circuit was likely added so user-typed instructions bypass file-existence check. The shape is wrong — anyone can write `user:fabricated` and get through.
- Two paths forward:
  - **A**: Remove the short-circuit. Any `--source` must be a file path that exists OR explicit `--user-instruction "..."` (separate flag) which logs to `E:source-log` for audit.
  - **B**: Drop check-source entirely. It's advisory anyway; no real gate.
- Recommend **A** — preserves auditability of user-instruction sources without trivial bypass.

## 4. Pitfalls
- Class-A (production-path): existing callers of `enforce check-source --source "user:..."` will break. Search before changing.
- Class-B (subprocess return code): even after fix, exit-0 behavior of check-arithmetic remains. Either implement (call calculator + verify) OR explicit `--advisory` flag + docstring update so callers know not to check return code.
- Class-E (rule violation): the new `--user-instruction` flag MUST log to E:source-log; otherwise the gate becomes ineffective again.

## 5. Interface sketch
```bash
# Old (broken):
python3 tools/enforce.py check-source --source "user:anything"  # → {"valid": true}

# New (fixed):
python3 tools/enforce.py check-source --source path/to/file     # → {"valid": <file exists>}
python3 tools/enforce.py check-source --user-instruction "do X" # → {"valid": true, "logged": true}
                                                                # AND appends to E:source-log
python3 tools/enforce.py check-source --source "user:..."       # → exit 2, "use --user-instruction"
```
