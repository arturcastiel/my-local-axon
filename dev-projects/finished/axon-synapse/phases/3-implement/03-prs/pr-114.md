## Implementation outline

1. **G2 — safety-audit row**: append a row that calls into existing
   shadow index (`tools/shadow.py`) and aggregates covered vs total
   source artifacts under the loaded project's globs.
2. **G3 — bulk-phase shadow**: extend the shadow program with a flag
   that walks `phases/{slug}/` and shadows every matching source.
3. **G4 — audit field**: code-dev-audit's JSON output gets a top-level
   `shadow.coverage` object so PR-117 can consume it without re-parsing.
4. **G5 — release-readiness stub**: a section in audit output that
   reads `shadow.coverage.percent` and fails (exit ≠ 0) when below the
   project-declared threshold (default 100; configurable per project).
5. **G1 — author-time check**: a soft warning in `code-dev-pr-author`
   when no synapse contract is present for any new `tools/*.py` file
   (does NOT block; logs WARN). Hard-gating happens in PR-117.
