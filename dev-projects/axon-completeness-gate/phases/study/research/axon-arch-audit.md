## 1. [CRIT] PreToolUse write-gate matcher omits Bash — native shell writes to axon/ bypass the only mechanical R9 chokepoint
- kind: gate-coverage · file: `axon/.claude/settings.json` · locator: hooks.PreToolUse[0].matcher = "Write|Edit|NotebookEdit" (no Bash); enforce_pretooluse is the sole mechanical R9 enforcer
- confidence: 0.88 · fp_risk: low
- **evidence:** settings.json registers exactly one PreToolUse hook with matcher `Write|Edit|NotebookEdit`. There is NO `Bash` matcher in axon/.claude/settings.json (or ~/.claude/settings.json). enforce_pretooluse.py's own comment calls itself "the ONLY mechanical R9 enforcer". tools/shell.py top docstring documents F-D8-001 vec-4: `TOOL(shell,"cp x axon/y")` writing into axon/ regardless of dev-mode, and says PR-1.1 closes it via a chokepoint — but that chokepoint is `python3 tools/shell.py exec --argv ...`, which the agent must VOLUNTARILY invoke. - **why arch bug:** R9 ("axon/ writes require L:dev-mode") is the keystone of the 3-layer model: axon/ (kernel/law) must be immutable to programs and to ordinary sessions. - **fix:** Add a `Bash` (and `MultiEdit`) PreToolUse matcher that routes through an argv-inspecting gate (reuse tools/shell.py's _is_axon_path + dev-mode check on every write-shape command and redirect), denying axon/-targeting writes when dev-mode!=true.
