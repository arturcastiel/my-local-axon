## Summary
Pin the AXON boot sequence with a state-machine driven test suite using
the PR-005 `workflow_test.py` harness plus a focused behavioural file.
Every session starts with boot; an unintended drift here breaks every
downstream workflow.

## Goal Context
Study finding F-7: boot is fully specified in `axon/BOOT.md` and W-08
but never asserted end-to-end. Co-output: new
`AXON-DOCS-WORKFLOWS.md § W-08 Boot` lists this PR's cases.

## Entry Conditions
- PR-005 merged (`tools/workflow_test.py` available, registered).
- `axon/BOOT.md` and `axon/KERNEL-SLIM.md § Boot Steps` unchanged
  since last audit.
- Workspace fixtures dir created in PR-005.

## Changes Required

### tests/test_boot_contract.py  (new)
- **What:** Behavioural assertions independent of the harness:
  · step-1 STORES (`L:cognition-frame`, `W:reasoning-mode`,
    `L:host-harness`, optionally `L:host-model`),
  · G-10 HALTs with the canonical message when `ws-programs` map
    cannot resolve,
  · G-11 routes correctly for CLAUDECODE, COPILOT, and generic
    runners (per `workspace/harness/*`),
  · my-axon detection branches: missing → QUERY; present → MYAXON.md
    ops evaluated; absent + skip → degraded-mode warning.
- **Why:** These are decision-point assertions; cleaner as Python
  cases than as a state replay.

### tests/test_workflows/test_w08_boot.py  (new)
- **What:** Drive `workflow_test.py` against
  `tests/fixtures/workflows/w08-boot.flow.jsonl`: simulate a clean
  boot and assert final state contains the banner + menu markers and
  expected STORE side-effects on disk.
- **Why:** Locks ordering and output ordering.

### tests/fixtures/workflows/w08-boot.flow.jsonl  (new)
- **What:** Sequenced JSONL of `{input, expected_state, expected_files}`
  steps capturing one full boot.

### axon/BOOT.md (read-only)
- **What:** No edits — this PR asserts current behaviour. A follow-up
  may polish wording; out of scope here.

### workspace/AXON-DOCS-WORKFLOWS.md
- **What:** Add a `## W-08 Boot` section with `## Guarded by` block.
- **How:** Per PR-004 template.
