# PR-003 — Install scan_pre_push.py as a real git hook
Project:     axon-tests
Created:     2026-05-16
Complexity:  S
Depends on:  none
Status:      not-started
AXON score:  8/10

## Summary
`tools/scan_pre_push.py` (PR-5 in the original axon work) is a
working secret-scan implementation that was never wired as a real
hook. This PR adds a one-time installer that creates `.githooks/`,
copies a `pre-push` script that invokes the scan, and configures
`git config core.hooksPath .githooks` so contributors get the hook
on the next clone without per-machine setup.

## Goal Context
From 01-study.md F-2: "`scan_pre_push.py` exists but isn't installed".
Phase-D PR-019 layers a smoke-pytest gate on top of this same hook.

## Entry Conditions
- Repo's `.git/hooks/` contains only `*.sample` files.
- `tools/scan_pre_push.py` runs cleanly on a known-bad fixture (test
  in `tests/test_redact.py` adjacent).

## Changes Required

### .githooks/pre-push  (new, executable)
- **What:** Bash wrapper that resolves repo root and runs the
  secret scanner. Exits non-zero on hit.
- **How:**
  ```bash
  #!/usr/bin/env bash
  set -euo pipefail
  ROOT="$(git rev-parse --show-toplevel)"
  python3 "$ROOT/tools/scan_pre_push.py" || {
    echo
    echo "✗ pre-push: secret scan blocked the push."
    echo "  Fix: see workspace/safety/redact-allowlist.md if it's a"
    echo "  legitimate fixture (add an 'allow:' line)."
    exit 1
  }
  ```
- **Why:** Self-contained; portable across Linux/macOS/WSL; no
  per-machine python path baked in.

### scripts/install-hooks.sh  (new, executable)
- **What:** One-time installer.
- **How:**
  ```bash
  #!/usr/bin/env bash
  set -euo pipefail
  cd "$(git rev-parse --show-toplevel)"
  git config core.hooksPath .githooks
  chmod +x .githooks/*
  echo "✓ Hooks installed (.githooks/ → core.hooksPath)"
  echo "  Active hooks:"
  ls -1 .githooks/
  ```
- **Why:** Single command, idempotent, no symlinks (Windows-safe).

### SETUP.md
- **What:** Add a "Git hooks (recommended)" section.
- **How:** Two lines: `bash scripts/install-hooks.sh`. Note that
  PR-019 will add a smoke-pytest gate that runs in ≤ 5 s.
- **Why:** Discoverability; we expect every new clone to run it.

### axon/HOWTO.md
- **What:** Cross-reference from the existing setup guidance.
- **How:** One bullet under the "Per-machine setup" section pointing
  at `scripts/install-hooks.sh`.

### tests/test_install_hooks.py  (new)
- **What:** 2 cases.
  1. `scripts/install-hooks.sh` is executable and contains
     `core.hooksPath .githooks`.
  2. `.githooks/pre-push` is executable, has a shebang, and invokes
     `tools/scan_pre_push.py`.
- **How:** Plain file-system assertions; no shell exec.
- **Why:** Catches accidental file-mode loss on Windows/WSL.

## Architecture Impact
Pure dev-tooling addition; no runtime, no kernel.

## Acceptance Criteria
- ✓ `bash scripts/install-hooks.sh` sets `core.hooksPath` to
  `.githooks` (verify via `git config --get core.hooksPath`).
- ✓ Re-running the installer is a no-op (idempotent).
- ✓ Adding a fake secret (e.g. `AWS_SECRET = "AKIA...."`) and
  running `git push` blocks with exit 1 and a clear message.
- ✓ `git push` of a clean diff passes immediately.
- ✓ The two new pytest cases pass.

## Risks & Gotchas
- ⚠ Existing local hooks under `.git/hooks/` are not moved; users
  who already wrote their own hooks need to migrate or merge them.
  Document this in SETUP.md.
- ⚠ `git config core.hooksPath` is per-clone, not committed; the
  installer is run-once-per-clone — that's expected.
- ⚠ The hook must not call any `axon/` tool that itself runs the
  hook (no infinite loop possible because the hook only invokes the
  scanner; but worth a comment).
- ⚠ On Windows + git-bash, the shebang `/usr/bin/env bash` works;
  no changes needed.
