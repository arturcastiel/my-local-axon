## Summary
Scaffold `tests/test_rules/` with **one stub file per rule module**
under `tools/rules/r*.py`, plus a **meta-test** that fails CI when:
1. a new `tools/rules/r*.py` is added without a matching
   `tests/test_rules/test_<rule>.py`,
2. a `rule_id` exists in the registry but no assertion references
   it anywhere in `tests/`,
3. a rule module is missing one of `phase`, `severity`, `rule_id`.

Stubs are intentionally empty (xfail-marked) so this PR doesn't
touch behaviour. PR-009 (R9) and PR-011 (the rest) fill them in.

## Goal Context
From 01-study.md F-4 and `helpers/rules-crosswalk.md`: 9 rule
modules, 1 has dedicated tests, 0 have positive+negative pairs.
The meta-test is what makes "every rule must have a test" mechanical.

## Entry Conditions
- PR-001 is in: full pytest suite runs in CI.
- `tools/rules/registry.py` exposes `ALL_RULES` (it does).

### tests/test_rules/conftest.py  (new)
- **What:** Shared fixtures for rule tests.
- **How:**
  - `rule_ctx_factory()` — builds a minimal context dict matching
    the shape each rule's `check(ctx)` expects (`program_text`,
    `output_text`, `action`, `state`, `registry`).
  - `clean_program_text` — fixture returning a known-clean
    program text (e.g. the body of `workspace/programs/menu.md`
    truncated, with no symbolic ops and no PLANNED tools).
  - `axon_target` — fixture for an action dict targeting an
    `axon/` path.
- **Why:** Halves the boilerplate in 9 stub files.

### tests/test_rules_meta.py  (new)
- **What:** Three cases.
  ```python
  def test_every_rule_module_has_a_test_file(): ...
  def test_every_rule_id_referenced_somewhere_in_tests(): ...
  def test_every_rule_module_declares_phase_severity_rule_id(): ...
  ```
- **How:**
  - Case 1: glob `tools/rules/r*.py`, glob `tests/test_rules/r*.py`
    and `test_r*.py`, assert bijection (minus `__init__`,
    `conftest`, `registry`).
  - Case 2: import `rules.registry.ALL_RULES`, grep
    `grep -rE "['\"]<rule_id>['\"]" tests/` for each — must hit ≥1.
    (Allow xfail-only references? **No** — must be a real assertion;
    we re-evaluate this after PR-011 lands.)
  - Case 3: for each rule module, `importlib`-load it and assert
    the three module-level constants are set and non-empty.
- **Why:** Mechanical enforcement. The only way to add a rule
  without a test is to deliberately delete a meta-case.

## Acceptance Criteria
- ✓ `pytest tests/test_rules/ tests/test_rules_meta.py` is green
  (stubs xfail-pass, meta-tests pass).
- ✓ Deleting `tests/test_rules/test_r9_axon_write.py` causes
  `test_every_rule_module_has_a_test_file` to fail.
- ✓ Adding a hypothetical `tools/rules/r_new.py` (in a follow-up
  branch) without a matching test fails the meta-test.
- ✓ Removing `phase` from a rule module fails meta-case 3.
- ✓ Existing `TestDriftGate` in `test_tools_kernel.py` remains
  untouched (migration deferred to PR-011) — meta-case 2 still
  passes because `R_DRIFT_GATE` is referenced there.

## Risks & Gotchas
- ⚠ `test_governance.py` references `tools/rules.py` (singular) —
  a completely different subsystem. The meta-test must NOT confuse
  them. Use file paths under `tools/rules/` (dir) explicitly.
- ⚠ Naming clash: `tools/rules.py` (workspace/safety loader) and
  `tools/rules/` (runtime predicates). Add a comment in the
  meta-test to call this out. PR-020 / PR-021 docs should note it.
- ⚠ xfail markers must use a strict reason string so PR-011 can
  flip them off mechanically.
