# PR-018 — Make doc-co-output verifier BLOCKING
Project:     axon-tests
Created:     2026-05-16
Complexity:  S
Depends on:  PR-004, PR-011, PR-013, PR-014
Status:      not-started
AXON score:  9/10

## Summary
Flip `docgen_verify.py` from advisory (PR-004) to blocking in CI for
`AXON-DOCS-*.md` files lacking a `## Guarded by` block. Add an
`axon-audit` check that flags any `tools/rules/` rule whose Guarded-
by row is empty.

## Goal Context
Closes F-12 fully. This is the contractual lever: a future
contributor who adds a doc page without tests, or a rule without a
doc, cannot merge.

## Entry Conditions
- PR-004 merged (verifier exists, advisory mode).
- PR-011/013/014 merged (all rules + workflows have Guarded-by
  rows) — otherwise this PR's CI would fail immediately.

## Changes Required

### tools/docgen_verify.py
- **What:** Add `--strict` flag that returns non-zero when any
  AXON-DOCS-* file lacks `## Guarded by`. Default behaviour
  unchanged (advisory).

### .github/workflows/ci.yml
- **What:** Add a `docgen-strict` job:
  ```yaml
  - name: Doc co-output gate
    run: python3 tools/docgen_verify.py --strict
  ```
  Required for merge.

### tools/axon_audit.py
- **What:** Add a rule that, for each `tools/rules/r*.py`, walks
  AXON-DOCS-GOVERNANCE and asserts a Guarded-by row references it.
  Empty row → audit failure.

### tests/test_docgen_strict.py  (new)
- **What:** 4 cases —
  · file with Guarded-by → strict passes,
  · file without → strict fails with rc != 0,
  · non-AXON-DOCS file → ignored,
  · empty Guarded-by row → audit flags it.

### workspace/AXON-DOCS-GOVERNANCE.md
- **What:** Add `## Doc co-output gate` section + Guarded-by.

## Architecture Impact
A new mandatory CI gate. Onboarding doc (CONTRIBUTING — PR-020)
must explain it.

## Acceptance Criteria
- ✓ `python3 tools/docgen_verify.py --strict` returns 0 on the
  current repo (post-merge of PR-011/13/14).
- ✓ Removing the Guarded-by from any AXON-DOCS-*.md fails CI.
- ✓ Adding a stub `tools/rules/r_dummy.py` without a
  Governance row fails audit.
- ✓ 4 new tests pass.

- ⚠ `--strict` exit-code contract must match what GH Actions parses.
