## Summary
Add a **subprocess-replay test harness** for whole-workflow tests:
load a `<wf>.flow.jsonl` fixture (sequence of `{input,
expected_state, expected_files}`), run `python3 axon.py …` against a
sandboxed workspace, assert state snapshots + file artefacts after
each step. This is the substrate for PR-008 (boot), PR-010
(workspace-backup), PR-013/014 (workflows).

`workflow_test` is distinct from the existing mock-model
behavioural harness (`tests/_mock_model.py` + `test_behavior.py`),
which we keep for prose-output assertions. Two harnesses, two
purposes:
- **mock-model harness** — checks the *text* the agent emits.
- **workflow harness** (this PR) — checks the *state machine* axon
  drives (W:/L: keys, files written, exit codes, banner shape).

## Goal Context
From `helpers/workflows-catalog.md`: 15 workflows, 0 e2e. Boot
(W-08), identity (W-14), workspace-backup (W-15) are state-machine-
heavy and don't need a mock model; they need a deterministic replay
harness.

## Changes Required

### tools/workflow_test.py  (new)
- **What:** CLI tool + library, ~250 LOC.
- **How:** Public API:
  ```
  workflow_test.py run --fixture tests/fixtures/workflows/w08-boot.flow.jsonl
                       --tmp-workspace /tmp/wfX
                       [--keep] [--json]
  workflow_test.py list                     # discovers fixtures
  workflow_test.py validate <fixture>       # schema-check, don't run
  ```
  Fixture schema (one line per step):
  ```json
  {"step": 1, "input": "boot axon",
   "expected_state": {"W:active-program": "menu",
                      "L:cognition-frame": "AXON-OS"},
   "expected_files": [{"path": "workspace/memory/working/active-program.md",
                       "contains": "menu"}],
   "expected_exit": 0}
  ```
  Implementation:
  1. Copy the repo's `workspace/` skeleton into `--tmp-workspace`
     (or use an existing sandbox under `tests/fixtures/sandbox/`).
  2. For each step: run `python3 axon.py <input>` with
     `AXON_WORKSPACE=<tmp>` env, capture stdout/stderr/exit.
  3. Read expected W:/L: keys via `python3 tools/kv_store.py get`
     against the tmp workspace; assert match.
  4. Read each expected file; assert `contains` substring.
  5. On mismatch, return JSON with step-level diffs.
- **Why:** Deterministic, fast (no model calls), independent of
  network and credentials.

### tests/fixtures/workflows/  (new dir)
- **What:** Initial fixture set — `_sample.flow.jsonl` plus an
  empty `.keep` for each W-NN we'll fill later.
- **How:** `_sample.flow.jsonl` is a 2-step fixture used by this
  PR's own tests (e.g. `boot axon → menu`); subsequent PRs populate
  the per-workflow files.

### tests/test_workflow_test.py  (new)
- **What:** 6 cases on the harness itself.
  1. `validate` accepts the `_sample.flow.jsonl` fixture schema.
  2. `validate` rejects a malformed fixture (missing `step`).
  3. `run` against `_sample.flow.jsonl` exits 0.
  4. `run` against a fixture with wrong `expected_state` exits 1
     and the JSON diff names the offending key.
  5. `run` against a fixture with wrong `expected_files.contains`
     exits 1 and the JSON diff names the offending path.
  6. `list` discovers fixtures under `tests/fixtures/workflows/`.

### tools/REGISTRY.json
- **What:** Register `workflow-test` as ACTIVE.

### workspace/AXON-DOCS-TESTING.md
- **What:** Add a "## Workflow harness" section.
- **How:** Cover purpose, fixture schema, two-harness split,
  examples; ends with `## Guarded by` listing the 6 cases above.

## Architecture Impact
- Adds a new tool — no kernel impact.
- Establishes a workspace-sandbox convention (`AXON_WORKSPACE` env
  override). `tools/_axon_paths.py` already honors this; this PR
  pins it via tests.
- Becomes the **state-machine test substrate** for PR-008, 010,
  013, 014, 016.

## Acceptance Criteria
- ✓ `python3 tools/workflow_test.py validate
  tests/fixtures/workflows/_sample.flow.jsonl` exits 0.
- ✓ Mutating one expected key to a wrong value causes `run` to exit

- ✓ Harness uses ≤ 5 s per fixture on a 2-step sample.
- ✓ Harness sandboxes correctly: tmp workspace is created, used,
  and cleaned up (unless `--keep`).
- ✓ All 6 test cases pass.

## Risks & Gotchas
- ⚠ `axon.py` invokes many subprocesses internally; pinning a tmp
  workspace requires `AXON_WORKSPACE` (or equivalent) to be honored
  end-to-end. If `tools/_axon_paths.py` is bypassed anywhere, the
  harness leaks into the real workspace. Validate by asserting the
  real workspace is untouched after a run.
- ⚠ State snapshots via `kv_store.py` must work on the tmp
  workspace; if any tool hardcodes paths, fix that tool (PR-017
  will pick up coverage gaps).
- ⚠ Don't replicate the mock-model harness. Document the split in
  AXON-DOCS-TESTING.
