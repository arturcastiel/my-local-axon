## Problem
Goal A mandates measure-before-optimize, but token spend is invisible: instrumentation
fragments exist (tokenizer, usage, dispatch-stats, gain) with no unified per-scenario
picture of what the AGENT READS — the only spend that matters (subprocess output is
free until it enters context).

## Change
`tools/token_bench.py` (NEW tool — justified as Goal A's instrument; registered, tested):
- Measures AGENT-READ cost per scenario = file-reads (tokenized via tokenizer/tiktoken,
  word-count fallback if tiktoken absent) + tool-envelope sizes (run the real tool,
  tokenize its stdout).
- Scenarios (deterministic definitions in-code):
  `boot-menu`  — boot chain reads (startup.md, KERNEL-SLIM.md, menu.md, harness
                 contract) + boot/prefs envelopes + the ~10 menu state probes.
  `pr-cycle`   — program reads a PR cycle EXECs (pr-create, review+subprograms,
                 finalize) + representative gate envelopes (registry-drift, freshness).
  `study`      — code-dev-study.md + study-modes/shadow envelopes.
  `chat-turn`  — scripted: dispatch match envelope + mode-detect read (fixed query).
- `run [--scenario X] [--out <json>]` → per-scenario {file_tokens, envelope_tokens,
  total, breakdown}; `baseline --out <path>` writes the full report.
- Baseline artifact: dev-projects/axon-plus/baselines/baseline-2026-06-11.json + .md
  summary (project artifact, not repo).

Tests: scenario definitions resolve to existing files · token counting matches
tokenizer on a fixture · report shape stable · missing-tiktoken fallback works.

## Out of scope
Any optimization (W1). Live per-session metering (later if needed).

## Gates
R13 tests · lint-paths · freshness · crucible green.

## Acceptance
Baseline report exists with real numbers for all 4 scenarios; rerunnable; PR-028 will
