## Problem
The kernel's compiler spec mandates TOOL(compile-write) — which never existed; the
pipeline sat at 0/191 while every EXEC paid the full source read (study 6,646 tokens,
plan 5,797, menu 5,762 per read).

## Change
- `tools/compile_write.py` (NEW, registered, kernel category): Phase-4 writer per
  COMPILER.md — deterministic hybrid .cmp.md render + compiler log + benchmark rows.
- Six programs compiled (evidence-picked; usage data was empty): menu 10% ·
  code-dev-study 22% · code-dev-plan 24% · code-dev-review 43% ·
  code-dev-knowledge-impact 35% · mode-detect 19%.
- Compile method: deterministic comment/doc strip — EVERY functional line survives
  verbatim in order (equivalence BY CONSTRUCTION; no op rewriting in the pilot).
- CI invariants: equivalence test (functional-line survival) + STALENESS test
  (source newer than compiled = failure → recompile is mechanically forced).

## Pilot findings (for the W1 re-plan checkpoint)
- Comment-strip pays 19–43% on prose-heavy programs, only 10% on op-dense ones (menu).
- Next tier (checkpoint candidates): sectional reads for router programs
  (code-dev.md 8k → ~1k per matched branch) · semantic op-compression as compile
  target (owner's denser-vocabulary idea) · kernel diet (owner-only).
