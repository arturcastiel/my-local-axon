## Problem
Goal G's worst observed failure: weak models MIMIC execution — they render plausible
tool output without running anything (Core Rule 6 fabrication). Today nothing
mechanical can tell a real TOOL() run from a hallucinated one.

## Design decision (vs spec sketch)
Spec said "tool envelopes emit a nonce" — per-tool change ×163. Stronger and smaller:
the DISPATCHER writes the receipt. Every `axon.py <tool>` invocation appends one JSONL
line to `workspace/log/receipts/<date>.jsonl` (ts, tool, argv, rc, nonce) — one edit
covers all tools, present and future. Fail-safe: receipt write can never break dispatch.

## Change
- `axon.py`: append receipt line after subprocess.run (try/except — never raises).
- `tools/rules/r_tool_receipts.py` (NEW rule, R_GROUNDED_CLAIMS shape): opt-in via
  `L:tool-receipts-required` — silent by default, BLOCK when on. Scans the turn's
  output/reasoning-trace for claimed `TOOL(name` executions; each claimed tool must
  have a receipt line TODAY. Claimed-but-no-receipt = mimicry signal = BLOCK.
- `tools/verify.py`: read the new flag into rule state; manifest registration.
- Honest limit documented: a model invoking scripts directly (not via axon.py) writes
  no receipt — the rule's posture is "claims must be receipted", which forces weak
  models ONTO the dispatcher path; direct-run mimicry is caught because fabricated
  output has no receipt either way.

Tests: dispatcher writes receipts (real subprocess) · rule fires on claimed-without-
receipt (flag on) · passes with matching receipt · silent with flag off · receipt
write failure doesn't break dispatch.

## Gates
R13 tests · lint-paths · freshness · crucible green.

## Acceptance
Mimicry is mechanically detectable: with the flag on, a response claiming TOOL() runs
that never happened is BLOCKED at the response gate.
