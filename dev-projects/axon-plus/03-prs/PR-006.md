## Problem
Baseline: boot-menu = 32,593 tokens; ~12,004 of it is tool envelopes, dominated by the
~10 separate state probes the menu program needs (clock, dispatch-index, drift,
prompt-log consent, todos, workflows, auto-audit, health…) — each returning a verbose
envelope the agent must read.

## Change
`tools/axon_state.py` gains `menu-snapshot` (extends existing tool — reduce-surface):
ONE subprocess gathers every menu input in-process and returns ONE compact JSON of
exactly the fields the menu renders (no verbose envelopes — counts and values only).
`workspace/programs/menu.md` LOAD CONTEXT consumes TOOL(axon-state, menu-snapshot)
with per-field fallback to the old probes (fail-degrade).
Protected core: menu CONTENT unchanged — equivalence test pins snapshot fields ==
individually-probed values; token-bench scenario updated to the new path measures the
reduction honestly.
