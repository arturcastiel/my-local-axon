## Problem
Core Rule 12 ("menu always rendered IN FULL") is honor-system — exactly what observed
weak models violate first (they skip or truncate the menu to save tokens). Nothing
mechanical catches it.

## Change
`tools/rules/r_menu_rendered.py` (opt-in via L:menu-render-required, BLOCK when on):
- Partial render: output looks like a menu (MODES header + [1] entry) → every canonical
  section must be present (OS STATE · MODES · CODE DEVELOPMENT · WORKFLOWS · QUALITY /
  SELF-IMPROVEMENT · DISCOVER · SELF-OBSERVE · META TOOLS); missing = BLOCK naming them.
- Skipped render: state.active_program == "menu" and output is not a menu → BLOCK.
Wiring: registry import + manifest row + verify.py flag (L:menu-render-required).
Tests: tests/test_rules/test_r_menu_rendered.py (convention learned in pr-2):
full menu passes · truncated menu blocks with section names · skipped menu blocks via
active_program · silent when flag off.

## Gates
R13 tests · lint-paths · freshness · crucible green.

## Acceptance
With the flag on, a truncated or skipped menu render is mechanically BLOCKED (Rule 12
becomes enforcement, not discipline).
