## Problem
- Review scope = literally-changed files only; transitive callers stay invisible until a
  late failure.
- test-map is a filename heuristic (`find -name '*{base}*'`) — misses cross-name
  coverage, false-positives same-named files.
Both have the study graph available since PR-007 and don't use it.

## Change
`tools/graphify_bridge.py`:
- `affected` gains `--tests-only` + result key `affected_files` (sorted unique
  source_files of affected nodes). tests-only keeps nodes whose source_file is a test
  (basename `test_*` / `*_test.*` or a `/tests/` path segment). Deterministic.

`workspace/programs/code-dev-review-scope.md`:
- Graph block after the changed-list: ∀ changed file → file-nodes → affected (depth 3)
  → union of caller files NOT in the diff → rendered as the advisory caller cone
  ("review these too"). Graph absent → silent skip.

`workspace/programs/code-dev-test-map.md`:
- Graph block per source file: file-nodes → affected --tests-only → real call-based
  covering tests rendered alongside the filename heuristic; empty set = precise
  untested-node gap. Graph absent → heuristic only (today).

Tests: tests-only filter + affected_files key (fixture graph with a test caller),
program text pins for both consumers.

## Out of scope
Adaptive synapse (PR-010) · changing the heuristic path (it remains the degrade mode).

## Gates
R13 tests · lint-paths · freshness · crucible green.

## Acceptance
- Fixture: changing a callee surfaces its caller file in the review cone; a test that
  calls the symbol shows as real coverage; both programs silent without a graph.
