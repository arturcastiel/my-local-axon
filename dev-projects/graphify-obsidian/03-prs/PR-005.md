# PR-005 — Richer Obsidian projection (wikilinks, community pages)

**Project:** graphify-obsidian · **Phase:** 2-living-map · **Complexity:** M
**Status:** merged (squash f92ead3, main, 2026-06-11) · **Branch:** graphify-obsidian/pr-5-rich-projection
**Created:** 2026-06-11 · **Depends on:** PR-004 (merged 4e8017f)

## Problem
The map is a flat dump: no frontmatter, no internal navigation, god-nodes don't say
which subsystem they belong to, members carry no source location. In Obsidian it reads
as one long list — the "navigable map" payoff is not realized.

## Design decision
ONE canonical file, enriched — NOT a 92-file vault directory. Rationale: the per-page
vault explodes churn (92 files regenerated weekly), forces the PR-004 byte-compare
reconciler into directory semantics, and grows surface. Obsidian renders
`[[#heading]]` wikilinks natively, so a single file with an index + cross-links is
genuinely navigable. Multi-page vault recorded as NICE/deferred in phase-2 file.

## Change
`tools/code_graph.py` — `export_markdown()` v2 (pure function, deterministic, no
timestamps):
- YAML frontmatter: title, root, node/edge/community counts, deterministic flag.
- Index section: wikilinks to god-nodes + every community heading.
- God-nodes: each hub links to its community section ([[#Cn ...]]).
- Communities: heading per community (stable anchor), members rendered with
  `file:lineno` source locations; >40-member truncation kept.
- Node→community lookup built from the cluster result (deterministic).

Tests: frontmatter shape, index present, god-node→community wikilink resolves to an
existing heading, member lines carry file:lineno, byte-determinism (two exports equal).
Regenerate the committed map; freshness `code_map` stays green (same reconciler).

## Out of scope
Multi-page vault (deferred NICE) · graph-view frontmatter tuning · discoverability (PR-006).

## Gates
R13 tests · lint-paths · freshness (code_map must be green post-regen) · crucible.

## Acceptance
- Map opens in Obsidian with a working index; clicking a god-node jumps to its subsystem.
- `freshness check` green with the regenerated artifact committed.
