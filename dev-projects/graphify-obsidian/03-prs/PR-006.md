## Problem (owner addendum 2026-06-11)
The graph tools "get lost inside AXON." Measured baseline (dispatch match):
- "what calls default_workspace" → deps @ 0.433 (wrong, below 0.65 → agent fallback)
- "map this repo" → code-dev-test-map (wrong)
- "show me the most connected modules" → noise @ ~0
- "is this function dead code" → axon-graph but low confidence
Root cause: the dispatch index is name + `# desc:` line ONLY, and axon-graph's desc
lacks the words users actually say (calls/callers, dead code unhyphenated, map,
connected, hubs).

## Change
`workspace/programs/axon-graph.md`:
- `# desc:` rewritten keyword-rich but readable: covers "what calls X", callers,
  blast radius, dead code, most-connected hubs, refactor risk, subsystem map, Obsidian.
- synapse: `next-suggests: [axon-audit]`; invocation_source gains `program`.

`workspace/programs/code-dev-review.md` + `code-dev-plan.md`:
- One anticipation line at phase entry: suggest `axon-graph affected <symbol>` (review:
  caller cone before changing) / `axon-graph cluster` (plan: subsystem grouping).
  Smallest honest version of "the right tool at the right moment" — phase-entry
  suggestion text, no routing changes.

Pattern doc: short section in `workspace/AXON-DOCS-ARCHITECTURE.md` — "tool
discoverability pattern": desc-line carries user phrasings (the dispatch index is
desc-only); next-suggests carries the phase-flow; anticipation lines at phase entry.
Generalizable to any lost tool; graph tools are the pilot.

Index rebuilt (`dispatch index`). Tests: the four baseline phrasings rank axon-graph
FIRST (tests/test_dispatch_graph_routing.py — pins the discoverability contract).
