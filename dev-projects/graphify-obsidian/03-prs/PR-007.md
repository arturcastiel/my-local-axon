# PR-007 — P-CD: s0 graphify-map study step + shadow node-id cache

**Project:** graphify-obsidian · **Phase:** 4-pcd · **Complexity:** M

**Created:** 2026-06-11 · **Depends on:** PR-003 (merged 7b507c1)

## Problem
The P-CD spine — "build the target-repo graph ONCE at study, persist per-project, reuse
through every phase" — is designed but unbuilt. Today code-dev-study reads files with no
graph; nothing persists `my-axon/dev-projects/<slug>/graph/graph.json`; shadows carry no
node-ids for later graph queries. Also: `graphify_bridge.build(repo, out)` accepts `out`
and silently ignores it (latent defect — exposed by wiring this).

## Change
`tools/graphify_bridge.py`:
- `build` honors `out`: copies the produced `graphify-out/graph.json` to `out`
  (parents created). Result keeps `graph` = final location.
- New subcommand `file-nodes --graph <json> --file <rel-path>` — node ids whose
  `source_file` matches; the bridge owns node-id resolution (design rule).

`workspace/programs/code-dev-study.md` — s0 graphify-map block after LOAD CONTEXT:
- IF codebase set AND graphify available (`TOOL(graphify-bridge, check)`) →
  `build --repo {codebase} --out {project-dir}/graph/graph.json` (graphify update is
  incremental — re-runs cheap on resume). Narrated state line (R_STATE_SURFACED).
- Fail-degrade: graphify absent / build degraded → one info line, study proceeds as today.
- File-ingest loop: when the graph exists, `file-nodes` for the studied file → appended
  to the shadow as a `graph-node-ids` section (zero-token graph queries later).

`workspace/domains/code-dev/workflows/code-dev.canonical.yml`:
- DEVIATION from the original design (correctness): a separate s0 node naming
  code-dev-study would execute the study program TWICE in a fixed run. S0 lives
  INSIDE code-dev-study; the yml documents it as a note on s1. DAG unchanged,
  multiple-code-dev inherits the behavior for free.

Tests: build-honors-out (present-path, skipif), file-nodes resolution over the fixture
graph, workflow yml has s0→s1 chain, study program contains the s0 block + degrade path.
