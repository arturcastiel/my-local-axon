## Problem
The adaptive orchestrator (`synapse_suggest.rank`) cannot prefer graph-backed steps:
chat got discoverability in PR-006, but the workflow ranker has no signal that a
per-project graph exists and that a candidate consumes it. The P-CD design named this
`graphify_bonus`, mirroring the existing `shadow_bonus`.

## Change
`tools/synapse_suggest.py` (exact shadow_bonus shape):
- `graphify_bonus(state, candidate)` → +1 iff the graph is available AND the candidate
  consumes it: `state["graph-available"]` (explicit) or derived from
  `state["graph-path"]` existence; candidate opts in via `consumes-graph: true` or
  membership in the known consumer set (axon-graph, code-dev-test-map,
  code-dev-review-scope, code-dev-knowledge-impact).
- `DEFAULT_WEIGHTS["graphify"] = 0.10`, appended to ADDITIVE_KEYS, wired into the
  signals block.

`workspace/tools/synapse-suggest.md`: scoring formula line for the new signal.

Tests (mirroring the shadow_bonus tests' style): bonus fires only with graph present ×
consumer candidate; explicit flag and path-derivation both work; weights/additive-keys
registered.

## Out of scope
Orchestrator state assembly changes beyond the path-derivation fallback.

## Gates
R13 tests · lint-paths · freshness · crucible green.

## Acceptance
- With a graph present, a graph-consumer candidate ranks above an otherwise-equal
  non-consumer; without a graph, ranking is unchanged.
