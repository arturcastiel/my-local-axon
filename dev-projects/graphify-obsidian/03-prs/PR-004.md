# PR-004 — Persist the code map + freshness reconciler

**Project:** graphify-obsidian · **Phase:** 2-living-map · **Complexity:** M
**Status:** merged (squash 4e8017f, main, 2026-06-11) · **Branch:** graphify-obsidian/pr-4-living-map
**Created:** 2026-06-11 · **Depends on:** PR-001 (merged 2b2596b)

## Problem
The Obsidian "map of AXON" exists only as a transient export: `workspace/_dashboards/
axon-code-map.md` is absent from the repo, nothing regenerates it, and the weekly
freshness cron knows nothing about it. The user-facing artifact of the whole
integration is, in practice, not there.

## Change
`tools/code_graph.py`:
- `DEFAULT_MAP_PATH` constant (workspace/_dashboards/axon-code-map.md) — the single
  sourced location for the canonical map.
- `export --canonical` flag (literal add_argument) → writes to DEFAULT_MAP_PATH.
  `--out`/stdout behavior unchanged.

`tools/freshness.py` (composition-only, follows the existing reconciler pattern):
- check `code_map`: regenerate to a temp file (`export --out <tmp>`), byte-compare with
  DEFAULT_MAP_PATH (the export is deterministic — no volatile-line stripping needed);
  absent map = stale.
- refresh step `code_map`: `code_graph.py export --canonical` (auto-heal; the weekly
  `freshness refresh` cron now keeps the map alive with zero manual steps).

Artifact: generate and COMMIT `workspace/_dashboards/axon-code-map.md` (the map itself).

Tests: code_map reconciler (fresh after refresh; stale on mutation; restored via
try/finally), `--canonical` resolution, map regeneration determinism.

## Out of scope
Projection richness — wikilinks, per-community pages (PR-005). Program render
changes beyond the export path (axon-graph export already targets the canonical path).

## Gates
R13 tests · lint-paths · freshness (now including code_map) · crucible green.

## Acceptance
- `workspace/_dashboards/axon-code-map.md` exists on main, opens in Obsidian.
- `freshness check` reports `code_map: true`; editing the map flips it false;
  `freshness refresh` heals it.
