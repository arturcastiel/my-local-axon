# PR-001 — Make dead-code REGISTRY-aware (kill false positives)

**Project:** graphify-obsidian · **Phase:** 1-checkup · **Complexity:** M
**Status:** merged (squash 2b2596b, main, 2026-06-11) · **Branch:** graphify-obsidian/pr-1-dead-code-registry
**Created:** 2026-06-11 · **Depends on:** none

## Problem
`code_graph.py dead_code()` reports 195 candidates over `tools/` — overwhelmingly false
positives. Root cause: the graph only sees intra-tree Python `imports`/`calls` edges, but
AXON's real call topology runs through layers the ast cannot see:
1. **REGISTRY dispatch** — `axon.py <tool>` invokes registered scripts via subprocess;
   every registered tool's CLI surface is live by construction.
2. **Cross-tool subprocess calls** — `subprocess.run([sys.executable, ".../x.py", ...])`
   (e.g. graphify_bridge → aegis_policy) creates no ast call edge.
3. **Library modules** (`_axon_lib`, `_axon_paths`, `_axon_registry`) consumed by axon.py
   (repo root — outside the default graph root) look caller-less.

## Change
`tools/code_graph.py`:
- New helper `_registry_entry_modules(root)` — parse `tools/REGISTRY.json` (stdlib json),
  map registered `script` paths under the graph root to module ids. Fail-soft: missing/
  unparseable registry ⇒ empty set (behavior degrades to today's).
- New helper `_subprocess_edges(...)` — deterministic regex scan per module source for
  `<name>.py` references inside the file (sys.executable / script-path invocations);
  emits `invokes` edges tagged **INFERRED** (regex provenance, mirroring code_symbols).
  EXTRACTED-only consumers are unaffected; `dead_code` may use them as liveness signals.
- Rewrite `dead_code()` as **reachability** from entry roots instead of "no direct
  inbound call edge": roots = `main`/`_build_parser` of every registry-registered module
  (+ those names tree-wide as today, conservative) ∪ modules referenced by `invokes`
  edges; BFS over `contains`/`calls`/`invokes`. Public functions/methods unreachable from
  any root = candidates. Keep output shape `{dead_candidates, count}` + add
  `entry_roots`, `registry_aware: true|false` for transparency.
- **Design decision (measured 2026-06-11):** the graph itself stays 100% EXTRACTED —
  `build()` is untouched (the invariant is load-bearing and pinned by
  `test_real_tools_tree_builds_deterministically`). All new liveness signals live
  INSIDE `dead_code()` only. Measured composition of the 195: 38 test-called,
  large `cmd_*` argparse-handler class (name-referenced, not called), instance-method
  calls unresolvable by ast. Signals: name-references (set_defaults/dispatch dicts),
  attribute-call method names, subprocess `*.py` strings, registry `cmd_*` convention,
  test-caller split.

`tests/test_code_graph.py`:
- Fixture tree with a registered tool (fixture REGISTRY), a helper called only via
  subprocess string, a genuinely dead public function, and a function reachable only
  from a registered main. Assert: dead set contains exactly the genuinely-dead one;
  `invokes` edge tagged INFERRED; missing registry degrades cleanly; byte-determinism
  of build() with the new edges (two builds identical).

Docs: regenerate freshness-gated docs (tool purpose row mentions registry-aware
dead-code); FAILURE-MODES note for the old false-positive mode if a row exists.
