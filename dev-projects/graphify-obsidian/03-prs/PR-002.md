# PR-002 — Repair axon-graph usage doc + full-output UX

**Project:** graphify-obsidian · **Phase:** 1-checkup · **Complexity:** S
**Status:** merged (squash 3e2736d, main, 2026-06-11) · **Branch:** graphify-obsidian/pr-2-axon-graph-ux
**Created:** 2026-06-11 · **Depends on:** none

## Problem
1. `axon-graph` HELP usage line lists `build|affected|dead-code|god-nodes|stats` but the
   program also routes `cluster` and `export` — doc drift inside the neuron.
2. `affected`/`dead-code` renders truncate at 20 items with no path to the full result.
3. The dead-code render predates PR-001: it shows only `count`+`dead_candidates`, hiding
   the new `test_only` / `registry_aware` / `signals` fields.

## Change
`tools/code_graph.py` (small, additive):
- Add optional `--out <json>` to `affected`, `dead-code`, `god-nodes`, `cluster`, `stats`
  (literal `add_argument` per subparser — the static verifier scans literals).
- `main()` writes the JSON result to `--out` when provided (and prints a confirmation
  envelope), else prints to stdout as today.

`workspace/programs/axon-graph.md`:
- Usage line lists all 7 subcommands + the `--out` option.
- `dead-code` render surfaces the PR-001 fields: candidates, test-only count,
  registry-aware flag.
- `affected`/`dead-code` renders: when count > 20, emit a "… and N more — rerun with
  --out <file> for the full list" pointer instead of silently truncating.

Tests (`tests/test_code_graph.py`):
- CLI `--out` writes valid JSON (subprocess run into tmp_path) for dead-code.
- Program contract test extended: usage line mentions every routed subcommand
  (kills this class of doc drift permanently, not just this instance).

## Out of scope
Synapse/dispatch/anticipate wiring (PR-006) · map persistence (PR-004).

## Gates
R13 tests · registry-drift · lint-paths · freshness · crucible green before merge.

## Acceptance
- `explain axon-graph` / HELP shows all 7 subcommands.
- `python3 tools/code_graph.py dead-code --out /tmp/x.json` writes the full result.
- Contract test fails if a routed subcommand is ever missing from the usage line.
