## Summary

`my-axon/MYAXON.md` sets 14 W:myaxon-* path keys using the stale prefix
`/home/arturcastiel/projects/axon-sections/my-axon`. These resolve today via
symlink (`new-axon/axon/my-axon → /home/arturcastiel/projects/axon-sections/my-axon`),
but the symlink is a hidden dependency: remove it and all 14 W: keys silently
point to non-existent directories.

Fix: one search-replace of the base prefix. The symlink itself stays unchanged
(other repos and tools may depend on it). All 14 paths have been verified to
exist under the new canonical form (readlink -f ✓ on each, 2026-06-19).

Plan stated "13 STORE ops" — source read corrects this: 14 path STORE ops
(W:myaxon-path through W:myaxon-generated). W:myaxon-name = "arturcastiel" is
a name, not a path, and is left unchanged.

## Goal Context

From study phase (G5 · ADR-001): my-axon is a symlink → `axon-sections/my-axon`.
Paths work via symlink. Gap is hygiene only — no migration, no data move.
Council R1 initially flagged this as urgent (broken paths); source verification
corrected that to LOW (symlink makes them work). Fix still warranted: symlinks are
infrastructure that can change; W: keys should not rely on one silently existing.

**What:** Replace the stale `axon-sections` base prefix in all 14 path STORE ops.

**How:** Single search-replace (applies to all 14 lines in one pass):


**Why:** Eliminates the hidden symlink dependency. W:myaxon-* keys now resolve
directly without relying on `new-axon/axon/my-axon` symlink staying in place.
W:myaxon-name is unchanged ("arturcastiel" is not a path).

**Implementation note (Edit strategy):** Use a single Edit call with `replace_all=True`
on the base path string (`/home/arturcastiel/projects/axon-sections/my-axon` →
`/home/arturcastiel/projects/new-axon/axon/my-axon`). This replaces all 14 occurrences
in one pass. Do NOT use 14 individual Edit calls — they add round-trip overhead and
one missing call leaves a stale path unnoticed.

**Implementation note (OS: line):** The `OS:` line at line 12 already uses the correct
`/home/arturcastiel/projects/new-axon/axon` form — no change needed there.
Only the `axon-sections/my-axon` prefix in the PATH MAP block (lines 23-36)
requires updating.

## Architecture Impact

Zero runtime change. The symlink makes both path forms resolve to the same
physical directory. Programs reading W:myaxon-* keys will get different string
values but identical filesystem targets.

The change is significant for resilience: if the symlink is ever removed or
retargeted, code-dev and all memory-writing programs will continue to work
correctly using the new canonical path.

KERNEL-SLIM line 350-355 documents `W:myaxon-*` scope as "loaded from
`my-axon/MYAXON.md` at boot" — this PR makes that contract consistent with
the actual physical location of the my-axon directory.

## Risks & Gotchas

- ⚠ **W:myaxon-path trailing slash**: Current value has no trailing slash
  (`/home/arturcastiel/projects/axon-sections/my-axon` — no `/` at end).
  New value must also have no trailing slash to preserve consistency with callers
  that may do `W:myaxon-path + "/dev-projects/"`. All other keys have trailing
  slashes — preserve that pattern.
- ⚠ **Grep before edit**: Run `grep -c "axon-sections" my-axon/MYAXON.md` before
  editing to confirm the count is exactly 14. If count ≠ 14:
  - Count < 14: some paths already updated; edit only remaining stale ones
  - Count > 14: extra occurrences outside PATH MAP block; run
    `grep -n "axon-sections" my-axon/MYAXON.md` to find all lines and
    update each one individually before using replace_all
- ⚠ **Trailing slash correctness**: W:myaxon-path has NO trailing slash;
  all 13 other path keys DO. The replace_all on the base string preserves
  this automatically (only the prefix is replaced, not the suffix). Verify
  with: `grep "W:myaxon-path" my-axon/MYAXON.md` — must NOT end with `/`.
- ⚠ **Do not edit the symlink target**: `readlink my-axon` should still show
  `/home/arturcastiel/projects/axon-sections/my-axon` after this PR. The symlink
  is not part of this change.
