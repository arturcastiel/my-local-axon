### workspace/programs/authoring-guide.md — 3 edits

---

**Edit 1 — §5 body: add phase-ledger block after L123**

Insert after: `→ "  Read-only programs (!NORM | read-only): skip phase tracking."`

```
→ ""
→ "  Phase-ledger recording (required when program takes OWNERSHIP):"
→ "  A program that does STORE(W:active-program) holds the context lock."
→ "  Record phase transitions so ops tooling can observe program lifecycle:"
→ ""
→ "    TOOL(phase-ledger, record, '--program {W:active-program} --phase start')"
→ "    ...program body..."
→ "    TOOL(phase-ledger, record, '--program {W:active-program} --phase done')"
→ ""
→ "  Optional — verify phases recorded (post-run check or in tests):"
→ "    TOOL(phase-ledger, verify, '--program X --expected-phases start,done')"
→ ""
→ "  Scope: only programs that explicitly STORE(W:active-program) need this."
→ "  Check §12 Neuron Contract to confirm whether your program takes ownership."
```

**Why:** The §5 section described W:active-phase (kernel interrupt recovery) but
not phase-ledger (ops observability). They serve different purposes:
- W:active-phase → kernel resumes interrupted sessions
- phase-ledger → ops tooling tracks which programs ran and completed

Both are required for ownership-holding programs. §5 should document both.

## Risks & Gotchas

- ⚠ **§5 EXISTS — do not create a new §5**: The plan said "§5 doesn't exist."
  Source read confirms §5 "PHASE TRACKING" is at L96-124. This PR EXTENDS §5,
  does NOT insert a new section between existing sections. Inserting a new §5
  would renumber existing §6-§12 references.
- ⚠ **Phase-ledger is additive, not enforcement**: Adding this content to the
  authoring guide does not retroactively require existing programs to call
  phase-ledger. The requirement is forward-looking (new programs / next edit).
  Enforcement via R_PHASE_TRACKED (warn mode per test_phase_ledger.py L13) is
  separate and pre-existing.
- ⚠ **Line number drift**: §5 additions shift all subsequent line numbers.
  Locate edits 2 and 3 by content anchor, not absolute line number.
- ⚠ **`{W:active-program}` in phase-ledger call**: The TOOL() invocation uses
  `{W:active-program}` to pass the program name. This assumes W:active-program
  is set before the phase-ledger record call. Per KERNEL-SLIM, ownership-holding
  programs STORE(W:active-program) at entry — so the key is in scope when the
  record call fires.
