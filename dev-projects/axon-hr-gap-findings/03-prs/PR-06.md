## Changes Required

### workspace/scheduler/cron.json — add one job

**What:** Append a new job entry for weekly coverage gate check.

**Command to produce the change (do NOT run — human task):**
```bash
python3 tools/cron.py add \
  --program "coverage_gate.py check --min 4.0" \
  --schedule "weekly Wed 09:15" \
  --label "Weekly coverage baseline check (--min 4.0)"
```

**Expected new entry (auto-ID will be CRON-002 or next available):**
```json
{
  "id": "CRON-002",
  "program": "coverage_gate.py check --min 4.0",
  "schedule": "weekly Wed 09:15",
  "priority": "!NORM",
  "label": "Weekly coverage baseline check (--min 4.0)",
  "enabled": true,
  "added": "<timestamp>",
  "last_run": null,
  "next_run": "<next Wednesday 09:15 UTC>"
}
```

**Why:** 4.0% is the established system-wide baseline (from study phase). The
`--min 4.0` flag causes the job to fail if coverage drops below this floor.
Weekly cadence on Wednesday morning gives the team a full day to respond before
the standard Monday reporting cycle.

**Implementation:** Run the `cron.py add` command above (canonical method).
Direct JSON editing is a last resort — manual edits risk formatting errors
(missing comma, trailing comma, wrong field types).

## Risks & Gotchas

- ⚠ **`cron.py add` is NOT idempotent**: Unlike `seed-defaults` (which checks for
  existing IDs before inserting), the `add` subcommand directly appends without
  checking for duplicate programs. Running `cron.py add` twice creates two
  identical entries. Pre-flight: always run
  `grep "coverage_gate" workspace/scheduler/cron.json` before adding to confirm
  count is 0. If count > 0, the job already exists — do not add.
- ⚠ **ID auto-generation**: `cron.py add` assigns the next `CRON-NNN` ID
  automatically via `new_id()`. Expected: `CRON-002`. This may change if another
  job is added between now and when this PR is implemented.
- ⚠ **`coverage_gate.py report` scope removed**: Plan included finding and fixing
  stale `coverage_gate.py report` references. Source search (`grep -r
  "coverage_gate.*report" workspace/`) found zero matches. No doc edits needed.
