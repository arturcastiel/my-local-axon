## Changes Required

### workspace/scheduler/cron.json — add one job

**What:** Append a new job entry for the weekly self-care sweep.

**Command to produce the change (do NOT run — human task):**
```bash
python3 tools/cron.py add \
  --program "self-care" \
  --schedule "weekly Wed 09:00" \
  --label "Weekly AXON self-care sweep"
```

**Expected new entry (auto-ID will be CRON-002 or next available):**
```json
{
  "id": "CRON-002",
  "program": "self-care",
  "schedule": "weekly Wed 09:00",
  "priority": "!NORM",
  "label": "Weekly AXON self-care sweep",
  "enabled": true,
  "added": "<timestamp>",
  "last_run": null,
  "next_run": "<next Wednesday 09:00 UTC>"
}
```

**Note:** If PR-06 is merged first, the next CRON-NNN ID will be CRON-003
(PR-06 takes CRON-002). The actual ID is irrelevant — verify by program name.

**Implementation:** Use `cron.py add` (canonical method). Direct JSON editing
risks formatting errors and should be avoided.

**Why:** Weekly cadence on Wednesday at 09:00 gives self-care 15 minutes ahead
of the coverage-gate check (PR-06 at 09:15). Both share the Wednesday window —
self-care first so a degraded health score is visible before coverage runs.

## Risks & Gotchas

- ⚠ **`cron.py add` is NOT idempotent**: Same issue as PR-06. Pre-flight:
  `grep '"self-care"' workspace/scheduler/cron.json` must return 0 before adding.
  If count > 0, the job already exists — do not add.
- ⚠ **Program name is `"self-care"` (not a path)**: Non-path slug pattern confirmed
  by existing non-path jobs in cron.json: `session-save`, `auto-improve`,
  `axon-audit`, `igap report --days 7`, etc. `"self-care"` matches this pattern.
  Test assertion uses `program == "self-care"` (exact slug match, not path).
- ⚠ **test_cron.py does NOT exist**: Plan said "extend tests/test_cron.py."
  Source verified: no such file. Use `tests/test_cron_conformance.py` (exists).
- ⚠ **ID ordering with PR-06**: If PR-06 and PR-08 are implemented in the same
  session, CRON IDs increment sequentially. The tests should match by `program`
  field (`"self-care"`), not by `id` field (`CRON-NNN`), to be order-independent.
- ⚠ **Wednesday 09:00 vs 09:15 ordering**: `cron.py tick` is rate-limited to
  **at most 1 auto-run per tick** (confirmed: L391 of cron.py, FA-13 fix).
  self-care at 09:00 and coverage-gate at 09:15 are 15 minutes apart — each
  fires in its own tick cycle. A self-care failure at 09:00 does NOT block
  coverage-gate at 09:15 (independent ticks, L370: failed job advances next_run
  regardless of outcome).
