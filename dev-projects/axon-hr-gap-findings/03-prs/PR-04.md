## Summary

The igap daily log (`workspace/log/igap/YYYY-MM-DD.md`) is only written by the
`!BG` behavioral tracker in KERNEL-SLIM, which fires on 4 runtime conditions
(low-confidence, semantic-search, fallback-exec, absent-instruction). During a
normal code-dev session that completes successfully, zero igap entries are
produced. `igap stats` is always empty unless the session hit a behavioral gap.

Fix: wire `TOOL(igap, record, ...)` calls at key lifecycle sites in 3 code-dev
programs: before meaningful HALT/FAIL stops + at every DONE(). Add `session-close`
as a 5th type to `tools/igap.py` for the DONE() heartbeats.

Also fixes: `code-dev-meta-igap.md` L75 phantom reference (`code-dev-dispatch.md`
does not exist — planning artifact that was never created).

### 1. tools/igap.py — add session-close type

**Change 1a — L70 (VALID_TYPES):**
```python
# Before:
VALID_TYPES = {"low-confidence", "semantic-search", "fallback-exec", "absent-instruction"}

# After:
VALID_TYPES = {"low-confidence", "semantic-search", "fallback-exec", "absent-instruction", "session-close"}
```

**Change 1b — L211 (type_order in cmd_report):**
```python
# Before:
type_order = ["absent-instruction", "low-confidence", "semantic-search", "fallback-exec"]

# After:
type_order = ["absent-instruction", "low-confidence", "semantic-search", "fallback-exec", "session-close"]
```

**Why:** `session-close` records at DONE() sites make every code-dev run produce
≥1 igap entry. Without this type, `--missing "" --suggestion ""` would be rejected
by cmd_record's type-validation check at L76. The type is appended last in
type_order so it appears at the bottom of `igap report` output, after genuine gaps.

## Risks & Gotchas

- ⚠ **igap TOOL call format in AXON-LANG**: `TOOL(igap, record, "...")` passes the
  flags as a single string. If the igap CLI parser tokenises on spaces, quoted
  strings with commas in `--missing` values (e.g. `{JOIN(...)}`) may need escaping.
  Test the TOOL call manually on first run; if tokenisation fails, simplify the
  `--missing` string to remove commas.
- ⚠ **L113 is FAIL not HALT**: Do not confuse with earlier program structure.
  The igap insertion goes INSIDE the `IF result.ok ≡ false →` block, before FAIL.
- ⚠ **CLEAR before DONE ordering**: Sites 2c, 3c, 4a: CLEAR(W:active-program)
  is the last op before DONE. Keep order: CLEAR → TOOL(igap) → DONE.
  The igap record should be the final op before DONE so it always fires.
- ⚠ **code-dev-state-handoff.md has NO HALT to wire**: Source confirmed.
  The identity assertion HALT at L30 is behavioral (kernel-level), not a lifecycle
  stopping point relevant to code-dev observability. Do not wire igap there.
- ⚠ **AXON-LANG {variable} interpolation in TOOL() flags**: AXON-LANG supports
  `{var}` interpolation inside TOOL() argument strings (confirmed by BOOT.md L188
  pattern: `"--input {RETRIEVE(W:recent-input)...}"`). Program-local vars like
  `{filtered-count}` should interpolate correctly at the TOOL() call site. If the
  executor does NOT expand the variable, the --missing field becomes a literal
  template string — not a bug (igap still records it), but less informative.
  On first run, verify the igap log shows the actual count, not `{filtered-count}`.
- ⚠ **session-close in igap report**: `igap report` produces "improvement
  suggestions" grouped by type. For session-close type, `--suggestion "none"` means
  the report shows one "none" suggestion under session-close — not ideal but benign.
  A future PR could suppress the suggestion block for session-close entries in
  cmd_report. For now, "none" in the suggestions column is acceptable.
- ⚠ **Insertion line numbers shift**: Absolute line numbers (L303, L454, L472, etc.)
  are pre-insertion anchors. After each TOOL(igap, ...) block is inserted, subsequent
  line numbers shift. Locate each site by CONTENT ANCHOR (the HALT/DONE keyword and
  surrounding text), not by absolute line number.
- ⚠ **`igap.py clear` (not `session --reset`)**: Pre-merge step uses `igap.py clear`
  subcommand. `session --reset` does NOT exist as a CLI command. See Pre-merge step.
- ⚠ **session-close semantics**: igap is designed for GAPs, not lifecycle events.
  The session-close type is a pragmatic extension for observability. The `--missing`
  field should say "none (session completed)" and `--suggestion` "none" to be
  honest in the log. Do not repurpose the gap fields for other content.
