# PR-001 — Wire the full pytest suite into CI
Project:     axon-tests
Created:     2026-05-16
Complexity:  S
Depends on:  none
Status:      not-started
AXON score:  8/10

## Summary
Replace the path-only `test` job in `.github/workflows/ci.yml` with a
full `pytest tests/` run. Today CI runs ~10 cases (TestAxonPaths +
TestLintPaths); this PR makes CI execute all 315 (and any tests added
by later PRs). This is the single highest-leverage change in the
project — every later PR depends on a working full-suite CI.

## Goal Context
From 01-study.md F-1: "CI enforces ~3 % of the test surface". Without
this PR, every test added by PR-006..PR-017 is invisible to CI.

## Entry Conditions
- `tests/` is currently green locally (verified by `pytest tests/`).
- `requirements.txt` lists `pytest` (or it's added in this PR).
- `.github/workflows/ci.yml` exists and is the only CI surface.

## Changes Required

### .github/workflows/ci.yml
- **What:** Replace the `test` job's two pinned-class invocations
  with a full-suite run.
- **How:** New job body:
  ```yaml
  - uses: actions/checkout@v4
  - uses: actions/setup-python@v5
    with:
      python-version: "3.12"
      cache: pip
  - name: Install deps
    run: |
      python3 -m pip install --upgrade pip
      python3 -m pip install -r requirements.txt
      python3 -m pip install pytest
  - name: Run full pytest suite
    run: python3 -m pytest tests/ -v --maxfail=1 --durations=20
  ```
  Rename the job to `tests-full` for clarity. Keep `lint-paths` job
  unchanged.
- **Why:** Fail fast on first regression; surface the 20 slowest cases
  so PR-002 (coverage) and PR-019 (smoke pre-push) can target them.

### requirements.txt
- **What:** Ensure `pytest>=8.0` is listed (currently the CI installs
  it inline; we make it explicit in the file).
- **How:** Append `pytest>=8.0` if not present.
- **Why:** One source of truth; local devs match CI.

### CHANGELOG.md
- **What:** Add an entry: "CI: full pytest suite now runs on every
  push/PR (was: path tests only)."
- **How:** Append under the next-release "## Unreleased" heading
  (create that section if absent).
- **Why:** Visibility — this is a contract change for contributors.

## Architecture Impact
None at the code level. CI matrix grows: previous job took ~30 s,
the new job is expected to take ~3–5 min on the 315-case suite.
Cached pip keeps it predictable. No `axon/` writes, no kernel impact.

## Acceptance Criteria
- ✓ Pushing a branch with a deliberate one-line failure in
  `tests/test_tools_core.py` causes CI to fail with `--maxfail=1`
  fast and a clean diff in the GitHub annotation.
- ✓ Pushing the same branch reverted causes CI to pass.
- ✓ `--durations=20` output is visible in the action log.
- ✓ `lint-paths` job continues to run independently.
- ✓ Job is named `tests-full` and is required for merge in branch
  protection (config out-of-scope; documented in PR-020).

## Risks & Gotchas
- ⚠ Some existing tests may rely on `tests/.venv` (see `conftest.py`
  fallback to `sys.executable`). On CI there is no `.venv`; the
  fallback is correct but should be verified by running the suite in
  a clean container locally before merge.
- ⚠ `test_tools_kernel.py` (176 cases) is the heaviest file; the
  `--durations=20` flag will reveal whether it dominates runtime.
- ⚠ The mock-model harness (`tests/_mock_model.py`) must not require
  network or model API keys. Confirm by reading it before merge.
- ⚠ `pytest-cov` is NOT introduced here — that's PR-002. Avoid
  premature scope creep.
