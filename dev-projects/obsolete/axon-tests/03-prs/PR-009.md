## Summary
R9 (no writes to `axon/` without `L:dev-mode ≡ true`) is the most-
quoted Core Rule and has zero dedicated tests. This PR turns the
PR-006 stub into a real file with positive + negative + 8 edge cases.

## Goal Context
Study finding F-4 (rules under-tested) + the kernel rule itself.
Co-output: AXON-DOCS-GOVERNANCE § "Rule R9 — axon/ write gate" gets
its Guarded-by block from this file.

## Entry Conditions
- PR-006 merged (rules scaffold + meta-enforcer in place).
- `tools/rules/r9.py` reachable; `rule_id` registered.

## Changes Required

### tests/test_rules/test_r9_axon_write.py
- **What:** Replace the PR-006 stub with the full class
  `TestR9AxonWrite` covering:
  1. positive: WRITE to `axon/foo.md` with `dev-mode=false` → violation
  2. positive: APPEND to `axon/x` with `dev-mode=false` → violation
  3. negative: WRITE to `axon/foo.md` with `dev-mode=true` → allowed
  4. negative: READ of `axon/foo.md` → always allowed
  5. edge: path normalisation `./axon/x` and `axon//x` both blocked
  6. edge: symlink `link → axon/x` blocked (resolves through link)
  7. edge: relative path from `axon/programs/` to `axon/HOWTO.md`
     still blocked
  8. edge: user-explicit "please write to axon/x" still blocked
     (no-queue rule)
  9. edge: program-initiated WRITE to `axon/` blocked even with
     dev-mode (programs may never WRITE to axon/, kernel rule)
  10. edge: writing to `my-axon/`, `workspace/`, `tests/` always
      permitted (asserts gate is narrow).
- **How:** Use the rule's `evaluate(ctx)` interface; build minimal
  ctx dicts inline.
- **Why:** Locks the most safety-critical predicate.

### workspace/AXON-DOCS-GOVERNANCE.md
- **What:** Add `## Rule R9 — axon/ write gate` section with
  `## Guarded by` block.
- **How:** Per PR-004 template; cite each `test_*` id.
