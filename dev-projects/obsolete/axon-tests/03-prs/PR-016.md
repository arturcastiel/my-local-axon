## Summary
Populate behavioural fixtures for the 10 tier-A programs (menu,
identity, code-dev root + new/load/study/plan/pr/resume, workspace-
backup) using the existing mock-model harness. The harness is
parametrised; one fixture per program is enough.

## Goal Context
F-5 + F-11: ~30 programs, but only a fraction exercised in tests. We
draw the line at tier-A (= load-bearing, frequent paths) for this
project; tier-B/C tracked separately.

## Entry Conditions
- PR-005 merged (so PR-013/014 patterns are established).
- Mock-model harness is the existing one in `tests/_mock_model.py`.

## Changes Required

### tests/fixtures/programs/<name>/  (10 new dirs)
- **What:** Per program: `responses.jsonl` (mock-model script) +
  `expected.md` (golden rendered output) + `_harness.yaml` (kernel
  keys to set, if any).
- **List:** menu, identity, code-dev, code-dev-new, code-dev-load,
  code-dev-study, code-dev-plan, code-dev-pr, code-dev-resume,
  workspace-backup.

### tests/test_programs_tier_a.py  (new)
- **What:** Parametrised test that loads every `fixtures/programs/*`
  dir, runs the mock-model harness, diffs actual vs golden.
- **How:** A directory missing `expected.md` → test fails (no
  silent skip).

### workspace/AXON-DOCS-TESTING.md § T3
- **What:** Extend the T3 section with the tier-A list + Guarded-by
  per program.

## Architecture Impact
None to programs; tests are read-only.

## Acceptance Criteria
- ✓ `pytest tests/test_programs_tier_a.py -v` shows 10 passing
  parameter cases.
- ✓ Mutating any tier-A program's deterministic output causes its
  case to fail with a clean diff.
- ✓ Documentation lists all 10 programs as Guarded-by.

## Risks & Gotchas
- ⚠ Golden outputs include timestamps/UUIDs — strip non-deterministic
  fields before diffing (use a normaliser in the harness).
- ⚠ `code-dev-resume` depends on a prior project state — fixture
  must seed it.
- ⚠ Identity program overlaps with PR-007; this fixture asserts the
  *rendered menu* path, not the gate.
