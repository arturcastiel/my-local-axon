# PR-019 — Mandatory smoke pre-push hook
Project:     axon-tests
Created:     2026-05-16
Complexity:  S
Depends on:  PR-001, PR-003

AXON score:  8/10

## Summary
Extend the PR-003 hook with a fast smoke-test pass (`pytest
tests/test_smoke.py`, ≤ 5 s) so any push runs at least a contract
check locally before hitting CI. Adds a new `test_smoke.py`.

## Goal Context
Defence-in-depth for the contributor experience: catch the obvious
breaks (identity gate broken, boot crashes, no rules registered)
before the round-trip through GitHub Actions.

## Entry Conditions
- PR-001 merged (full pytest in CI).
- PR-003 merged (`scripts/install-hooks.sh` installs the pre-push).

## Changes Required

### tests/test_smoke.py  (new)
- **What:** ~5 fast cases, total runtime ≤ 5 s:
  · identity program file exists and is non-empty,
  · `python3 axon.py boot --dry-run` returns 0,
  · `tools/REGISTRY.json` parses and lists ≥ N tools,
  · `tools/verify.py --list-rules` returns ≥ 9 rules,
  · `python3 axon.py prefs` returns 0.
- **Why:** Each is a < 1-s sanity check on the most break-prone
  surfaces.

### scripts/install-hooks.sh
- **What:** Extend the `.githooks/pre-push` body to call:
  ```sh
  python3 -m pytest tests/test_smoke.py -q --maxfail=1 || exit 1
  python3 tools/scan_pre_push.py || exit 1
  ```

### .githooks/pre-push
- **What:** The installed hook now runs smoke + scan in order.

### workspace/AXON-DOCS-GOVERNANCE.md
- **What:** Extend `## Pre-push gates` with the smoke section +
  Guarded-by row.

## Architecture Impact
Adds a ≤ 5 s tax on every push (locally). Acceptable.

## Acceptance Criteria
- ✓ `pytest tests/test_smoke.py -v` runs in < 5 s on the dev
  machine spec used in CI.
- ✓ Pushing with a deliberately-broken identity file fails the hook
  with a clear message.
- ✓ Pushing with all smokes green passes through to `scan_pre_push`.

## Risks & Gotchas
- ⚠ Hook must call `python3` via the venv if one is active; respect
  `core.hooksPath`.
- ⚠ Some users disable hooks; document `git push --no-verify` as
  an explicit opt-out — but mandatory PR template (PR-020) requires
  the author to attest tests ran.
- ⚠ Keep `test_smoke.py` ruthlessly short — if it grows past 5 s,
  split: smoke stays in pre-push, slow ones move to CI only.
