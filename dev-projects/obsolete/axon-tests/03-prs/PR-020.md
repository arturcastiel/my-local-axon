# PR-020 — CONTRIBUTING + AGENTS + PR-template: tests are mandatory

AXON score:  8/10

## Summary
Codify the "tests + docs are mandatory for any `axon/` change" rule
in plain English. Updates `CONTRIBUTING.md`, `AGENTS.md`, and adds
`.github/PULL_REQUEST_TEMPLATE.md`. PR template forces author to
tick: (a) tests added, (b) coverage not regressed, (c) doc page
updated with Guarded-by.

## Goal Context
The technical gates (CI, hooks, audit) are useless if contributors
don't know about them. This PR is the human-facing half.

## Entry Conditions
- PR-018 merged (the gate is real and blocking).

## Changes Required

### CONTRIBUTING.md
- **What:** Add a "Test & doc requirements" section:
  · Every change to `axon/` requires ≥1 new or updated test.
  · Every new program/rule/tool requires a Guarded-by doc row.
  · CI gates: `tests-full`, `coverage`, `docgen-strict`,
    `axon-audit`. All required.
  · Pre-push hook installation (`scripts/install-hooks.sh`).
- **How:** Insert near the top; link out to AXON-DOCS-TESTING and
  AXON-DOCS-GOVERNANCE.

### AGENTS.md
- **What:** Append a "Tests are mandatory" clause to the agent
  contract — agents writing to `axon/` MUST add tests in the same
  PR and update the relevant AXON-DOCS-* page.
- **Why:** AGENTS.md is the prompt contract for non-Copilot agents;
  must mirror CONTRIBUTING.

### .github/PULL_REQUEST_TEMPLATE.md  (new)
- **What:** Markdown template with the three required tick-boxes
  and a required "AXON-DOCS page updated:" line.

### README.md
- **What:** One-paragraph mention of the mandatory test+doc rule
  near top, pointing at CONTRIBUTING.
