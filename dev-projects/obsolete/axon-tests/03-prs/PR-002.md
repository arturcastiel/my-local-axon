# PR-002 — Add pytest-cov with coverage gates
Project:     axon-tests
Created:     2026-05-16
Complexity:  S
Depends on:  PR-001
Status:      not-started
AXON score:  8/10

## Summary
Configure `pytest-cov` and fail CI when coverage drops below
**100 % line+branch on `tools/rules/`** or **80 % line on `tools/`**.
Other paths are measured but advisory. Coverage artifact is uploaded
to the run for inspection. A meta-test guards the config itself so
the gate can't be silently weakened.

## Goal Context
From 01-study.md F-3: "No coverage measurement of AXON itself."
The plan's success metric requires per-rule coverage to be machine-
verified.

## Entry Conditions
- PR-001 merged: CI runs the full pytest suite.
- `tools/rules/` is the canonical home of rule predicates (no
  splinter modules elsewhere).

## Changes Required

### pyproject.toml  (or pytest.ini if pyproject absent)
- **What:** Add `[tool.pytest.ini_options]` with `addopts` enabling
  `--cov` and add `[tool.coverage.run]` + `[tool.coverage.report]`
  sections.
- **How:**
  ```toml
  [tool.pytest.ini_options]
  addopts = "--cov=tools --cov-report=term-missing --cov-report=xml --cov-branch"

  [tool.coverage.run]
  branch = true
  source = ["tools"]
  omit = ["tools/__pycache__/*", "tools/rules/__pycache__/*"]

  [tool.coverage.report]
  fail_under = 80
  precision = 1
  show_missing = true
  ```
- **Why:** One config; works locally and in CI; XML for Codecov-style
  artifact uploads later.

### tools/coverage_gate.py  (new, small)
- **What:** Tiny script that parses `coverage.xml` and enforces the
  per-package thresholds (100 % rules / 80 % tools); exits non-zero
  on breach with a per-file delta table.
- **How:** ~60 LOC; std-lib only (`xml.etree.ElementTree`). Reads
  the XML, groups by `filename` prefix, prints a table, exits 1 on
  any breach. Registered in `tools/REGISTRY.json` as `coverage-gate`.
- **Why:** `pytest --cov-fail-under` is single-threshold; we need
  per-package thresholds.

### .github/workflows/ci.yml
- **What:** After the pytest job, add a `coverage-gate` step.
- **How:**
  ```yaml
  - name: Install coverage deps
    run: python3 -m pip install pytest-cov
  # (in the pytest step) addopts already pick up --cov from pyproject
  - name: Coverage gate (rules 100 / tools 80)
    run: python3 tools/coverage_gate.py coverage.xml
  - name: Upload coverage artifact
    if: always()
    uses: actions/upload-artifact@v4
    with:
      name: coverage-${{ github.sha }}
      path: coverage.xml
  ```
- **Why:** Make the gate part of the required CI status.

### requirements.txt
- **What:** Add `pytest-cov>=4.1`.

### tests/test_coverage_config.py  (new)
- **What:** Meta-test asserting the config is intact.
- **How:** 3 cases:
  · `tool.pytest.ini_options.addopts` contains `--cov` and
    `--cov-branch`,
  · `tool.coverage.run.branch is True`,
  · `coverage_gate.py` exists, is executable, and registered as a
    tool in REGISTRY.json.
- **Why:** Without this, a future PR could remove the gate silently.

### workspace/AXON-DOCS-TESTING.md
- **What:** Add a new "## Coverage" section.
- **How:** Documents the 100 / 80 split, how to read the artifact,
  how to add an exclusion (and the policy that exclusions for
  `tools/rules/` are not allowed).
- **Why:** Doc co-output rule (PR-004 will enforce it; we comply
  proactively).

## Architecture Impact
Adds two new dev deps (`pytest-cov`, `coverage`). Adds one new tool
(`coverage_gate`). No runtime impact, no kernel impact.

## Acceptance Criteria
- ✓ `pytest tests/` locally produces a `coverage.xml`.
- ✓ Deleting one line from `tools/rules/r9_axon_write.py` (without
  updating tests) causes `coverage_gate.py` to exit 1 with that file
  listed.
- ✓ Removing the `--cov` flag from pyproject causes
  `tests/test_coverage_config.py` to fail.
- ✓ Run artifact appears in the GitHub Actions UI under
  `coverage-<sha>`.
- ✓ Per-package thresholds applied: `tools/rules/` ≥ 100 % branch +
  line; `tools/` (excluding rules) ≥ 80 % line.

## Risks & Gotchas
- ⚠ The 100 %-rules threshold may fail on day one — rule modules
  are nearly empty, but some defensive branches will be uncovered
  until PR-009/011 land. **This PR sets the gate but `fail_under`
  is set to 80 globally; the 100 % rules threshold is enforced by
  `coverage_gate.py`, which we add WITH an initial allowlist** so
  CI stays green. PR-011 removes the allowlist.
- ⚠ Branch coverage can be sensitive to inlined imports; the omit
  list excludes `__pycache__` only — verify no other noise.
- ⚠ Don't include `tests/` in the coverage source — that's standard
  but worth a comment in pyproject.
