# pr-18 — Dispatch corpus (seed 30)

**Wave**: W3 · **Goals**: G.test.02 (seed) · **Depends-on**: PR-14 (routers in place)

## Why (problem statement)
F-D1 (free-text routed to wrong verb) cannot be measured today — there is no ground-truth dispatch corpus. R6 marks `dispatch quality measurement` as U-12; the prereq is a labeled corpus of prompts → expected programs.

## Evidence (from studies)
- `helpers/cd-gap-c2-p3-test-surface.md` → "Tier 2: dispatch corpus — labeled prompts with expected program".
- `helpers/cd-gap-c2-p4-failure-modes.md` → F-D1 mid-priority (#7 in mitigation top-10).
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.test.02.

## Design notes
- `tests/fixtures/dispatch-corpus.jsonl`: one line per case:
  ```json
  {"prompt":"list all PRs ready for review","expected":"code-dev-pr-list","args":["--state=ready-for-review"]}
  ```
- Seed with **30 cases** covering the top-10 verbs (3 phrasings each). Selected from `_actions.log` history + R4 cookbook recipes.
- `tests/test_dispatch.py`:
  - Loads corpus.
  - Invokes dispatcher in mock-model mode (no real model calls).
  - Computes P@1 (top pick correct) and P@3 (correct in top-3).
  - Emits per-case results to `my-axon/log/dispatch-metrics/<date>.json` (consumed by PR-19).
- Cases are read-only; no side effects.

## Pitfalls (from failure-mode catalog)
- **F-D1 mis-dispatch** → measured by this PR.
- Corpus drift (verb renames) → linked to rename-safety snapshot (PR-12); corpus regenerates when verbs change.

## Interface sketch
```text
$ pytest tests/test_dispatch.py -v
PASSED tests/test_dispatch.py::test_corpus_loaded  (30 cases)
PASSED tests/test_dispatch.py::test_p1_at_least_0_7  (P@1 = 0.73 — baseline)
PASSED tests/test_dispatch.py::test_p3_at_least_0_9  (P@3 = 0.93)
```
