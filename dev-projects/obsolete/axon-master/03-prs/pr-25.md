# pr-25 — Idempotence harness

**Wave**: W3 · **Goals**: G.test.04, NS-2 · **Depends-on**: PR-1 (T1), PR-18 (test infra)

## Why (problem statement)
A code-dev program should be **idempotent** — running it twice with the same inputs should produce structurally-overlapping outputs (≥ 80%). Today this is asserted nowhere; subtle non-determinism (timestamps in output, randomized ordering) can silently break compaction-recovery (PR-15) and replay flows. R5 NS-2 calls this the second-priority next-study.

## Design notes
- `tools/idem_test.py`:
  - Runs `code-dev <program>` twice on a fixture project (copy-to-tmp).
  - Diffs the two outputs (file by file) with whitespace-normalization.
  - Computes structural overlap: `(common-sections / total-sections) × 100`.
  - Threshold: ≥ 80% (advisory in W3, gate in W4).
  - Emits per-program `_trace/<prog>-idem.json` with metrics + diff summary.
- `tests/test_idempotence.py`: pytest entry, runs against ≥ 3 programs (`status`, `next`, `pr-list`).
- Allow-list: timestamps and version strings excluded from diff via regex.

## Pitfalls
- True non-determinism (LLM output) → measure structural overlap (sections), not exact bytes.
- Side effects (mutations to `_actions.log`) → run on tmp copy.

## Interface sketch
```text
$ pytest tests/test_idempotence.py -v
PASSED test_status_idempotent     (overlap 94%)
PASSED test_next_idempotent       (overlap 88%)
PASSED test_pr_list_idempotent    (overlap 100%)
```

## Codebase grounding
- **new**: `tools/idem_test.py` — `run_twice(program, fixture_path) -> OverlapReport`. Copies fixture to `tempfile.mkdtemp()` (twice), runs program in mock mode, diffs outputs file-by-file with whitespace normalization. Allow-list regex: ISO timestamps, version strings.
- **new**: `tests/test_idempotence.py` — parametrized over 3 programs initially: `status`, `next`, `pr-list`.
- **per-program output**: `_trace/<program>-idem.json` with `{overlap_pct, diffs:[...], threshold:80}`.
- **threshold**: ≥ 80% structural overlap (advisory W3; gate W4 via flip).
- **fixtures**: reuse fixtures from PR-3 (`tests/fixtures/projects/v1-minimal/`) where applicable.
