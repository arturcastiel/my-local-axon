# pr-20.6 — `meta board` ASCII Kanban

**Wave**: W3 · **Goals**: G-I8 (R4) · **Depends-on**: PR-9.5 (`pr list` aggregator)

## Why (problem statement)
R4 G-I8 ranks "Kanban board view of PRs" as a top-15 industrial gap — engineers expect a board view, but AXON has only flat text. The aggregator (PR-9.5) provides the data; this PR provides the visualization.

## Evidence (from studies)
- `helpers/cd-wf-c2-p1-industrial-gaps.md` → G-I8.
- `helpers/cd-wf-c4-p2-roadmap.md` Wave 2 PR-2.2 → "ASCII Kanban".
- `helpers/cd-wf-c1-p4-web-findings.md` → Graphite/Aviator both offer board view as primary UI.

## Design notes
- `tools/board.py`:
  - Reads `pr_aggregate.list(project)` (PR-9.5).
  - Groups by state: `backlog | in-progress | blocked | ready-for-review | done`.
  - Renders 5-column ASCII table, ≤ 200 cols wide.
  - `--project <slug>` scopes; default all (separator row per project).
- `workspace/programs/code-dev-meta-board.md`: invokes tool and prints.

## Pitfalls
- Wide terminals → respects 200-col cap; long slugs truncate with `…`.
- Empty columns → still rendered (empty cell), so the structure is stable.

## Interface sketch
```text
$ code-dev meta board
project: axon-master
| backlog       | in-progress  | blocked | ready-for-review | done    |
|---------------|--------------|---------|------------------|---------|
| pr-4          | pr-3         |         | pr-2             | pr-1    |
| pr-5..pr-7    |              |         |                  | pr-v1   |
```

## Spec
- **Files**: new `workspace/programs/code-dev-meta-board.md`, `tools/board.py`.
- **Acceptance**:

  2. Data via PR-9.5 aggregator.
  3. `--project <slug>` scope; default = all projects.
