# pr-26 — Rename wave A (5 low-risk renames)

**Wave**: W4 · **Goals**: partial G.umb.05 · **Depends-on**: PR-12 (snapshot), PR-14 (routers)

## Why
R3 Wave-A renames touch programs with **low cross-ref count** (≤ 3 refs each). They make the user-facing surface align with the umbrella vocabulary from R3 without risking dispatch breakage.

## Evidence
- `helpers/cd-wf-c3-p2-rename-proposal.md` → full rename table, columns include "cross-ref count" used to pick A/B/C waves.
- `helpers/cd-tools-p3-migration.md` → Wave-1 strategy: rename in order of ascending cross-ref count.
- `helpers/cd-gap-c2-p4-failure-modes.md` → F-D2 risk class.

## Design notes
- 5 renames (drawn from `cd-wf-c3-p2-rename-proposal.md` table, top 5 by lowest cross-ref count):
  - `code-dev-impact` → `code-dev-knowledge-impact`
  - `code-dev-tour` → `code-dev-lifecycle-tour`
  - `code-dev-explain` → `code-dev-knowledge-explain`
  - `code-dev-metrics` → `code-dev-state-metrics`
  - `code-dev-undo` → `code-dev-state-undo`
- Each rename:
  1. Copy file under new name; preserve content.
  2. Replace old file with a 5-line alias stub: `# desc: DEPRECATED — alias for code-dev <new>. Removed next release.` + `EXEC(<new> $@)` + `LOG(WARN, "<old> deprecated")`.
  3. Update `tools/REGISTRY.json`.
  4. Update every internal `EXEC(<old>)` ref.
  5. Update tour cross-refs and cheatsheet auto-section (PR-34.5 input).
- Run `rename_snapshot diff --allow-rename old→new` to confirm clean.

## Pitfalls
- F-D2 stub typo → snapshot diff catches.
- F-F3 tour drift → PR-31.5 tour cross-ref lint.
- User muscle memory → alias stubs valid 1 release; drop in R3 Wave T5 (Post-1.0).

## Interface sketch

```
$ git mv workspace/programs/code-dev-impact.md workspace/programs/code-dev-knowledge-impact.md
$ cat > workspace/programs/code-dev-impact.md << EOF
# desc: DEPRECATED — alias for code-dev-knowledge-impact
EXEC(code-dev-knowledge-impact $@)
LOG(WARN, "code-dev-impact deprecated; use code-dev-knowledge-impact")
EOF
$ python3 tools/rename_snapshot.py diff --allow-rename code-dev-impact→code-dev-knowledge-impact
OK
```
