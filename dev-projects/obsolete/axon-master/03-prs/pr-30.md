# pr-30 — Per-mode budgets (full)

**Wave**: W4 · **Goals**: G.study.05 + plan budgets · **Depends-on**: PR-8 (study modes), PR-16 (plan modes), PR-20 (per-program budgets)

## Why
PR-20 declares per-program budgets but `study` and `plan` are now multi-mode. Each mode has its own expected cost (e.g. `study --mode=overview` is small, `--mode=deep` is large). PR-30 puts the per-mode budget block in place and gates overflow.

## Evidence
- `helpers/cd-study-c1-p2-modes-taxonomy.md` → per-mode token budgets table.
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.study.05.
- `helpers/cd-study-c4-p2-targets.md` → T-S1.7 (`--budget tokens=N` HALT-on-overflow).

## Design notes
- Each mode declares its budget in the program file:
  ```
  modes:
    overview:    {input-cap: 8000,  output-cap: 4000}
    subsystem:   {input-cap: 16000, output-cap: 6000}
    deep:        {input-cap: 32000, output-cap: 12000}
  plan-modes:
    tactical:    {input-cap: 4000, output-cap: 6000}
    strategic:   {input-cap: 4000, output-cap: 1500}
    operational: {input-cap: 4000, output-cap: 3000}
    decision:    {input-cap: 4000, output-cap: 4000}
  ```
- `tools/budget_lint.py` extended to validate per-mode blocks.
- Runtime: when actual exceeds cap by > 1.5× → HALT with partial output; `_index.md` row marks `status=partial`.

## Pitfalls
- Hard caps too tight on initial run → ship as warn first, flip to HALT after one round of measurements.
- Mode added but budget forgotten → lint requires block per declared mode.

## Interface sketch

```
$ python3 tools/budget_lint.py workspace/programs/code-dev-study.md
WARN: mode 'deep' missing input-cap declaration
FAIL: mode 'overview' declared output-cap 4000, observed run produced 5800 tokens (1.45x)
```
