# pr-29 — Behavioral T3 for 5 critical programs

**Wave**: W4 · **Goals**: G.test.03 · **Depends-on**: PR-1 (T1 infra), PR-20.8 (pr-review split)

## Why
T1 (structural) and idempotence harness (PR-25) cover shape; T3 covers **behavior** — given a fixture project state, does program X produce the expected change. Without T3 the 5 most critical programs are untested at the behavior level.

## Evidence
- `helpers/cd-gap-c2-p3-test-surface.md` → "Tier 3: behavioral T3 with mock-model harness".
- `helpers/cd-gap-c1-p3-goals-extracted.md` → G.test.03.

## Design notes
- Targets (5): `code-dev-plan`, `-study`, `-pr-ready`, `-resume`, `-migrate`.
- For each target:
  - 1+ fixture project under `tests/fixtures/programs/<target>/<case>/` with `before/` state and `expected/` after.
  - Mock-model harness `tests/_mock_model.py` stubs LLM calls with canned responses (recorded once, kept in fixture).
  - Test runs program in `mock` mode → diffs `after` against `expected`.
- Pass criterion: all "expected" files present with matching structure; key facts match.

## Pitfalls
- Mock-model drift (real model changes outputs) → fixtures recorded versioned; refresh PR documented in helpers.
- Side effects beyond fixture → tests use temp directory.

## Codebase grounding
- **new**: `tests/_mock_model.py` — monkeypatches the model boundary; canned response per `(program, fixture_id)` tuple loaded from `tests/fixtures/programs/<prog>/<case>/responses.jsonl`.
- **new**: `tests/test_behavior.py` — parametrized over 5 targets × ≥1 case each.
- **target programs**: [`code-dev-plan.md`](../../../../workspace/programs/code-dev-plan.md), [`code-dev-study.md`](../../../../workspace/programs/code-dev-study.md), [`code-dev-pr-ready.md`](../../../../workspace/programs/code-dev-pr-ready.md), [`code-dev-resume.md`](../../../../workspace/programs/code-dev-resume.md), `code-dev-migrate.md` (created in PR-3).
- **fixture layout**: `tests/fixtures/programs/<prog>/<case>/{before/, expected/, responses.jsonl}`.
- **pass criterion**: every `expected/<file>` exists in `<tmp-run>/<file>` with matching H2 structure + key-fact regex from `expected.facts.json`.
- **uses**: `conftest.run()` + `tmp_ws` (already in [`tests/conftest.py`](../../../../tests/conftest.py)).
