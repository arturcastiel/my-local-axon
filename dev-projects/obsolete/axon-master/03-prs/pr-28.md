# pr-28 — Rename wave C (10-15 high-risk renames in core flow)

**Wave**: W4 · **Goals**: partial G.umb.05 · **Depends-on**: PR-27

## Why
Final rename wave covers the **core daily-driver verbs** (`pr-*`, `plan`, `study`, `preflight`). Highest cross-ref count and highest user-impact; each gets per-rename HUMAN sign-off.

## Evidence
- `helpers/cd-wf-c3-p2-rename-proposal.md` → high-touch verbs listed.
- `helpers/cd-tools-p3-migration.md` → "Wave-3+: high-touch — per-rename review".

## Design notes
- 10-15 renames in `pr-*`, `plan`, `study`, `preflight`, `audit`, etc., e.g.:
  - `code-dev-pr` → `code-dev-pr-create`
  - `code-dev-pr-link` → `code-dev-pr-link` (kept; under `pr` umbrella)
  - `code-dev-preflight` → `code-dev-safety-preflight`
  - `code-dev-audit` → `code-dev-safety-audit`
  - `code-dev-check-structure` → `code-dev-safety-audit --structure`
  - `code-dev-self-review` → `code-dev-review --mode=self`
  - `code-dev-scope-check` → `code-dev-review --mode=scope`
  - `code-dev-diff` → `code-dev-review --mode=diff`
  - `code-dev-suggest-tests` → `code-dev-review --mode=tests`
  - `code-dev-reviewer-track` → `code-dev-knowledge-reviewer-track`
- **Per-rename** HUMAN sign-off, not batch.
- Old names kept as stubs (1 release).

Same pattern as PR-26/27. High-touch programs; HUMAN sign-off per rename. Note: some renames are absorbed (`scope-check`, `diff`, `suggest-tests` → `review --mode=<X>`); for those, the alias-stub becomes a 3-line wrapper that prepends `--mode`.
