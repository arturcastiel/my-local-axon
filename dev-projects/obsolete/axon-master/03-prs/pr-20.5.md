# pr-20.5 — Caches bundle (T-B1, T-B2, T-B3, T-B5)

**Wave**: W3 · **Goals**: T-B1, T-B2, T-B3, T-B5 (R2 #10-13) · **Depends-on**: PR-13 (usage to measure hit-rate)

## Why (problem statement)
R2 cycle-3 identified four caching opportunities clustered in the same hot loop (resume + pr-review):
- T-B1 session-scoped read cache (W:code-dev-cache-* hits)
- T-B2 resume briefing cache (mtime-keyed)
- T-B3 `reviewer-state.json` sidecar (persists per-project)
- T-B5 shadow LRU (in-process)

Each is small alone; landing them as one bundle pays the design cost once and lets PR-20.5's acceptance assert a measurable ≥ 50% hit on second resume in-session.

## Evidence (from studies)
- `helpers/cd-c4-p3-improvements.md` Rank 10-13 → T-B1, T-B2, T-B3, T-B5 (Wave 4: "caching the hot reads").
- `helpers/cd-c3-p2-workflows.md` → CW1..CW15 workflows that exploit caching.
- `helpers/cd-c3-p4-web-findings.md` → Anthropic prompt-caching + LRU patterns.
- `helpers/cd-c2-p1-internals.md` → resume reads `_meta.md` + last journal entries + last 3 PR specs every time.

## Design notes
- `tools/cd_cache.py`:
  - Session-scoped read cache backed by `W:code-dev-cache-<sha>` keys; mtime included in the cache key so stale reads invalidate.
  - Resume briefing cache: separate sub-cache keyed `resume:<project>:<meta-mtime>`.
  - Shadow LRU: max 32 entries, eviction by recency.
- `_reviewer-state.json` sidecar per project (not per chat) — persists pr-review progress across sessions: `{pr: 3, last-phase: 4, decisions: {...}}`. Atomic-write (PR-9).
- Invalidation rules:
  - File mtime change → invalidate.
  - Schema bump → invalidate all (one global epoch counter in `_meta.md`).
  - No TTL (every key is content-addressable).
- Hit-rate measurement: `tools/cd_cache.py` emits a `code-dev.cache.hit/miss` event (PR-15.5 bus); PR-21 aggregates.

## Pitfalls (from failure-mode catalog)
- Cache-invalidation bugs → strict (path, mtime) keys; no TTL; tested.
- **F-B2 race** → atomic_write for `reviewer-state.json`.
- LRU memory growth → 32-entry cap.

## Interface sketch
```text
$ code-dev resume          # first time this session
✓ resume briefing (1.4 KB, cache MISS — stored under resume:axon-master:1747538100)

$ code-dev resume          # again, same session
✓ resume briefing (1.4 KB, cache HIT)

$ code-dev pr-review 3 --phase=2
✓ reviewer-state.json loaded (cache HIT, last-phase=1)
```

## Spec (canonical)
- **Files**:
  - new: `tools/cd_cache.py`, `tests/test_cache.py`.
  - modified: `workspace/programs/code-dev-resume.md`, `code-dev-pr-review*` (post PR-20.8); per-project `_reviewer-state.json` schema documented.
- **Acceptance**:
  1. Session-scoped read cache hits ≥ 50% on second `code-dev resume` in same session.
  2. Resume briefing cache mtime-keyed; invalidates on `_meta.md` touch.
  3. `reviewer-state.json` sidecar persists across sessions; atomic.
  4. Shadow LRU bounded ≤ 32 entries.
  5. Cache invalidation on file mtime verified by test.
  6. Cache hit/miss events emitted on bus (PR-15.5).
  7. `tools/lint_paths.py` clean.
- **Rollback**: revert; reads go uncached again.
- **Owner**: AGENT writes; HUMAN runs `code-dev resume` twice in a session.

## Codebase grounding
- **new**: `tools/cd_cache.py` — functions: `get(key) / set(key, value, mtime_key) / invalidate(key) / lru_evict()`.
  - Backing store for session cache: `W:code-dev-cache-<key>` via [`tools/memory.py`](../../../../tools/memory.py) STORE/RETRIEVE.
  - Resume-briefing cache: file `_session.cache.json` per project (mtime-keyed on `_meta.md`, `_session.md`, `_actions.log`).
  - Reviewer-state: `_reviewer-state.json` per project (sidecar to existing `reviewer-state.md` referenced in [`workspace/programs/code-dev-next.md`](../../../../workspace/programs/code-dev-next.md)).
  - Shadow LRU: bounded `OrderedDict` (max 32 entries); overflow evicted, files preserved on disk.
- **modify**: [`workspace/programs/code-dev-resume.md`](../../../../workspace/programs/code-dev-resume.md) — first action: `TOOL(cd_cache, get, --key resume-briefing-{project})`; cache miss → build briefing then `set`.
- **modify**: post-PR-20.8 `code-dev-pr-review-p*.md` — use `_reviewer-state.json` for objection tracking.
- **modify**: [`tools/shadow.py`](../../../../tools/shadow.py) — add LRU eviction in `init` action; bounded by `--max-entries 32`.
- **new tests**: `tests/test_cache.py` — hit/miss ratio assertions; mtime-based invalidation verified.
