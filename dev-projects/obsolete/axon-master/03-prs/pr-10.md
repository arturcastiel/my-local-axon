# pr-10 — Governance `--strict` (full)

**Wave**: W2 · **Goals**: G.gov.04, G.gov.05 · **Depends-on**: PR-4 (schema + stub)

## Why (problem statement)
PR-4 shipped the governance schema, precedence doc, and a `pr ready --strict` stub. This PR puts teeth on it: strict mode actually gates on rules, stale studies, failing tests, missing acceptance fields. Without teeth the trace is just decoration and F-E2 (false-green) remains open.

## Evidence (from studies)
- `helpers/cd-gap-c3-p1-governance.md` → strict-mode acceptance: blocks on rules violation, stale relevant studies, failing tests, missing acceptance items.
- `helpers/cd-gap-c2-p4-failure-modes.md` → F-E2 "pr ready false-green" (Class E).
- `helpers/cd-study-c4-p2-targets.md` → T-S5.8 "`pr ready --strict` blocks merge gate when stale relevant studies exist".

## Design notes
- `tools/rules.py.evaluate(target, mode='strict')` returns `{ok: bool, blocks: [...], warns: [...]}`.
- Strict gates (any fail → block):
  1. Active rules in `safety/rules.md` applied to target's diff/spec.
  2. Relevant studies in `study/_index.md` with `staleness ∈ {stale, strict-block}` (thresholds set by PR-17).
  3. Latest `pytest` summary from `my-axon/log/` shows failures.
  4. PR's `_meta` block missing required acceptance items.
- `--strict-explain` flag: enumerates each gate with pass/fail and reason; non-fatal exit 0 so it can be piped.
- `code-dev-plan --strict` halts on rule conflict (instead of just filtering).

## Pitfalls (from failure-mode catalog)
- **F-E2 false-green** → multi-gate enforcement.
- **F-E4 stale study not flagged** → staleness gate.
- Strict false-positives → `--strict-explain` shows what is blocking.

## Interface sketch
```text
$ code-dev pr ready 3 --strict
✗ blocked by 2 gate(s):
  - rule R-002 (no migrations without `--restore` test) — PR-3 missing restore test
  - study staleness — auth-surface study 92d old, PR touches auth
fix or run with --strict-explain for full report.
exit 1

$ code-dev pr ready 3 --strict-explain
gate 1/4 rules:     ✗ R-002 violated
gate 2/4 staleness: ✗ auth-surface 92d
gate 3/4 tests:     ✓ last green at 2026-05-17T14:00
gate 4/4 acceptance:✓ 7/7 items present
```
