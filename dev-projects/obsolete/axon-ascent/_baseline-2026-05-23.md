## The headline finding
**12 robustness PRs left the usefulness score UNCHANGED at 72.6.**

Not because the work was wasted — structural integrity held (0 regressions),
93 tools, every gate now mechanical. But the *usefulness composite* is gated
by RUNTIME TELEMETRY (dispatch index, usage log, active plans, prompt log) —
all still zero. Code quality doesn't move that number; USE does.

The audit predicted this exactly:
> "Skipping the user-side actions and only building features is the failure
>  mode that makes self-improving systems look like vapourware. AXON has the
>  substrate; what's missing is use."

## Implication for axon-ascent
This is the empirical case for sequencing **1-telemetry FIRST**. Until
prompt-log/turn-log/dispatch/usage are non-zero, no later phase can be
measured, and the 72.6 won't move regardless of how many features ship.

`axon-audit --section 1b` is the project's tracking metric — re-run after
each phase and watch the composite finally move once telemetry is live.
