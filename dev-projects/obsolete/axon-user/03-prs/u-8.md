# U-8 — `pr_drift` heuristic + cheatsheet truncation + SCHEMA refs

**Wave**: U.C · **Severity addressed**: S2 (F-013, F-018) + S3 (F-015) · **Depends-on**: —
**Owner**: AGENT · **Status**: ready

## Why

Three small, independent surface fixes that share a "polish" theme:

1. **F-013** — `tools/pr_drift.py` silently passes acceptance criteria that
   produce no ≥4-char tokens (e.g. "Fix bug"). False-positive gate.
2. **F-018** — `workspace/AXON-DOCS-SCHEMA.md` has 3 dead cross-refs;
   `docgen_verify` is currently failing.
3. **F-015** — `tools/cheatsheet_gen.py` truncates verb descriptions at 54
   chars, cutting mid-word. Hides deprecation info from new users.

Grouped into one PR because each is < 5 LOC and they share the same
review surface (lint tools + doc fix).

## Evidence

- [findings/F-013](../findings/F-013-pr-drift-heuristic-gaps.md)
- [findings/F-015](../findings/F-015-cheatsheet-truncation.md)
- [findings/F-018](../findings/F-018-docs-schema-broken-refs.md)
- `python3 tools/docgen_verify.py` currently exits non-zero (3 violations).
- `head -20 workspace/AXON-DOCS-CHEATSHEET.md` shows the truncation.

## Design notes

- **pr_drift**: when token extraction returns empty, append to `unmet` with
  a `reason="no checkable tokens"` field rather than `continue`-ing.
- **cheatsheet_gen**: widen column to 76 chars, word-boundary truncate with
  ellipsis when over.
- **AXON-DOCS-SCHEMA**: fix two refs (`templates/v4-meta.md` path + remove
  `code-dev-migrate.md` ref since the program was absorbed in W4).

## Pitfalls

- **F-A (heuristic over-strictness)**: pr_drift may now flag previously-passing
  PRs. That's the intent. Acceptable: false-positives are easier to
  triage than false-negatives.
- **F-B (cheatsheet width)**: if any downstream tool parses the cheatsheet as
  fixed-width, this breaks it. None known.
