# U-15 — Schema docs name the new hierarchy

**Wave**: U.E · **Depends-on**: U-10, U-11, U-13 (templates must exist before docs name them)
**Status**: ready

## Why

The new directory shape (`02-roadmap.md`, `02-phases/`, `03-decisions/`)
must be documented in the canonical schema files or the next contributor
won't know it exists.

## Evidence

- `workspace/AXON-DOCS-SCHEMA.md` is the human-facing schema doc.
- `workspace/templates/v4-schema.md` is the machine-checkable spec.
- Neither mentions phases, roadmap, or ADRs today (because they didn't exist).

## Design notes

- Add a "Plan hierarchy" section to `AXON-DOCS-SCHEMA.md` with the v3-plan §1.3
  diagram and a 1-paragraph description of each tier.
- Add a "v4.2 — hierarchical planning" block to `v4-schema.md` (bump schema
  version label from `v4.1` to `v4.2` in the same edit).
- Schema version bump is non-breaking: v4.1 projects continue to parse;
  v4.2 just permits the new directory shape.

## Pitfalls

- **F (cheatsheet update needed)**: cheatsheet_gen reads schema-version,
  may need a re-run after this PR. Handled in U-V1.

## Interface sketch

`AXON-DOCS-SCHEMA.md` new section:

```
## Plan hierarchy (v4.2)

  02-roadmap.md             — strategic, multi-plan vision
  02-plan.md                — tactical, plan index
  02-phases/phase-*.md      — tactical, per-phase deep description
  03-prs/pr-*.md            — operational, per-PR atomic specs
  03-decisions/adr-*.md     — decision sidecar, ADRs
```

## Spec

- `workspace/AXON-DOCS-SCHEMA.md` — add hierarchy section.
