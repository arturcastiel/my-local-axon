# U-2 — `tools/session.py list` subcommand

**Wave**: U.A · **Severity addressed**: S1 (F-005) · **Depends-on**: —
**Owner**: AGENT writes; HUMAN runs `pytest tests/test_integration.py -k session`
**Status**: ready

## Why

PR-31 introduced `workspace/programs/code-dev-chats.md`. Its `list` sub calls
`TOOL(session, list, "--dir {sessions-dir}")`. `tools/session.py` exposes
`start`, `checkpoint`, `transition`, `recover`, `status` — but **no `list`**.
The entire `chats list` user-facing flow is dead-on-arrival.

This is one of the four S1 issues that blocked persona P4 (recovery-rio) at W-12.

## Evidence

- [findings/F-005](../findings/F-005-chats-list-missing.md)
- `tools/session.py` `main()` — no `list` branch.
- `workspace/programs/code-dev-chats.md#L23` — the dead call.

## Design notes

- Add `list_sessions(session_dir: Path) -> list[dict]` to `tools/session.py`.
- Reuse existing `_parse` helper to read `_session.md` headers.
- Output is JSON array on stdout (consistent with other tool subcommands).
- Each row: `{id, state, started, last-event, last-program}`.
- Filenames live one directory deep: `{session_dir}/{chat-id}/_session.md`.

## Pitfalls

- **Empty dir** → return `[]`, exit 0. Not an error.
- **Malformed `_session.md`** → skip with a `LOG(WARN)`; do not crash the list.
- **Symlinks** → follow by default (`Path.glob` does).

## Interface sketch

```bash
$ python3 tools/session.py list --dir my-axon/dev-projects/axon-user/sessions
[
  {"id": "s-2026-05-16-01", "state": "active",  "started": "...", "last-event": "...", "last-program": "code-dev-plan"},
  {"id": "s-2026-05-15-99", "state": "frozen",  "started": "...", "last-event": "...", "last-program": "code-dev-study"}
]
$ echo $?
0
```

## Spec

python3 tools/session.py list --dir /tmp/u2-test | python3 -c "
import json, sys
rows = json.load(sys.stdin)
assert len(rows) == 2
assert {r['state'] for r in rows} == {'active', 'frozen'}

  template; copy it.
