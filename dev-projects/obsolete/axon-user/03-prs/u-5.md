# U-5 — Absorbed-alias stubs use `STORE` + EXEC; add `diff` router branch

**Wave**: U.B · **Severity addressed**: S2 (F-009, F-010) · **Depends-on**: U-1
**Owner**: AGENT · **Status**: ready

## Why

PR-28 absorbed five verbs into `code-dev-review --mode=X`:
`scope-check`, `self-review`, `suggest-tests`, `diff`, `check-structure`.
The stubs forward via `EXEC(code-dev-review --mode=X $@)`. But
`code-dev-review.md` reads `sub ← RETRIEVE(W:code-dev-review-sub) | "all"` —
it does **not** parse CLI `--mode` flags. So the `--mode=X` flag is silently
dropped and the router falls through to `sub = "all"`, running every sub-program.

Additionally, the `diff` branch doesn't exist in the router at all
(`code-dev-review-diff.md` is orphaned).

Result: `code-dev scope-check` actually runs the *full* review (3-4× the
expected token cost). This is a token-economy regression invisible at audit time.

## Evidence

- [findings/F-009](../findings/F-009-review-alias-flag-vs-w-key.md)
- [findings/F-010](../findings/F-010-diff-no-router-branch.md)
- Inspection: `grep "sub ≡" workspace/programs/code-dev-review.md` shows
  `"scope"`, `"gaps"`, `"tests"` — no `"diff"`, no `--mode` parsing.

## Design notes

- Each absorbed-alias stub does the W:key store *before* the EXEC. The router
  reads the store and dispatches correctly.
- Naming alignment: `code-dev-self-review`'s alias should set the router key
  to `"self"`, matching the EXEC target name. The current router branches on
  `"gaps"` — rename the branch to `"self"` in `code-dev-review.md` for clarity.
- Add an `IF sub ≡ "diff" OR sub ≡ "all"` branch to the router.

## Pitfalls

- **D2 (state leak)**: `W:code-dev-review-sub` persists across calls. The
  router should `CLEAR(W:code-dev-review-sub)` on exit (it already does, via
  `CLEAR(W:active-program)` pattern — verify).
- **F-D1 (legacy naming)**: the `gaps` → `self` rename is user-visible to
  anyone who knew about the old `code-dev review gaps` form. Acceptable —
  `code-dev review` was new in W4 and the `gaps` name was never widely advertised.

## Interface sketch

```
$ code-dev scope-check
LOG(WARN, "alias-deprecated: use code-dev review --mode=scope")
STORE(W:code-dev-review-sub, "scope")
EXEC(code-dev-review)
# → only the SCOPE branch runs, not "all"
```

## Spec

### Files

- `workspace/programs/code-dev-scope-check.md`
- `workspace/programs/code-dev-self-review.md`
- `workspace/programs/code-dev-suggest-tests.md`
- `workspace/programs/code-dev-diff.md`
- `workspace/programs/code-dev-check-structure.md`
- `workspace/programs/code-dev-review.md` — add `diff` branch; rename `gaps` → `self`

### Diff (stub example — scope-check.md)

```diff
- LOG(WARN, "alias-deprecated: use code-dev-review --mode=scope")
- EXEC(code-dev-review --mode=scope $@)
+ LOG(WARN, "alias-deprecated: use code-dev review --mode=scope")
+ STORE(W:code-dev-review-sub, "scope")
+ EXEC(code-dev-review)
 DONE
```

For `check-structure.md`, target is `code-dev-safety-audit --structure`
(absorbed into safety-audit). Same pattern with `W:code-dev-safety-audit-sub`
or a flag handled by `code-dev-safety-audit.md` (see F-004 for the
safety-audit-side fix; this PR handles the stub side only).

### Diff (router — code-dev-review.md)

```diff
 IF sub ≡ "scope" OR sub ≡ "all" →
   → ""
   → "▼ SCOPE"
   EXEC(code-dev-review-scope)

-IF sub ≡ "gaps" OR sub ≡ "all" →
+IF sub ≡ "self" OR sub ≡ "all" →
   → ""
-  → "▼ GAPS (acceptance vs diff)"
+  → "▼ SELF-REVIEW (acceptance vs diff)"
   EXEC(code-dev-review-self)

 IF sub ≡ "tests" OR sub ≡ "all" →
   → ""
   → "▼ TEST SCENARIOS"
   EXEC(code-dev-review-tests)

+IF sub ≡ "diff" OR sub ≡ "all" →
+  → ""
+  → "▼ DIFF (file-level changes)"
+  EXEC(code-dev-review-diff)
+
 → "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
 CLEAR(W:active-program)
+CLEAR(W:code-dev-review-sub)
 DONE(code-dev-review)
```

### Acceptance

- `grep -n "EXEC(code-dev-review --mode=" workspace/programs/code-dev-*.md`
  returns no matches (all stubs use STORE+EXEC).
- `grep -n "sub ≡ \"diff\"" workspace/programs/code-dev-review.md` returns 1 match.
- Manual trace: invoking `code-dev scope-check` produces a single "▼ SCOPE"
  header (not all four sections).

### Rollback

`git revert`. Pre-U-5 state has the silent-fall-through behavior.

## Codebase grounding

- The router pattern (`sub ← RETRIEVE | "all"`) is well-established and used
  by other umbrella programs (e.g. `code-dev-meta`).
- `STORE` + `EXEC` pattern is the canonical way for stubs to forward
  arguments (used by `code-dev-pr.md` alias forwarding too).
