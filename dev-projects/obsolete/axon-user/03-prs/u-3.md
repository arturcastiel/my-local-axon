# U-3 — `code-dev-chats.md switch` arg signature fix

**Wave**: U.A · **Severity addressed**: S1 (F-006) · **Depends-on**: U-2
**Owner**: AGENT · **Status**: ready

## Why

PR-31's `code-dev-chats.md` `switch` block calls:

```
TOOL(session, transition,
     "--session {sessions-dir}/{chat-id}/_session.md --from frozen --to active")
```

But `tools/session.py transition` accepts `--path` + `--state`, not
`--session`/`--from`/`--to`. The switch is silently broken.

## Evidence

- [findings/F-006](../findings/F-006-chats-switch-signature.md)
- `tools/session.py` `transition(session_path, new_state)` signature.

## Design notes

Cheapest fix is **program-side**: align the program's TOOL call to the
existing tool surface. The alternative (extend `transition` to accept
`--from/--to`) is more invasive and not needed for the simple "switch chat"
use case.

## Pitfalls

- **Lost --from validation**: `--from frozen` was an implicit assertion the
  chat is currently frozen. We lose that defensive check. Acceptable for v1
  — the caller (`code-dev chats switch`) is expected to be a user-initiated
  switch, not a defensive transition.

## Interface sketch

```
TOOL(session, transition, "--path {sessions-dir}/{chat-id}/_session.md --state active")
```

## Spec

### Files

- `workspace/programs/code-dev-chats.md` lines 24-25 (the switch block).

```diff
 IF sub ≡ "switch":
-  TOOL(session, transition,
-       "--session {sessions-dir}/{chat-id}/_session.md --from frozen --to active")
+  TOOL(session, transition,
+       "--path {sessions-dir}/{chat-id}/_session.md --state active")
   STORE(W:code-dev-chat, "{chat-id}")
```

### Acceptance

- `grep "TOOL(session, transition" workspace/programs/code-dev-chats.md` shows
  the new `--path/--state` form, not `--session/--from/--to`.
- Manual: `python3 tools/session.py transition --path <fake>/_session.md --state active`
  exits 0 on a real session file.

## Codebase grounding

- `tools/session.py` `transition` enforces valid state transitions via a
  `TRANSITIONS` table. We rely on that for safety, not on `--from`.
- Other callers of `session.transition` already use `--path/--state` (e.g.
  `workspace/programs/code-dev-state-resume.md` post-PR-27).
