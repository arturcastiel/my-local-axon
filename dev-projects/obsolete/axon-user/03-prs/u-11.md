# U-11 — `--mode=tactical` writes `02-phases/phase-N-<slug>.md`

**Wave**: U.E · **Depends-on**: U-10 · **Status**: ready

## Why

Phases are the unit of human comprehension between "the whole plan" and "a
single PR". Today they exist only as table rows in `02-plan.md`. Promoting
them to first-class files lets each phase carry its own rationale, MUST/NICE
split, gates, and risk register — without bloating the parent plan doc.

## Evidence

- axon-master `03-plan.md` lines 24-78 — three waves rendered as three `##`
  blocks in one file. Lines per wave: 35 / 50 / 40. Phase doc would carry
  rationale that today gets squeezed into the table.
- v2 of this plan already has `## WAVE U.A / U.B / U.C / U.D` table headers
  — every one of them deserves its own file.

## Design notes

- `code-dev plan --mode=tactical` continues to write `02-plan.md` (now the
  *index* / envelope), and ALSO writes one `02-phases/phase-N-<slug>.md`
  per declared phase.
- Phase slug derives from the phase title's first-noun (best-effort heuristic);
  user is shown the proposed slugs and may override before write.
- Phase file template (new — `workspace/templates/v4-phase.md`):
  1. Phase envelope (number, title, status, parent plan + roadmap links)
  2. Why this phase
  3. PRs in this phase (table — slug, one-line, depends-on)
  4. MUST / NICE split
  5. Entry gate (from previous phase)
  6. Exit gate (to next phase)
  7. Phase-local risks
  8. Iteration log
- `02-plan.md` no longer embeds full wave tables — it points to phase files.

## Pitfalls

- **R (phase-slug collisions)**: two phases named "foundation". Mitigation:
  suffix with `-2`, surface to user.
- **F (legacy flat plan parsing)**: U-12 must handle BOTH flat (legacy) and
  hierarchical (new) plans. U-14 only enforces link discipline when
  `02-phases/` exists.
- **R (too-thin phase doc)**: phases with only 1 PR may not warrant a full
  doc. Threshold: ≥ 2 PRs per phase to create a phase file. Single-PR phases
  collapse into the plan index.

## Spec

### Files

- `workspace/programs/code-dev-plan.md` — extend tactical dispatch to write
  phase files + slim 02-plan.md to an index.
- `workspace/templates/v4-phase.md` — **NEW**.

### Diff sketch

```diff
 IF plan-mode ≡ "tactical" →
   # ... existing analysis ...
+  phase-list ← DERIVE(plan-sections, output="ordered list of phases with
+    {number, title, slug, prs, must, nice, entry-gate, exit-gate}")
+  → "Proposed phase files:"
+  ∀ p in phase-list → → "  02-phases/phase-{p.number}-{p.slug}.md  ({len(p.prs)} PRs)"
+  ack ← QUERY(user, default="y"): "Accept slugs? [y/edit]"
+  IF ack ≡ "edit" → phase-list ← interactive-rename(phase-list)
+  ∀ p in phase-list →
+    tpl      ← READ("{W:ws-os}/templates/v4-phase.md")
+    rendered ← RENDER(tpl, p)
+    WRITE("{project-dir}/02-phases/phase-{p.number}-{p.slug}.md", rendered)
+  # 02-plan.md becomes an INDEX of phase files instead of containing them
   WRITE("{project-dir}/02-plan.md", plan-index)
```
