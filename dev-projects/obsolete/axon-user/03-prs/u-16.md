# U-16 — `code-dev-plan.md` HELP rewrite — modes documented per tier

**Wave**: U.E · **Depends-on**: U-10, U-11, U-12, U-13 · **Status**: ready

## Why

After U-10..U-13 land, `code-dev plan --help` (which renders from the
`## HELP` block in `code-dev-plan.md`) is stale: it still describes modes
as "per-wave PR breakdown" / "high-level wave map" without saying which
*artifact* each mode emits. New users will run `--mode=decision` expecting
a plan and get an ADR. Fix the HELP.

## Evidence

- `workspace/programs/code-dev-plan.md` HELP block lines ~15-25.

## Design notes

- Rewrite each `--mode=X` tip line to name the tier and the file output.
- Add a new "Hierarchy" line in HELP pointing to AXON-DOCS-SCHEMA.md.

## Pitfalls

- **F (cheatsheet regen)**: cheatsheet_gen.py reads `# desc:`, which is not
  changed; only the HELP block changes. No regen needed.

## Interface sketch

```
## HELP
# usage:   code-dev plan [--mode=tactical|strategic|operational|decision] [--budget N] [--rule "<text>"]
# outputs (by mode):
#   tactical    → 02-plan.md  +  02-phases/phase-*.md         (default)
#   strategic   → 02-roadmap.md                                (vision tier)
#   operational → 03-prs/pr-*.md (delegates to code-dev pr)
#   decision    → 03-decisions/adr-NNN-<slug>.md               (ADR sidecar)
# tips:
#   - tactical (default): generates the plan envelope AND a phase doc per wave
#   - strategic: produces a roadmap that may host multiple plans / releases
#   - operational: ordered run-book — usually you'd run `code-dev pr` instead
#   - decision: durable ADR; surfaces alternatives and consequences
#   - hierarchy: see workspace/AXON-DOCS-SCHEMA.md "Plan hierarchy (v4.2)"
```
