# PR-014 — Unify the L: memory scope to a single backend (ADR-004)
Project:     axon-bugfix01 · Created: 2026-07-07 · Complexity: M · Depends on: none
Status:      implemented · AXON 8/10 · User 8/10 (standing)

## Summary
C9 (live-verified split-brain): config.md wrote L:inference-mode etc. via kv_store (diskcache)
while every kernel gate resolves L: via _longterm.py over workspace/memory/longterm/*.md — the
config wizard was a silent no-op for the autonomy behavior it was supposed to change; independently
confirmed for L:health-score (a frozen .md orphan with no current writer).

## Changes (decision D3 / ADR-004 accepted)
- Canonical + only L: backend: the .md longterm store (single reader F18, single writer F43).
- kv_store.py REFUSES the L: namespace loudly in both directions (exit 2, ADR pointer) — the
  split-brain cannot be silently recreated; non-scope data unaffected.
- config.md rewired to TOOL(memory, set --scope L ...) for all four write sites.
- health.py persist_health writes the canonical store (atomic + rollback) — the health-score
  orphan is now LIVE; stale kv-encoding doc lines corrected in two programs.
- tests/test_health_persist.py rewritten: canonical round-trip (what persist writes, the kernel's
  reader reads back), rollback sidecar, the L:-refusal guard both directions, non-scope pass-through;
  the 2026-06-18 date-drop regression intent preserved.

## Acceptance: config writes visible to kernel gates; kv-store L: refusal pinned; full suite
green (04-log).
