## Changes Required
### tools/synapse_suggest.py
- **What (corpus):** New default fallback `_load_candidates_from_programs()` reading
  workspace/programs/REGISTRY.json (program name, desc, status, synapse header fields where the
  registry carries them; dispatch-index phrases merged into desc tokens when available). Add
  `--corpus programs|tools` (default `programs`); `--registry` retained for the tools corpus.
  Degraded-record fabrication is labeled: fallback records carry `"contract": "degraded-fallback"`.
- **What (confidence):** Replace relative normalization with absolute calibration:
  `score = clamp(raw / sum(additive_weights), 0, 1)` (the additive weight mass is the theoretical
  max of `raw`). Ordering still by raw + FL-04 ladder. `raw` stays in the output envelope.
  `should_retrieve` keeps its 0.65 threshold — now meaningfully absolute.
- **What (docstring):** Rewrite the fallback paragraph to describe reality (no synapse-infer call).
- **Why:** Ranking "run the code review workflow" must return programs with honest confidence,
  not `lint-code` at a fabricated 1.0.

### tests/test_synapse_suggest_corpus.py (new) + existing test updates
- **What:** Fallback-corpus test (programs registry fixture → program names ranked, no tool names);
  calibration tests (weak overlap raw 0.119 → score ≈ 0.09, NOT 1.0; strong multi-signal → high
  absolute score; monotonicity: score order == raw order); degraded-record label present;
  any existing tests pinning score==1.0 updated to the calibrated expectation.
