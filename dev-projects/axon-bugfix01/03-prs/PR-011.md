## Summary
Root-cause fix for C6/C7/M7/M8/H19: the router matched only single hyphenated tokens while the
docs promised two-word commands. `code-dev pr list/drift/sync/export/suggest-reviewer` all fell
into `cmd ≡ "pr"` (PR-CREATE) silently; the 8 umbrella hubs, the meta cluster, and chats were
reachable only by typing literal filenames; and W:code-dev-cmd / W:library-dev-cmd had no
mechanical writer (correctness-by-agent-convention — the identical bug code-dev fixed once before,
reproduced verbatim in library-dev).

## Changes (as implemented)
- code-dev.md: explicit tokenizer bridge (`cmd ← W:code-dev-cmd | W:_arg1`, `sub ← ... | W:_arg2`);
  TWO-TOKEN RESOLUTION block before the single-token table (first-match-wins preserved):
  numeric sub = argument ("pr 3" → W:code-dev-pr-n, never a subcommand); existing
  `code-dev-{cmd}-{sub}.md` → direct EXEC. Single-token entries added for the 8 umbrella tokens
  + chats (bare form renders the hub; two-word forms short-circuit above).
- library-dev.md: same H19 bridge (subcommand ← W:library-dev-cmd | W:_arg1).
- tests/test_two_token_router.py: bridge pins, resolution-block pins (incl. the numeric guard and
  the BEFORE-the-table ordering pin — the original C6 failure mode), 17 two-token target-existence
  checks, 9 umbrella routing pins.
