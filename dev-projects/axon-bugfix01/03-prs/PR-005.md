## Summary
C1 (naming half): all 4 shipped canonical workflows call gate predicates that predicate.py never
registered (review.passes/has-objections, tests.pass, build.passes/fails, ctest.passes/fails,
ruff.*, pytest.*, api-diff.*, changelog.updated) — every run died at its first review gate as
ERROR:undefined_function, and workflow-run.md swallowed the error into a silent BREAK
(status:partial). Fix: complete the registered vocabulary, add an author-variant alias table,
rename the shipped YAMLs to canonical names, and make gate ERRORS halt the run loudly (distinct
from legitimately-false gates, which remain a legal terminal).

## Changes (as implemented)
- tools/predicate.py: 8 new fail-closed builtins (review.has-objections, build.failing,
  lint.passing/failing, api-diff.compatible/breaking-unbumped, changelog.updated,
  prs.all-implemented) + BUILTIN_ALIASES (15 entries) resolved at Call-eval.
- 4 YAMLs renamed to canonical vocabulary (mechanical, verified by trap test).
- workflow-run.md: gate-errors collected separately; error + no-next → FAIL block with the
  failing expressions; all-gates-false stays a legitimate terminal BREAK.
- tests/test_predicate_workflow_vocabulary.py: truth tables for all new builtins, alias-parity
  matrix, undefined-function still raises, and the C1 REGRESSION TRAP: every `name()` call in
  every shipped workflow YAML must be registered or aliased.

(measurement blocks of all 4 YAMLs — never evaluated at runtime, but doc-rot). Registered as
prs.all-implemented + alias rather than narrowing the trap.
