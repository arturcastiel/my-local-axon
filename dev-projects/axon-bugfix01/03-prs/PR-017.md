didn't work — the force-skip branch called only stale-downstream, never `done --force`, so the
skipped phase stayed not-done, the next phase's dep check raised "deps not done", and the
project was permanently stuck at the boundary with no exposed recovery path.

  FAILs loudly with the cause.
- tests/test_force_skip_unsticks.py: the audit's exact sandbox repro (old flow blocked → new
