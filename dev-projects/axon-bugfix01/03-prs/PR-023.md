# PR-023 — quickstart dispatcher + run-tests scope fix
Project:     axon-bugfix01 · Created: 2026-07-07 · Complexity: S · Depends on: none
Status:      implemented · AXON 8/10 · User 8/10 (standing)

## Summary
H15: quickstart's "## STEP DISPATCHER" was a comment with no routing code — execution fell
through into STEP 1 unconditionally and every step's re-entry looped back to fall through
again: the resume banner rendered "step N/7" while steps 2-7 of the flagship onboarding tour
were NEVER displayed. H16: run-tests read every result field from `R:result` — R: is not a
recognized memory scope (W:/L:/E:/local/ only) and appears nowhere else in the corpus.

## Changes
- quickstart.md: explicit dispatch (IF step ≡ N → GOTO(SECTION:QS-STEP-N)) ahead of STEP 1;
  6 section anchors added; step 1 stays the fall-through default.
- run-tests.md: `result ← TOOL(test-runner, run, ...)` local + all reads rebased; the W: store
  for last-test-result kept.
- tests/test_quickstart_runtests_fixes.py: per-step route + anchor pins, dispatch-precedes-
  step-1 ordering pin, phantom-scope pins.

## Acceptance: full suite green (04-log).
