## Summary
C1 (ctx half): workflow-run built the gate ctx as `{"state": {steps, history, last-step}}` only —
gates reading review/tests/build/audit/W/prs evaluated against nothing and silently resolved
False even with correct vocabulary (incl. multiple-code-dev's `W.mcd-decision` gate). Fix: a
deterministic ctx builder (`workflow_run.py ctx` — deterministic-spine constraint) composing the
full domain ctx from canonical state sources, wired into workflow-run.md.

## Canonical state sources (the write-side contract)
tests ← memory/longterm/last-test-run.json (test_runner) · W ← memory/working/*.md + *.json ·
review/audit/lint/api-diff ← memory/working/<domain>-state.json · build ← working/build-status.md ·
changelog ← working/changelog-updated.md · prs ← <project>/02-prs.md status lines.
All fail-soft: absent source → absent key → the builtin's fail-closed False.

## Discoveries during implementation (grammar realities the audit under-specified)
1. Only W/L/E/state/... are recognized ref SCOPES — `audit.open_findings` parses as
   Ref(state, [audit, open_findings]). Builder mirrors every domain under ctx["state"] so both
   builtins (top-level) and bare dotted refs (state-scoped) resolve.
2. Hyphens inside REF segments lex as SUBTRACTION (`audit.open-findings` = audit.open − findings).
   YAML refs renamed to `audit.open_findings`; the builder auto-aliases hyphenated state-file keys
   to underscore spellings one level deep.
