implements the decision correctly, but `tools/test_runner.py` never references aegis/grant/policy —

agent-invoked runs only.

## Changes Required
### tools/test_runner.py
- **What:** At `cmd_run` entry, determine actor: explicit `--actor agent|human` flag wins; default
  `agent` iff `CLAUDECODE=1` or `AXON_AGENT=1` in env, else `human`. For actor=agent: load
  `_policy.md` via `aegis_policy.load_policy(ROOT/"_policy.md")`, grant via
  `autonomous_mode.load_grant`, gate_green from crucible's last cached verdict (fail-closed None
  when absent/stale>24h), call `resolve("test-execution", ...)`. `allowed:false` → exit 3 with a
  JSON refusal envelope `{ok:false, refused:true, capability, reason}` and NO pytest run.
  actor=human → ungated (humans are never blocked from their own suite).
- **How:** Lazy imports; a `--policy-override` escape is NOT added (fail-closed by design).
- **Why:** Makes the documented green-only delegation real at the runner it governs.

### tests/test_test_runner_policy.py (new)
- **What:** resolve-wiring tests: agent actor + policy human → refused; + policy green-only, grant
  active, gate green → allowed; gate not green → refused; human actor → always runs (mock pytest
  invocation); env-based actor detection.
- **How:** Monkeypatch env, policy path, grant loader, cached-gate reader; stub run_pytest.

## Architecture Impact
First production caller of `aegis_policy.resolve()` for a GATED capability. The gate-green source is
the cached crucible verdict — a live full crucible run inside test_runner would be circular. Known
honest limitation: an agent passing `--actor human` bypasses the gate; actor identity is
protocol-level (recorded in the audit row) — noted in the PR body and the S1 doc-honesty line (PR-030).
