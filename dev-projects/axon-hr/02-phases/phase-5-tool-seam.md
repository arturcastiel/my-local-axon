# PR-005 — tools/hr_team.py (CLI + run_seats seam + audit-bundle writer) + tools/REGISTRY.json entry
Phase 2 plan detail · size M

## Goal
Ship the thin AXON tool layer for hr-team (ADR-002 D1): an argparse CLI that parses the full M1/M2/M3 invocation surface, emits a §4.3-shaped advisory verdict JSON with advisory_only:true + verdict_distribution, houses the ONE swappable seam run_seats(messages[])->responses[] (harness sub-agent fan-out backend now), and writes the per-call audit bundle to my-axon/hr-team/councils/{call-id}/ in 3 v1 persistence modes (ADR-008). Register it in tools/REGISTRY.json using the REAL AXON entry schema, and land the contract test in the same change (Core Rule 13 / R_NEW_NEEDS_TEST). This is the CLI entry + future-portability quarantine point: a provider-API/MCP backend can later replace the fan-out WITHOUT touching the selector/convener/deliberator neurons.

- tools/REGISTRY.json remains valid JSON; the new entry uses ONLY the real schema keys (script/status/category/purpose/desc/args/health) and `python3 tools/registry_drift.py check` reports no registry-orphan / status-mismatch for hr-team (script exists on disk, status ACTIVE).
- `python3 tools/liveness.py check` exits 0 (hr-team reached via the PR-004 program reference, or grandfathered in liveness-allow.txt).
- `python3 tools/health.py` (or the registry health probe) passes for hr-team using the declared probe (--persistence no-logs, expect 'advisory_only').
- No advisory_only:false path exists: there is no flag, env var, or code branch that can set advisory_only to anything but true (grep-assertable; INVARIANT). The tool performs no writes outside the my-axon/hr-team/councils/ audit bundle.

## Risks
- Foreign-schema trap: copying the HANDOFF §13 H2 entry verbatim (entrypoint/capabilities/writes/requires/tests) would corrupt tools/REGISTRY.json's contract — _axon_registry, registry_drift, health, mcp_server all read {script,status,category,purpose}. MUST translate to real keys; test_registry_entry_real_schema guards this.
- Liveness ordering: PR-005 ships the ACTIVE tool but PR-006 (menu/dispatch wiring) is later in the ADR-009 sequence; if the PR-004 router's reference to tools/hr_team.py isn't present at merge, `liveness check` BLOCKs. Mitigation: depend on PR-004's reference, else grandfather in liveness-allow.txt (and remove the line in PR-006).
- Audit-path divergence: writing to the §10.1/H1 path (my-axon/dev-projects/... or {W:myaxon-dev-projects}) instead of the ADR-008 path (my-axon/hr-team/councils/) would violate the locked decision and scatter private runtime data into a dev-project tree. Resolve strictly via _axon_paths.under_myaxon.
- advisory_only escape: any branch that lets advisory_only become false (a flag, an env, a merge of caller JSON) breaks the non-overridable legal boundary. Structurally hardcode true and assert via the no-override grep test.
- Council write-authority leak: run_seats or the writer touching anything outside the audit bundle violates the 'no write/tool authority except the audit bundle' invariant. Keep run_seats pure (no file/tool side effects); confine writes to write_audit_bundle under councils/{call-id}/.
- Re-implementing council logic in the tool: drifting from ADR-002 D1 (thin tool) into a Python re-implementation of selector/convener/deliberator (rejected option C) would couple logic to Python and duplicate the neurons. The tool orchestrates seam+emit+persist only.
- Coverage floor: if the contract test only smoke-runs the CLI, hr_team.py may fall under the 80% tools/ line floor (coverage_gate BLOCK). Ensure tests cover seam shape, all 3 persistence modes, call-id, manifest, and invocation-mode derivation.
- Non-determinism breaking the recorded fixture (ADR-005): call-id embeds a UTC timestamp; the regression golden must pin a fixed clock (or assert by regex/schema, not exact call-id) so the contract test is stable.
- checksums.sha256 self-reference: hashing the checksums file into itself, or hashing in a non-deterministic file order, makes the integrity file unverifiable — hash the OTHER bundle files in a sorted, stable order and exclude checksums.sha256 itself.
