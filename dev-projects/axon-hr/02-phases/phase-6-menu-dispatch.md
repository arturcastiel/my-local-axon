# PR-006 — menu [10] HR TEAM section + dispatch-phrases + dispatch-index rebuild
Phase 2 plan detail · size S

## Goal
Make the hr-team council discoverable through both AXON routing surfaces without any kernel edit: (1) add an ADDITIVE "HR TEAM" menu SECTION block at the next free number [10] in workspace/programs/menu.md, modeled exactly on the CODE DEVELOPMENT block (L263-276), with every existing section left byte-intact (Core Rule 12); (2) add a "# dispatch-phrases:" line to the hr-team.md router header (built by PR-004) carrying council vocabulary for TF-IDF smart-dispatch; (3) ensure that same council vocabulary ALSO lives in hr-team.md name/desc/PURPOSE because in-mode find-program does NOT read phrases (ADR-007; phrases-extension is the separate PR-008); (4) rebuild the smart-dispatch index from program source via tools/dispatch_index.py rebuild so the new phrases enter the match corpus. This is the discoverability/routing wiring layer of the ADR-009 bottom-up sequence (D2/ADR-003: harness section, NOT a kernel mode).

## Risks
- Core Rule 12 regression: an off-by-one or reflowed line could drop or mutate an existing menu section. Mitigation: additive insert only at the L276/L277 boundary; test (2) asserts all six pre-existing section markers still render.
- dispatch-phrases falling outside the 2000-char header window if hr-team.md grows a large budget/synapse block before # desc: — then dispatch_index.py silently won't index the phrases (no error, just missing routing). Mitigation: place the line directly after # desc: (top of file) and test (4) asserts it is within [:2000].
- find-program blind spot: if council vocabulary lives ONLY in phrases (not name/desc/PURPOSE), in-mode find-program can't route to hr-team until PR-008. Mitigation: test (5) pins vocab in desc + PURPOSE; this is an explicit ADR-007 invariant.
- Index drift if the rebuilt dispatch-index.json is not committed, or is committed against a workspace that lacks hr-team.md (PR-004 not merged first) → stale_extras/missing drift gate fails (registry_drift / dispatch_index check). Mitigation: depends_on PR-004; run `dispatch_index.py status` post-rebuild and commit only when ok:true.
- ADR-003/D2 violation risk: accidentally editing Mode-menu / mode-hints / kernel files to surface HR Team as a mode. Mitigation: acceptance criterion + diff review — only menu.md section text, hr-team.md header, and the regenerated index may change.
- TF-IDF confidence below 0.65 for some council phrasings (corpus is 170+ programs; 'review'/'panel' may collide with other programs). Mitigation: smoke-check dispatch.match in acceptance, not a hard committed assertion (sklearn may be absent in CI — dispatch.py L100 returns a fallback when sklearn missing); tune the phrases line if a target phrasing under-scores.
