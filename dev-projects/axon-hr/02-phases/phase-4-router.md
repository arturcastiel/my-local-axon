# PR-004 — hr-team router neuron — standalone + workflow-embeddable advisory-council dispatcher
Phase 2 plan detail · size M

## Goal
Author workspace/programs/hr-team.md: the ACTIVE router neuron of the hr-team program family. It carries the synapse header (domain deliberation, family [council,advisory,decision-support], role router/mutator, invocation_source [program,workflow]), parses the M1/M2/M3 arg surface into working memory, owns the subcommand routing table, drives the mandatory pipeline EXEC(hr-team-selector) → EXEC(hr-team-convener) → EXEC(hr-team-deliberator), STOREs the canonical HANDOFF §4.3 verdict to W:hr-team-result, and enforces the advisory_only boundary on BOTH call paths (standalone axon.py + workflow-embed H4 guard). Lands status:ACTIVE with a static router regression test (Core Rule 13 / ADR-009), depending on its three layer neurons (PR-001/002/003) already existing.

## Depends on
- PR-001 (hr-team-selector neuron — produces W:selector-result with seats/protocol/advisory_only)
- PR-002 (hr-team-convener neuron — consumes W:selector-result, calls the run_seats(messages[])->responses[] seam, produces W:council-transcript)
- PR-003 (hr-team-deliberator neuron — consumes W:council-transcript, aggregates to W:deliberator-verdict per HANDOFF §4.3)

## Risks
- Role field ambiguity: task says role `router/mutator` but the neuron only mutates working memory (W:hr-team-result) with no file/tool writes. Spec sets synapse `role: router` and documents the W-memory mutation in desc; if the registry schema rejects `router` as a role value, fall back to `mutator` with a `# role-note:` comment — verify against neuron-audit's accepted role enum before finalizing.
- Cross-PR W-key contract drift: the pipeline assumes PR-001 writes W:selector-result, PR-002 writes W:council-transcript, PR-003 writes W:deliberator-verdict. If sibling specs name these keys differently, the router EXEC chain breaks. Mitigation: these exact key names come straight from HANDOFF §13 H1 L1062-1067; PR-001/002/003 must honor them — flag as the shared contract in all four specs.
- H1 sketch passes positional args to EXEC (e.g. `EXEC(hr-team-selector, task, registry, mode, protocol)`); AXON convention (library-dev) is arg-less EXEC + sub-neuron RETRIEVE. The spec normalizes to arg-less and documents it; reviewer must confirm the layer neurons RETRIEVE their inputs rather than expecting positional binding.
- The advisory-boundary guard depends on the deliberator actually emitting advisory_only:true; if PR-003 regresses, the router FAILs loudly (intended) but the smoke-test fixture (ADR-005 recorded run) must include it so the contract test catches a missing-boundary regression at build time, not runtime.
- Static-only test gives no runtime guarantee the EXEC chain actually fires in a live boot; that end-to-end coverage belongs to PR-005's tools/hr_team.py contract test + the recorded worked-example-01 fixture (ADR-005), not this router PR. Spec notes the test scope boundary explicitly.
- find-program free-text routing to hr-team relies on dispatch-phrases, which ADR-007 defers to a SEPARATE PR; PR-004 must NOT add `# dispatch-phrases` wiring (out of scope) — council vocabulary lives only in name/desc/PURPOSE/help here until that PR lands.
