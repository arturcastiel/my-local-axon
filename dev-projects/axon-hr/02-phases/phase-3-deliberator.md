# PR-003 — hr-team-deliberator neuron — schema-validate seats, BPC + WSV aggregation, dissent re-round, emit §4.3 advisory verdict
Phase 2 plan detail · size M

## Goal
Author workspace/programs/hr-team-deliberator.md — Layer 3 of the hr-team council (the aggregation layer). It consumes the per-round/per-seat transcript produced by hr-team-convener (PR-002), schema-validates each seat's {reason,answer,confidence} JSON (one retry → seat-failure event), runs Balanced Position Calibration (randomized + reversed aggregation order, flag order sensitivity), applies Weighted Score Voting with 1/N (Brier-updatable) weights, classifies each dissent SUBSTANTIVE vs PREFERENTIAL, triggers a re-round ONLY on SUBSTANTIVE dissent, and EMITS the canonical HANDOFF §4.3 verdict object with advisory_only:true and verdict_distribution ALWAYS present, in opinion-neutral wording. Ships status:ACTIVE WITH its contract test in the same change (ADR-009 / Core Rule 13). No write/tool authority except participating in the per-call audit bundle.

 3. STEP 1 — SCHEMA VALIDATION (HANDOFF §4.3 step 2 + §9.2 validation rules 4-5). For each seat's final-round output: validate it is JSON containing required keys reason (str), answer (str), confidence (float in [0,1]). weighted-vote/debate seats additionally may carry option_scores + dissent_class (see weighted-vote.md L68-78). On invalid JSON or missing required key: issue ONE retry (re-invoke that one seat via the convener seam with a correction instruction) — spec the retry as a single EXEC back through the convener's run_seats seam, NOT a new model policy. If the retry is still invalid → record a `seat_failure` event and EXCLUDE that seat from aggregation (do not abort the whole council; record the exclusion in verdict + manifest). Emit `seat_responded {valid_schema:false}` then `seat_failure` per §10.3 event schema.
 4. STEP 2 — BALANCED POSITION CALIBRATION (HANDOFF §4.3 step 3; A §5.3 Wang 2023 order-bias mitigation). Aggregate the validated seat set TWICE: once in randomized seat/option order, once in reversed order. Compare the top answer of both passes. If the top answer differs between passes → set `order_sensitive: true` and add an `ORDER_SENSITIVITY` flag to verdict.warnings. BPC runs BEFORE the canonical WSV pass and is a diagnostic, not the selector of the winner.
 5. STEP 3 — WEIGHTED SCORE VOTING (HANDOFF §4.3 'Default WSV algorithm' + weighted-vote.md L84-98). Spec the algorithm in the neuron exactly: initialize weight_s = 1/N (N = validated seats), normalize so Σweight_s = 1; for each distinct answer_i: weighted_score(answer_i) = Σ_s weight_s · confidence_s · indicator(seat_s.answer == answer_i); winner = argmax weighted_score; aggregate_confidence = Σ_s weight_s · confidence_s. For protocols supplying option_scores (weighted-vote), also compute agg_score(option_i) = Σ_s weight_s · score_s(option_i). State the Brier update as DEFERRED-when-ground-truth-known: weight_s ← weight_s·(1 − brier_s) then renormalize — v1 has no live ground truth so weights stay 1/N at emit time; the neuron documents the hook but does not invent truth. Document the declared Arrow trade-off line: 'WSV uses cardinal confidence; IIA is the sacrificed criterion' (HANDOFF §4.3 social-choice trade-off; G §10.2-10.3).
 6. STEP 4 — DISSENT CLASSIFICATION (HANDOFF §4.3 dissent table + weighted-vote.md L124-128; H §12.7 IETF). For every seat whose answer ≠ winner (or that carried dissent_class): classify SUBSTANTIVE (claims a factual/logical/safety/scope/evidence failure in the majority) vs PREFERENTIAL (disagrees on values/priorities/risk-appetite/aesthetics). Prefer the seat's self-reported dissent_class when present and well-formed; otherwise classify from the seat's reason text against the table definitions. Record class + reason in the verdict's `dissents` array AND inline in each `verdict_distribution` entry's `dissent` field.
 7. STEP 5 — RE-ROUND LOGIC (HANDOFF §4.3 steps 6-7; adversarial.md/weighted-vote.md re-round rule). IF any SUBSTANTIVE dissent exists AND rounds_run < protocol.rounds_max → request ONE re-round back through the convener seam (EXEC(hr-team-convener, ...request-reround...)) then re-enter STEP 1 with the new transcript. If the substantive dissent persists after the round cap → DO NOT silently drop it: set verdict status `contested` and keep the dissent in verdict_distribution. PREFERENTIAL dissent NEVER triggers a re-round (recorded only). Spec a hard re-round ceiling = protocol.rounds_max (cap 5 per §11.4) to guarantee termination.
 8. STEP 6 — EMIT §4.3 VERDICT OBJECT. Build the verdict EXACTLY to the HANDOFF §4.3 schema with these MANDATORY-on-every-verdict keys (ADR-005): advisory_only:true, human_review_required:true, recommendation (winner), ranked_alternatives (answers desc by weighted_score), weighted_scores (map), aggregate_confidence, verdict_distribution (one entry per seat: {seat, answer, confidence, dissent}), dissents (array of {seat, class, reason}). verdict_distribution MUST be present even when one answer wins unanimously (HANDOFF §4.3 step 5 / §11 constraint 8 / FOMC §12.8). Add v1 extension fields: status ∈ {resolved, contested}, warnings (incl ORDER_SENSITIVITY / LOW_DIVERSITY propagated from selector / seat-failure exclusions), excluded_seats. STORE(W:deliberator-verdict, <object>); the router then does STORE(W:hr-team-result, W:deliberator-verdict) (H1 OUTPUT block).

If PR-002 isn't specced yet, pin a minimal transcript contract here and flag it as a cross-PR dependency.
- The retry-on-invalid-JSON step re-invokes a single seat — that requires calling back through the convener's run_seats seam (PR-002/PR-005), so the deliberator is not fully self-contained for retries. Spec the retry as 'request one re-emit via the convener seam', not a model call the deliberator owns, to keep the 3-layer separation (ADR-002) intact.

- Contract test is partly static-over-markdown (the neuron is interpreted, not Python) — assertions must be robust to harmless wording/whitespace edits or they become brittle. Mitigate by asserting on stable tokens (W: keys, SUBSTANTIVE/PREFERENTIAL, '1/N', verdict_distribution) rather than full sentences, per the test_code_dev_router.py precedent.
- Fixture normalization: worked-example-01's dash-style transcript lacks advisory_only and uses '4/5 consensus' prose; the fixture must be the NORMALIZED §4.3 JSON (advisory_only:true added) per ADR-005 — do not copy the prose verdict verbatim or the contract test's advisory_only/verdict_distribution assertions will encode a non-canonical shape.
- Liveness/orphan: hr-team-deliberator is invocation_source:[program] (called only by the PR-004 router). Until PR-004 lands, it has no caller — could trip R_NO_ORPHAN_TOOLS/liveness. Bottom-up order (ADR-009) accepts this gap closes at PR-004; if the gate fires before PR-004, the next-suggests/router stub may need a placeholder caller or the gate temporarily acknowledged.
