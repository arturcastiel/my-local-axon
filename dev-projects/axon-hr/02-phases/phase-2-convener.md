# PR-002 — hr-team-convener neuron — fragment assembly + parallel sealed Round 1 orchestration via run_seats seam
Phase 2 plan detail · size M

## Goal
Author workspace/programs/hr-team-convener.md as an ACTIVE markdown neuron (CONVENER layer / M2 of the 3-layer architecture) that takes the SELECTOR's roster/protocol/mode/tier decision and instantiates the council: assembles per-seat messages in the FIXED fragment order (PERSONA→GUARDRAIL→SKILLS→[de-biasing] | MODE→PROTOCOL→FORMAT | TASK) split across system/developer/user roles per HANDOFF §4.2/§9.2, narrates the tier pick + announces-then-acts (30s objection window, cancel-only), runs parallel SEALED Round 1 with NO cross-seat leakage THROUGH the single run_seats(messages[])->responses[] seam (ADR-002), re-injects each seat's system prompt FRESH at the start of every round (Sydney/§13.5 mitigation), and assigns per-seat model-tier (Haiku 4.5/Sonnet 4.6/Opus 4.8) + reasoning effort for intra-harness diversity recording model_variant per seat (ADR-006). The convener is opinion-neutral and has NO write/tool/decision authority; it emits the per-round transcript for the DELIBERATOR and never aggregates or editorialises. Ships with static router-style tests + a fragment-order assertion test (Core Rule 13 / ADR-009).

## Risks
- Asset-pack timing: the prompt fragments (personas/guardrails/modes/protocols/formats) under workspace/hr-team/prompts/ land in PR-007, AFTER this PR (ADR-009 bottom-up). Mitigation: the neuron references the ADR-004 paths and routes to the catalog-incomplete refusal when a required fragment is absent, so it is ACTIVE+green without the assets present; tests assert over neuron text only, not loaded fragments.
- SELECTOR/CONVENER tier-ownership ambiguity: the persona heuristic lives in convener.md but §4.1 has the SELECTOR lock framing/criteria and emit warnings. The spec resolves it by: SELECTOR chooses roster/protocol/tier; CONVENER narrates the chosen tier and only re-applies the convener.md heuristic when the caller deferred (--size absent) — must keep these from double-picking. If PR-001 already finalizes the tier, the convener must narrate-not-override.
- run_seats seam signature must match exactly what tools/hr_team.py (PR-005) and the harness fan-out implement: run_seats(messages[]) -> responses[]. If PR-005 names it differently, the neuron+tests need a coordinated rename. Pin the name in this spec so PR-005 follows it.
- Brand-string trap: writing claude-* ids anywhere in the neuron or tests would trip the commit-trailer/brand filter and the model-name memory rule. Use 'Haiku 4.5'/'Sonnet 4.6'/'Opus 4.8' only; a test asserts no 'claude-' literal leaks.
- Over-scoping: it is tempting to compute verdict_distribution or run Balanced Position Calibration in the convener. That belongs to PR-003 deliberator — keep the convener to orchestration + transcript only, else the 3-layer separation (the mandatory reader-contract invariant) collapses.
- Fragment-order test brittleness: index-based ordering assertions can break on benign rewording. Anchor them to the canonical literal chain ('PERSONA → GUARDRAIL → SKILLS → MODE → PROTOCOL → FORMAT → TASK') plus role-key checks rather than free-text proximity, to keep the test stable across copy edits.
