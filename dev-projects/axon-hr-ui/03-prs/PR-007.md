## Problem
The active-program interrupt gate (`KERNEL-SLIM.md:204`) skips only when `W:active-phase` contains
`:done` or `:failed` — it does NOT check `:aborted`. But the boot resume gate (`:661`) checks all three,
and the gate's own comment (`:255`) says "Gate does not fire if phase is ∅, :done, :failed, or :aborted".
So a phase ending in `:aborted` WRONGLY fires the "PROGRAM IN PROGRESS" interrupt — inconsistent with both
the boot gate and the documented contract. (The boot re-entry summary PR-007 also called for already exists
at BOOT.md STEP 3c — the live gap is just this marker normalization.)

## Approach (kernel — KERNEL-SLIM.md)
Add `AND phase not contains ":aborted"` to the interrupt-gate condition at line 204, making it consistent
with line 661 + the line-255 comment. One conjunct; no other behavior change.
