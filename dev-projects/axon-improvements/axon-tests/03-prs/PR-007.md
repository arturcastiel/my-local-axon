## Summary
Author the first dedicated behavioural test file for `axon/programs/
identity.md`. The identity gate is the most user-visible safety surface
(it controls what AXON reveals about its execution layer); today it has
no automated coverage. This PR pins ≥12 cases covering trigger phrases,
disclosure toggles, fallback behaviour, and banned subject forms.

## Goal Context
Study finding F-9: identity gate fully specified in program form, zero
tests. Co-output: AXON-DOCS-GOVERNANCE § "Identity gate" gets a
Guarded-by block listing every case in this file.

## Changes Required

### tests/test_identity_gate.py  (new)
- **What:** Parametrised test class `TestIdentityGate` driven by
  fixtures in `tests/fixtures/identity/`.
- **How:** Use mock-model harness; each fixture supplies one
  user-message → expected-response pair. Assertions:
  · gate fires on each of ≥8 canonical triggers ("what are you",
    "who made you", "what model", "are you GPT", "are you Claude",
    "which AI", "what's your model", "what model is this"),
  · with `L:disclose-execution-layer = true` and both host keys set,
    response contains "Powered by … · Running on …",
  · with `L:disclose-execution-layer = false`, response omits the
    execution-layer line,
  · with `L:host-model` unset, response falls back silently (no
    fabricated model name),
  · refuses to name a model not declared by the harness contract,
  · refuses banned subject forms ("As an AI…", "I'm a model…",
    "I'm just an AI…", "I'm an OpenAI/Anthropic model…").
- **Why:** Locks the canonical render so a careless edit to the
  program is caught.

### tests/fixtures/identity/  (new dir)
- **What:** ≥8 `.jsonl` fixtures, one per trigger phrase, plus
  `_harness.yaml` declaring the test's harness contract
  (host-harness, host-model).
- **How:** Mirror existing fixture style in `tests/fixtures/`.
- **Why:** Future identity-program edits drop one fixture, no code change.

### workspace/AXON-DOCS-GOVERNANCE.md
- **What:** Add (or extend) `## Identity gate` section ending in a
  `## Guarded by` block listing the 12 tests.
- **How:** Follow the PR-004 template; reference each `test_*` by
  fully-qualified id.
- **Why:** Co-output rule.
