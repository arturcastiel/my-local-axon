## Summary
The only autonomous git operation the kernel permits is the
`workspace-backup` program pushing the `my-axon/` repo to its origin.
Three preconditions guard it; today none are asserted. This PR
covers the program end-to-end and the tool that performs the push.

## Goal Context
Study finding F-10. Co-output: `AXON-DOCS-GOVERNANCE.md § Autonomous
push rule` Guarded-by block lists this PR's cases.

## Entry Conditions
- PR-005 merged (workflow harness available).
- `my-axon/` repo exists in test setup OR is mocked via a tmp git
  repo fixture.

## Changes Required

### tests/test_workspace_backup_tool.py  (new)
- **What:** Cover `tools/workspace_backup.py` (or equivalent) at unit
  level:
  · op outside `my-axon/` → raises violation, no push,
  · op inside `my-axon/` but caller != `workspace-backup` program →
    violation, no push,
  · remote != `origin` of `my-axon/` → violation,
  · all three preconditions met → push invoked (mocked subprocess),
  · push failure surfaces error + writes status file.

### tests/test_workflows/test_w15_workspace_backup.py  (new)
- **What:** Drive `workflow_test.py` against
  `w15-workspace-backup.flow.jsonl`:
  · boot tail-call fires when `myaxon-backup-enabled.md = true`,
  · tail-call skipped when key absent / false,
  · success path writes status file under `my-axon/memory/local/`,
  · error path surfaces in the boot menu summary.

### tests/fixtures/workflows/w15-workspace-backup.flow.jsonl  (new)
- **What:** Replay of the boot-tail backup happy path + one failure.

### workspace/AXON-DOCS-GOVERNANCE.md
- **What:** Add `## Autonomous push rule` section + `## Guarded by`.

## Architecture Impact
None to production code; tests must use a tmp git repo (init, fake
remote) so they never touch the real `my-axon/`.

  must use `tmp_path` + env override.
- ⚠ Boot tail-call uses the same flag-file the user enables; do not
  read the user's actual setting.
