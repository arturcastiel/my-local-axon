## Changes Required

### tests/test_rules/test_r3_arithmetic.py  (new)
- Positive: floating-point op without calculator → violation.
- Negative: integer-only op → no violation.
- Negative: float op routed through `tools/calculator.py` → no violation.
- Edge: scientific notation, mixed int/float, integer cast.

### tests/test_rules/test_r7_log_significant.py  (new)
- Significant event missing LOG → violation; with LOG → pass; edge:
  multi-event with one missing.

### tests/test_rules/test_r_coherence.py  (new)
- Plan/reply coherence: contradiction with plan → violation; aligned
  reply → pass; partial-step variant.

### tests/test_rules/test_r_drift_gate.py  (new)
- Migrate `TestDriftGate` from `tests/test_tools_kernel.py`; delete
  the old class. Add the missing negative cases.

### tests/test_rules/test_r_no_planned_tools.py  (new)
- Planned tool ≠ actually-called tool → violation; aligned → pass;
  edge: dispatch fallback resolves to a different tool.

### tests/test_rules/test_r_reasoning_trace.py  (new)
- Missing required reasoning blocks → violation; full trace → pass;
  partial-trace edge cases.

### tests/test_rules/test_r_tool_exists.py  (new)
- Plan references unknown tool → violation; known tool → pass; edge:
  registry-disabled tool (must violate).

### tests/test_rules/test_r_w_budget.py  (new)
- Workspace budget exceeded → violation; under budget → pass; edge:
  exact-boundary; edge: budget=0 always violates.

### tests/test_tools_kernel.py
- **What:** Remove `TestDriftGate` (now lives in
  `test_rules/test_r_drift_gate.py`).
- **Why:** Naming/location clarity; meta-enforcer expects rule tests
  under `tests/test_rules/`.

### workspace/AXON-DOCS-GOVERNANCE.md
- **What:** Extend `## Rule predicates` table — one row per rule with
