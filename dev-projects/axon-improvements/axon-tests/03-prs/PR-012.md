## Summary
End-to-end coverage of `tools/verify.py` (or `tools/axon_audit.py`'s
verify entry) — feed real program files each crafted to trigger
exactly one rule, then assert the violation list matches.

## Goal Context
Per-rule predicates are covered in PR-009/PR-011, but the integration
path (load program → run all rules → produce report) is not. This PR
fills that gap with one fixture per rule.

## Entry Conditions
- PR-011 merged.
- `tools/verify.py` is stable; its return shape documented.

## Changes Required

### tests/test_verify_integration.py  (new)
- **What:** Fold legacy `TestVerify` cases in; add one e2e case per
  rule:
  · build a fixture program that triggers exactly one rule,
  · run the verifier,
  · assert violations list contains exactly that rule_id with the
    documented severity,
  · assert no spurious violations from other rules.
- **How:** Parametrised over the 9 rules.

### tests/fixtures/verify_programs/  (new dir)
- **What:** 9 minimal `.md` program files, one per rule.
- **Why:** Realistic input rather than synthetic ctx.

### workspace/AXON-DOCS-GOVERNANCE.md
- **What:** Add `## Verifier flow` section + Guarded-by block.
