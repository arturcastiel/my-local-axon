## Summary
Extend `test_compiled_regression.py` + `test_dispatch.py` and add a
new `test_compile_optimizer.py` to fill the dispatch + compiler edge
cases noted in F-8.

## Goal Context
Compiler and dispatch are the runtime spine of PYTHON_FAST output —
their corner cases are sparsely tested.

## Entry Conditions
- PR-001 merged (full pytest in CI).
- Compiler + dispatch source code unchanged since F-8 audit.

## Changes Required

### tests/test_compiled_regression.py
- **What:** Add cases for
  · idempotence: compile(compile(x)) == compile(x),
  · identical-prompt collision: same input from two programs →
    same compiled artifact id,
  · missing source: graceful error, no crash,
  · stale cache: source mtime > artifact mtime triggers rebuild.

### tests/test_dispatch.py
- **What:** Add cases for
  · low-confidence dispatch falls back to ASK,
  · `prefer-compiled` override beats heuristic,
  · feedback adjustment: positive feedback lifts ranking,
  · negative feedback demotes the same dispatch on next call.

### tests/test_compile_optimizer.py  (new)
- **What:** If the optimizer module exists, cover its hot paths
  (constant folding, dead-branch elimination, prompt-dedup). If it
  doesn't, this file documents the gap with a single skipped test
  and an XFAIL note pointing at the next phase.

### workspace/AXON-DOCS-COMPILER.md
- **What:** Append Guarded-by block listing the three test files.

## Architecture Impact
None — pure test additions.

## Acceptance Criteria
- ✓ `pytest tests/test_compiled_regression.py tests/test_dispatch.py
  tests/test_compile_optimizer.py -v` shows ≥15 passing.

- ✓ Mutating dispatch confidence threshold causes ≥1 case to fail.

## Risks & Gotchas
- ⚠ Compile artifacts are written to a cache dir — use tmp_path so
  the real cache isn't polluted.
- ⚠ Optimizer module presence is uncertain; the test file must
  detect-and-skip gracefully.
- ⚠ Feedback adjustment is stateful — reset between cases.
