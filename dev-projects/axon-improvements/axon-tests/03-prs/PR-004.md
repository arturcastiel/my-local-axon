# PR-004 — Doc co-output template + advisory linter
Project:     axon-tests
Created:     2026-05-16
Complexity:  M
Depends on:  none
Status:      not-started
AXON score:  8/10

## Summary
Establish the **doc co-output convention**: every `AXON-DOCS-*.md`
section that describes testable behaviour MUST end in a `## Guarded
by` block listing the test ids that pin it. This PR adds the
canonical page template, extends `tools/docgen_verify.py` to scan
for the block, and emits **WARN** (not BLOCK) in this PR. PR-018
flips to blocking. The PR establishes the rhythm that every Phase-3
test PR will follow.

## Goal Context
From 01-study.md F-12: "Docs ↔ tests are not cross-linked." The
project's `masterplan.md` commits to tests+docs as co-outputs.

## Entry Conditions
- `tools/docgen_verify.py` exists (it does; ~PR-026 in core axon).
- `workspace/AXON-DOCS-*.md` files are the canonical doc surface.

## Changes Required

### workspace/templates/axon-docs-page.md  (new)
- **What:** Canonical AXON-DOCS-* page template with required sections.
- **How:**
  ```markdown
  # AXON-DOCS — <AREA>

  > by Dr. Artur Castiel Reis de Souza · arturcastiel.github.io

  ## Purpose
  <one paragraph: what this subsystem does, why it exists>

  ## Contract
  <what callers can rely on; invariants>

  ## Behaviour
  <how it behaves under normal + edge cases>

  ## Guarded by
  <list of test ids that pin the behaviour above>
  - `tests/test_<file>.py::<class>::<test>`
  - …

  ## See also
  <cross-refs to other AXON-DOCS pages>
  ```
- **Why:** Single source of truth for what a "complete" doc page
  looks like; PR-018 makes "## Guarded by" non-empty mandatory.

### tools/docgen_verify.py
- **What:** Add a `check-doc-coverage` subcommand.
- **How:** Walks `workspace/AXON-DOCS-*.md`, for each file emits one
  row to stdout JSON:
  · path,
  · has_guarded_by (bool),
  · guarded_by_count (int, 0 if section missing),
  · status: `OK` / `WARN` / `FAIL`.
  This PR emits `WARN` for missing or empty `## Guarded by`. PR-018
  changes `WARN` → `FAIL` and wires CI.
- **Why:** Phased rollout — warn while migrating existing 11 doc
  pages, then enforce.

### tools/REGISTRY.json
- **What:** Update `docgen_verify` entry to advertise the new
  subcommand.

### workspace/AXON-DOCS-SCHEMA.md
- **What:** Add a section "Doc-page convention" describing the
  template and the Guarded-by contract.

### workspace/AXON-DOCS-TESTING.md
- **What:** Extend with a "## Guarded by" block listing tests of the
  TESTING doc itself (T1/T2/T3 cases). Sets the example.

### workspace/AXON-DOCS-CHEATSHEET.md
- **What:** Same — add Guarded-by listing tour-lint tests.

### tests/test_doc_coverage.py  (new)
- **What:** 4 cases.
  1. Template file exists and contains all required H2 sections.
  2. `docgen_verify check-doc-coverage` returns JSON with one row
     per AXON-DOCS-* file.
  3. A doc page missing `## Guarded by` produces a WARN row.
  4. A doc page with a populated Guarded-by produces OK.
- **Why:** Pins the linter so PR-018 can flip the toggle safely.

## Architecture Impact
Convention-only. No kernel impact. Adds one tool subcommand.

## Acceptance Criteria
- ✓ Running `python3 tools/docgen_verify.py check-doc-coverage`
  prints JSON with 11 rows (one per existing AXON-DOCS-* file).
- ✓ At least 2 rows have `status=OK` (the two doc pages updated in
  this PR).
- ✓ The other 9 rows have `status=WARN` (not yet migrated).
- ✓ The 4 new tests pass.
- ✓ The schema doc explains the convention in ≤ 1 page.

## Risks & Gotchas
- ⚠ The template proposes section names; downstream PRs will adopt
  them. Don't churn the names after this PR lands.
- ⚠ `docgen_verify` may already do schema work — read its
  existing subcommands before adding `check-doc-coverage` to avoid
  duplication. If a similar check exists, extend rather than add.
- ⚠ AXON-DOCS.md (the master index, 674 lines) is exempt — it's
  an index, not a doc page. The linter excludes it by name.

## Tests Added
4 cases in `tests/test_doc_coverage.py`.

## Rollout / Rollback
- Rollout: merge; existing pages emit WARN until later PRs migrate
  each one alongside its test work.
- Rollback: remove the subcommand + template + tests.

## Out of Scope
- Making the gate blocking (PR-018).
- Migrating every existing doc page (each later PR migrates its own
  area's doc; PR-021 sweeps anything left).
