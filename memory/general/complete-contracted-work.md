Complete the contracted work IN FULL — never substitute a skip/summary/partial for a specified full render, list, sweep, or step-sequence, and never rationalize the shortcut as token-saving/efficiency (that rationalization is the tell). Skipping is a false economy: it forces a re-run (the full work happens anyway + wasted turns + owner time) or ships a fabricated/incomplete result. Same root as menu-truncation, the MR!141 iter-1 'did the steps in my head', and summarizing-not-doing. If genuinely limited, say so + ASK; never silently shorten.
