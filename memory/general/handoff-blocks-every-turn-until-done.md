OWNER CORRECTION (2026-06-10, emphatic): when work produces a HUMAN ACTION (script to run, kernel patch, merge decision), the handoff must be (1) rendered as a BORDERED, highlighted block — never a sentence or a file-caption — AND (2) RE-SURFACED in every subsequent response while it remains pending, so the owner cannot miss it. The PR-2 kernel script was delivered via file+prose and the owner nearly missed it. Extends [[surface-handoffs-via-tool]]: that rule covers the RENDER; this adds PERSISTENCE-until-done. Applies to TOOL(human-handoff) and any owner-executed instrument.
